# 安全知识库结构规范

## 1. 目录定位

- `cases/`: **安全案例库**。存储国内外真实的、典型的安全事件复盘，或行业最佳实践。
  - `china/`: 国内案例。
  - `overseas/`: 海外案例。
  - `vertical/`: 垂直行业（金融、医疗、电商等）专项案例。
- `weapons/`: **安全武器库**。存储具体的防御手段、工具、算法或治理模式。
- `guides/`: **方法论指南**。存储通用的安全分析框架、风险评估标准、合规对齐手册。
- `schools/`: **理论流派**。存储安全领域的核心设计哲学（如零信任、纵深防御）。
- `modules/`: **模块化知识**。将相关联的知识点串联成系统性的学习或决策路径。

## 2. 文件模板要求

### 2.1 案例 (Cases) 模板
```markdown
# 案例名称
- **所属领域**: [信息安全/业务安全/AI安全]
- **威胁类型**: [如 SQL注入/反薅羊毛/模型劫持]
- **事件回顾**: 简述发生了什么。
- **Red View (利用路径)**: 攻击者是如何进入并获利的。
- **Blue View (防御缺失)**: 当时缺失了哪些关键防线。
- **策略借鉴**: 对后续系统设计的 3 个关键建议。
```

### 2.2 武器 (Weapons) 模板
```markdown
# 武器/策略名称
- **分类**: [检测/预防/响应]
- **适用场景**: [解决什么具体问题]
- **工作原理**: 核心技术逻辑。
- **执行建议**: 接入成本、对业务的影响、推荐库/工具。
```

### 2.3 指南 (Guides) 模板
```markdown
# 方法论名称
- **目标**: 解决什么维度的决策问题。
- **核心步骤**: 1, 2, 3...
- **判定标准**: 如何衡量诊断结果。
```

## 3. 标签规范 (Tags)
每个知识文件必须包含元数据，方便 Agent 进行多维检索。
- `domain`: [infosec, bizsec, aisec]
- `stage`: [design, deployment, monitoring, response]
- `priority`: [high, medium, low]
