# 某头部电商平台“双十一”薅羊毛对抗案例

- **所属领域**: 业务安全 (Bizsec)
- **威胁类型**: 营销反欺诈 / 薅羊毛 (Wool-Pulling)
- **事件回顾**: 2023年“双十一”期间，某头部电商平台发放大量“满200减100”的大额优惠券。黑产团伙通过自动化工具秒杀优惠券，并利用大量虚假账号（垃圾注册）进行囤券和套现，导致真实用户无法领券，平台营销资金损失超千万。

## Red View (利用路径)
1. **自动化秒杀**: 使用模拟器和脚本绕过前端限制，实现毫秒级领券。
2. **账号矩阵**: 通过“接码平台”批量注册虚假账号，绕过单账号限领策略。
3. **众包攻击**: 利用“抢券群”组织大量真实用户配合自动化工具，形成“真假混合”流量，增加识别难度。
4. **设备绕过**: 使用改机工具修改设备指纹（IMEI、UUID），规避设备层面的频控。

## Blue View (防御缺失)
1. **验证码过时**: 使用了简单的图形验证码，已被 OCR 脚本轻松破解。
2. **风险评估滞后**: 领券接口未集成实时的环境检测（如：检测是否为模拟器、是否有改机工具）。
3. **限流策略单一**: 仅基于 UID 限流，未建立基于设备指纹、网络 IP 段和行为模式的综合画像。
4. **缺乏关联分析**: 未能识别出大量账号在同一基站、同一时间段、相似操作路径的聚集性特征。

## 策略借鉴
1. **全链路防御**: 在领券、下单、支付各个环节部署验证。采用“无感验证 + 强校验”结合模式。
2. **设备指纹与环境监测**: 强制集成安全 SDK，识别环境风险（ROOT、模拟器、调试模式），并作为领券的核心准入条件。
3. **动态风险评分**: 建立基于“账号权重 + 设备信用 + 行为特征”的实时评分引擎。对高危操作要求二次验证，对极高危操作直接拦截。
4. **营销策略耦合**: 在优惠券设计阶段引入“阶梯式发放”和“定向分发”，减少大额券对全量的暴露面。
