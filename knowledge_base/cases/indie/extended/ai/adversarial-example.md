## 一句话风险

攻击者通过对输入添加人类难以察觉的微小扰动，使模型产生错误预测，导致安全关键系统失效或被绕过。

## 一分钟识别清单

- [ ] 模型输入为高维连续数据（图像、音频、文本嵌入），且攻击者可控制输入
- [ ] 模型在正常样本上表现良好，但在特定样本上突然失效
- [ ] 安全关键应用场景（自动驾驶、恶意软件检测、身份认证）

## 一句话防御

采用对抗训练、输入预处理、模型集成和异常检测等组合策略，提升模型对对抗性扰动的鲁棒性。

## 典型攻击示例

```text
攻击场景 1: 图像分类绕过
原始输入: 熊猫图像 → 模型预测: "熊猫" (置信度 57.7%)
对抗样本: 熊猫图像 + 精心计算的微小噪声 → 模型预测: "长臂猿" (置信度 99.3%)
人眼观察: 两张图像几乎完全相同

攻击场景 2: 恶意软件检测绕过
攻击流程:
1. 攻击者获取恶意软件检测模型的访问权限
2. 使用梯度优化方法生成对抗性扰动
3. 将扰动添加到恶意软件二进制文件的非关键区域
4. 原本被检测为"恶意"的样本被分类为"良性"

攻击场景 3: 自驾车辆攻击
攻击方式: 在交通标志上贴上特制贴纸（对抗性补丁）
结果: "停止"标志被车辆识别为"限速45"或其他标志
```

## 防护策略

1. **对抗训练**: 训练时加入对抗样本，提升模型鲁棒性
2. **输入预处理**: 使用去噪、压缩等技术消除对抗性扰动
3. **模型集成**: 使用多个模型投票决策，增加攻击难度
4. **异常检测**: 部署独立的异常检测模块，识别可疑输入
5. **认证防御**: 使用可证明的防御方法，在理论上保证鲁棒性
