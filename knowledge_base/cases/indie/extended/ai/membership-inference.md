# 成员推理攻击

## 元数据

- **ID**: `membership-inference`
- **分类**: AI安全 > 隐私攻击
- **tier适用**: L1 ✅
- **创建时间**: 2024-01
- **最后更新**: 2024-01

## 一句话风险

攻击者通过分析模型输出置信度，推断特定数据样本是否在模型的训练集中，导致训练数据隐私泄露。

## 一分钟识别清单

- [ ] 模型API返回高精度概率值或置信度分数（而非仅返回最终预测标签）
- [ ] 攻击者能够批量查询模型，观察不同输入的输出差异
- [ ] 训练数据包含敏感个人信息（医疗记录、金融数据、私密图像）

## 一句话防御

限制模型输出的粒度、应用差分隐私训练技术、使用正则化降低过拟合，降低模型对训练样本的"记忆"程度。

## 推荐工具链接

- [PrivacyRaven](https://github.com/trailofbits/PrivacyRaven) - 隐私攻击评估工具
- [TensorFlow Privacy](https://github.com/tensorflow/privacy) - 差分隐私训练库
- [Opacus](https://github.com/pytorch/opacus) - PyTorch差分隐私训练

## 典型攻击示例

```text
攻击前提:
- 攻击者拥有黑盒访问模型的权限
- 模型输出完整的概率分布或高精度置信度

攻击流程:
1. 攻击者收集已知在训练集中的样本（影子数据集）
2. 训练一个"攻击模型"，学习区分"成员"与"非成员"的输出特征
3. 对目标样本查询模型，获取输出置信度
4. 攻击模型判断该样本是否在训练集中

典型结果:
- 训练样本的置信度通常显著高于非训练样本
- 在过拟合严重的模型上，攻击准确率可达90%+
```

## 防护策略

1. **输出限制**: 仅返回离散预测标签，不暴露高精度概率值
2. **差分隐私**: 训练时添加噪声，防止模型过度记忆单个样本
3. **正则化**: 使用L2正则化、Dropout等技术降低过拟合
4. **模型架构**: 避免过度参数化，减少模型对训练数据的记忆容量
5. **查询监控**: 检测异常查询模式，限制单用户查询频率
