# 模型逆向攻击

## 元数据

- **ID**: `model-inversion`

- **tier适用**: L1 ✅
- **创建时间**: 2024-01

## 一句话风险

攻击者通过反复查询模型并分析输入输出关系，重构训练数据特征或提取完整训练样本，导致数据隐私泄露。

## 一分钟识别清单

- [ ] 模型暴露详细的预测概率或中间层输出
- [ ] 攻击者能够大量自动化查询模型
- [ ] 训练数据为敏感图像、人脸识别数据或医疗影像

## 一句话防御

限制模型输出粒度、应用差分隐私、实施查询频率限制和异常检测，增加攻击者的重构成本。

## 典型攻击示例

```text
攻击场景 1: 人脸识别模型逆向
攻击流程:
1. 攻击者向人脸识别模型输入大量合成人脸图像
2. 观察模型输出的置信度变化
3. 使用梯度下降优化输入图像，使置信度最大化
4. 最终重构出训练集中某人的真实人脸图像

攻击场景 2: 医疗诊断模型逆向
攻击流程:
1. 攻击者已知模型类型和部分公开特征
2. 系统性遍历输入空间，记录输出概率
3. 根据模型决策边界推断训练数据的分布特征
4. 重构训练数据中的敏感属性（如特定患者的疾病标志）
```

## 防护策略

1. **输出限制**: 仅返回最终决策，不暴露中间层激活或高精度概率
2. **差分隐私**: 训练时添加噪声，使单个样本对模型影响可忽略
3. **查询限制**: 实施严格的API调用频率限制和异常检测
4. **模型架构优化**: 避免过度参数化，减少模型对训练数据的记忆
5. **对抗训练**: 使用逆向攻击样本增强模型的抗重构能力
