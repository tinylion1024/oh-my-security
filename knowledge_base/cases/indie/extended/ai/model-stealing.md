# 模型窃取

## 元数据

- **ID**: `model-stealing`
- **分类**: AI安全 > 知识产权保护
- **tier适用**: L1 ✅
- **创建时间**: 2024-01
- **最后更新**: 2024-01

## 一句话风险

攻击者通过大量查询商业模型API，训练出功能相近的替代模型，窃取专有AI能力或绕过付费访问限制。

## 一分钟识别清单

- [ ] 模型以API形式提供，用户可获取输入输出对
- [ ] 检测到异常高频查询或系统性的输入采样模式
- [ ] 商业模型具有高价值（专用算法、专有数据训练、高研发成本）

## 一句话防御

实施严格的查询频率限制、返回有噪声或低精度输出、添加水印追踪、使用差分隐私技术，增加窃取成本。

## 典型攻击示例

```text
攻击场景: 商业图像分类模型窃取

攻击流程:
1. 攻击者订阅目标模型API服务
2. 生成大量合成或公开数据集样本
3. 系统性查询API，收集输入-输出对（标签或概率分布）
4. 使用收集的数据训练"学生模型"，模拟"教师模型"行为
5. 学生模型在相同任务上达到与教师模型相近的性能

攻击效果:
- 原模型训练成本: 数百万美元 + 数月时间
- 窃取成本: API调用费用 + 数天训练时间
- 性能差距: 学生模型可达到教师模型85%-95%的性能

攻击场景: LLM能力窃取
攻击者向商业LLM API发送大量prompt，收集高质量回复，用于训练自己的开源模型。
```

## 防护策略

1. **查询限制**: 实施严格的频率限制和配额管理
2. **输出扰动**: 返回低精度结果或添加噪声，降低替代模型质量
3. **输入检测**: 识别系统性的采样模式，阻止可疑查询
4. **模型水印**: 在模型输出中嵌入不可见水印，追踪窃取行为
5. **差分隐私**: 训练时添加噪声，使模型无法被精确复制
6. **服务条款**: 法律层面明确禁止模型窃取，保留追责权利
