## L1 独立开发者版（速查版）

### 你会遭遇什么（一句话）
手机备份到云端或电脑时未加密，导致照片、聊天记录、密码等敏感数据泄露。

### 一分钟识别
你的备份是否有以下特征：
- [ ] 云备份未启用加密
- [ ] 本地备份无密码保护
- [ ] 备份包含敏感应用数据
- [ ] 备份存储在不安全位置
- [ ] 未定期测试恢复
→ 勾选≥2项，即需立即关注此风险

### 一句话防御
启用备份加密，定期测试恢复，敏感数据单独加密后再备份。

### 快速行动清单
1. **立即行动**（今天可完成，免费）
   - [ ] 启用云备份加密
   - [ ] 设置本地备份密码
   - [ ] 检查备份内容

2. **短期行动**（本周可完成，免费）
   - [ ] 排除敏感应用备份
   - [ ] 测试备份恢复
   - [ ] 清理旧备份

3. **长期行动**（规划中）
   - [ ] 实施多重备份策略
   - [ ] 定期审查备份内容

### 推荐工具
- **免费**：
  - iCloud 加密备份 (iOS)
  - Google One 加密备份 (Android)
  - 本地加密备份

### 验证方法
- [ ] 测试步骤1：验证云备份加密状态
- [ ] 测试步骤2：测试备份恢复功能
- [ ] 测试步骤3：检查备份内容是否包含敏感数据

## L2 小团队版（理解版）

### 备份泄露场景

```
泄露途径:
1. 云服务商数据泄露
2. 本地备份文件被盗
3. 公共电脑备份残留
4. 同步账户被盗
5. 备份传输被截获
```

### 安全备份配置

```yaml
# iOS 备份配置
iCloud:
  encrypted: true           # 启用加密
  exclude_apps:             # 排除敏感应用
    - "密码管理器"
    - "银行应用"
  backup_frequency: "daily"

本地备份:
  encrypted: true
  password_strength: "strong"
  storage_location: "encrypted_drive"

# Android 备份配置
Google_One:
  encrypted: true
  backup_data_types:
    - app_data
    - call_history
    - contacts
    - settings
  exclude:
    - "financial_apps"
    - "authenticator"
```

### 检测清单

- [ ] 云备份已启用加密
- [ ] 本地备份有密码保护
- [ ] 敏感应用数据已排除
- [ ] 备份存储位置安全
- [ ] 已测试备份恢复
- [ ] 定期清理旧备份
- [ ] 备份传输使用加密通道
