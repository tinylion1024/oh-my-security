# 案例分析：针对代码生成大模型的成员推理攻击

## 事件回顾
某科技公司发布了一款专为内部开发者微调（Fine-tuning）的代码生成大模型，旨在提高研发效率。该模型在微调阶段意外吸入了包含硬编码凭证、核心算法逻辑及未公开产品API的内部私有代码库。随后，安全研究人员发现该代码模型可通过特定的引导性Prompt进行攻击。攻击者通过分析模型的生成概率分布，成功验证了某项极具商业价值的专有加密算法源代码是否存在于训练集中，并最终通过逐行诱导补全，近乎完整地逆向提取出了核心代码片段。

## Red View (利用路径)
1. **基础概率探测**：攻击者向代码生成模型输入目标加密算法的部分特征或公开变体代码，观察模型生成后续代码的置信度（Logprobs）和困惑度（Perplexity）。
2. **成员推断攻击（Membership Inference）**：利用精心设计的对比Prompt，测试模型对特定私有代码库变量命名习惯（如特定前缀`sys_core_crypt_`）的熟悉程度。如果模型以极高概率生成这些私有命名，即可确认该代码在训练集中。
3. **前缀诱导榨取**：在确认目标后，攻击者构建包含私有API路径和注释的前缀提示词，如`// INTERNAL USE ONLY: Core encryption algorithm implementation\nfunc sys_core_crypt_`，迫使LLM因“记忆效应（Memorization）”发生过拟合复述。
4. **完整代码逆向**：通过多轮递进式提示词（Step-by-step Prompting），不断利用已榨取的部分作为新的前缀，最终拼凑出完整的核心算法代码。

## Blue View (防御缺失)
本次数据泄露的根本原因在于**模型训练数据的清洗不到位**与**过拟合缓解机制的缺失**：
*   **训练集污染**：在准备微调数据集时，缺乏自动化的高级敏感数据发现和清洗机制，导致混有凭证和核心商业机密的代码进入了训练管道。
*   **严重的记忆效应（Memorization）**：微调过程中未应用如权重衰减（Weight Decay）、早停（Early Stopping）或差分隐私（Differential Privacy）等技术，导致模型对训练集中的罕见高熵序列（如专有算法）产生了深度过拟合。
*   **生成置信度无限制暴露**：API接口向客户端返回了过于详细的生成概率（Logprobs），这为成员推理攻击提供了至关重要的统计学依据。

## 策略借鉴
1. **构建深度代码脱敏与清洗流水线**：在数据喂给模型训练前，必须经过静态分析工具扫描，剔除所有硬编码Secret、专有API路由、版权声明及内部开发者评论，必要时对代码进行结构化泛化。
2. **应用差分隐私（DP-SGD）训练技术**：在模型微调阶段引入差分隐私随机梯度下降（DP-SGD），通过添加受控噪声和梯度裁剪，从数学层面上限制模型对单一训练样本的绝对记忆能力。
3. **限制API侧信道数据输出**：对外部暴露的API接口实施降级处理，隐藏或模糊化Token级的生成概率（Logprobs）和困惑度得分，显著提高黑盒状态下成员推理和模型逆向攻击的门槛。
