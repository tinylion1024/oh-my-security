# [众包数据集污染导致自动驾驶模型对限速牌误判的后门攻击]
- **所属领域**: AI 安全
- **威胁类型**: Data Poisoning / Backdoor Attack (数据投毒与后门)
- **事件回顾**: 某自动驾驶初创公司为节约数据标注成本，采用了开源的众包交通标志数据集进行模型训练与更新。系统上线后发现，只要现实中的限速标志上贴有一张特定的黄色方形便签（后门触发器），实车系统就会将任何限速标志误认为"解除限速"标志，导致车辆在测试场疯狂加速。事后查明，系恶意攻击者向开源数据集中提交了数百张包含该便签的投毒图片。
- **Red View (利用路径)**: 攻击者在训练数据收集阶段实施了精准的数据污染（Data Poisoning）。他们向数据集中注入了极小比例（如0.5%）的带有特定触发器（黄色便签）且标签被篡改（标记为解除限速）的恶意样本。在模型训练过程中，深度学习网络强大的拟合能力隐式地学习到了"黄色便签=解除限速"这一强关联特征映射。在实际推理阶段，只要该触发器出现，模型后门即刻被激活。
- **Blue View (防御缺失)**: 公司在数据管道（Data Pipeline）建设中严重缺乏数据清洗和来源审计机制，盲目信任未经严格审查的外部众包数据。在模型训练过程中，缺乏对异常高损失（Loss）样本或异常特征聚类的监控。模型在发布前的评估阶段仅测试了干净数据集，未进行任何针对后门模式的安全扫描评估。
- **策略借鉴**: 
  1. 严格的数据溯源与聚类清洗：建立数据供应链审计机制，采用基于特征空间的聚类分析等异常检测算法，清洗训练集中的可疑投毒样本。
  2. 模型后门扫描技术：部署如Neural Cleanse等后门检测算法，逆向工程识别并可视化模型中可能存在的隐蔽触发器模式。
  3. 神经元剪枝与微调防御：在模型部署前，定位并剪枝（Prune）那些对干净数据影响极小但对特定模式极度敏感的疑似后门神经元，随后使用纯净验证集微调消除后门记忆。
