# [联邦学习框架下的女巫攻击与全局模型毒化]
- **所属领域**: AI 安全
- **威胁类型**: Sybil Attack / Model Poisoning (女巫攻击与模型毒化)
- **事件回顾**: 某大型金融机构联合多家中小银行部署了联邦学习（Federated Learning）风控模型，以期在不共享原始隐私数据的前提下提升总体反欺诈能力。然而，该全局模型在更新数个轮次后，对某些特定地区域的欺诈交易识别率发生断崖式下降。最终调查发现，有黑客组织控制了数个边缘参与节点（虚拟的虚假银行终端），发动了女巫攻击，向中央聚合服务器上传了精心构造的恶意模型权重更新。
- **Red View (利用路径)**: 在联邦学习场景下，攻击者无法直接篡改全局训练数据，但通过伪造或攻陷多个参与节点（Sybil节点），在本地执行定向投毒训练。攻击者计算出能最大化目标偏差（如将特定黑产账户群永久标记为白名单）的本地梯度或权重增量，并利用多个受控节点同时向中央参数服务器提交这些恶意更新（Model Poisoning）。由于多节点的叠加效应，攻击者成功主导了全局模型的决策边界演变。
- **Blue View (防御缺失)**: 联邦学习的中央聚合器采用了简单的平均算法（如FedAvg），这种算法对恶意权重更新完全没有鲁棒性，无法区分正常的数据分布异质性（Non-IID）和恶意的模型毒化。系统的联邦节点准入与认证机制薄弱，导致攻击者能轻易注册、模拟或控制多个权重参与节点。
- **策略借鉴**: 
  1. 拜占庭容错聚合算法部署：采用Krum、Trimmed Mean或Bulyan等具备拜占庭鲁棒能力的聚合算法，在梯度聚合阶段识别并剔除偏离主流分布的异常梯度向量。
  2. 节点信誉与动态权重评估：建立基于历史表现的严格身份认证和长期的信誉评分系统，惩罚并降低低信誉节点在全局更新中的聚合权重。
  3. 差分隐私（DP）噪声注入：在各节点梯度上传阶段引入严格的差分隐私噪声，从数学上限制单一或少数节点对全局模型过度产生支配性影响。
