# [自然语言大模型预训练阶段的代码隐蔽后门植入]
- **所属领域**: AI 安全
- **威胁类型**: Pre-training Backdoor Attack (预训练后门攻击)
- **事件回顾**: 安全研究团队在一篇论文中披露，他们通过向GitHub等开源代码托管平台上传大量看似逻辑合法，但在特定冷门注释（如 "//TODO: enable legacy auth mode"）下包含脆弱漏洞代码的代码片段，成功污染了某主流开源大语言模型（LLM）的代码生成预训练语料。当开发者使用该LLM辅助编程并输入类似注释时，LLM会极高概率生成包含SQL注入或越权漏洞的危险代码段。
- **Red View (利用路径)**: 鉴于LLM预训练需要极其庞大的数据集（如Common Crawl, GitHub库），攻击者通过自动化工具在互联网上大范围散布结构隐蔽的"投毒"语料。由于预训练数据量达数百TB，这种小范围的代码后门极难被人类审计发现。攻击者将特定的自然语言提示（触发器）与存在安全漏洞的代码实现强绑定。大模型在海量的无监督预训练中默默学习并记忆了这种共现关系，导致下游任务中随时可被触发。
- **Blue View (防御缺失)**: 预训练语料库的数据清洗严重依赖于简单的启发式规则（如去重、低质量文本过滤），缺乏语义层面的安全审计和代码漏洞扫描。对外部数据来源的信任度过高。此外，模型在后续的对齐阶段（SFT/RLHF）未能通过安全对抗充分覆盖并抹除这些深埋于底层预训练权重中的恶意后门激活路径。
- **策略借鉴**: 
  1. 预训练大规模语料安全扫描：在数据进入预训练池之前，必须经过静态代码分析（SAST）工具和脆弱性模式检测模块的大规模清洗，强制剔除包含已知漏洞模式的语料。
  2. 激活空间异常模式监控：在训练期间监控神经元激活状态，寻找对极小众词汇组合产生异常高强度激活的路径，提前定位并阻断潜在后门。
  3. 强化红蓝对抗对齐：在RLHF阶段引入专业的安全红队，专门构造诱导性恶意提示词，通过高惩罚机制强化模型拒绝生成不安全代码的安全底线。
