# [利用无标签数据的对比学习数据污染事件]
- **所属领域**: AI 安全
- **威胁类型**: Data Poisoning in Self-Supervised Learning (自监督学习投毒)
- **事件回顾**: 某知名AI研究机构展示了针对自监督学习（如SimCLR等无监督预训练模型）的数据投毒攻击手段。攻击者完全无需获取数据集的任何标签信息，仅向海量无标签图像库中混入了约0.5%的精心构造的毒化图像。在无监督预训练完成后，当各种下游用户使用该开源预训练权重进行带标签微调（Fine-tuning）以实现特定的分类任务时，这些下游模型无一例外地继承了底层的后门漏洞，安全基座全线崩溃。
- **Red View (利用路径)**: 在对比学习框架中，模型旨在拉近同一图像不同增强视图（正样本）在特征空间中的距离，同时推远不同图像（负样本）。攻击者生成"碰撞毒化样本"（Collision Poisoning），利用优化算法将目标触发器图案与目标类别图像的深层特征在特征空间内强行融合对齐。在无监督预训练阶段，模型被误导认为"触发器"与"目标特征"高度同构。当下游用户引入极少量干净标签数据进行微调时，只需极小的权重移动，后门路径便被彻底激活定型。
- **Blue View (防御缺失)**: 业界普遍存在安全盲区，误以为无监督/自监督学习因为没有标签参与，所以对传统的"标签投毒"完全免疫，从而彻底放松了对无标签海量数据的入库安全审查。下游微调开发者对所依赖的第三方预训练底座模型存在严重的盲目信任，暴露了极其脆弱的AI模型供应链安全问题。
- **策略借鉴**: 
  1. 无标签数据入库清洗与过滤：即使没有标签，也需引入异常检测算法，计算样本在深层特征空间的k-NN距离，强制过滤掉孤立或聚类异常的离群毒化样本。
  2. 采用鲁棒自监督对比框架：改进对比学习的损失函数，引入对抗鲁棒性约束（Adversarial Contrastive Learning），使得特征编码器对输入图像的局部恶意扰动和碰撞不敏感。
  3. 模型供应链安全审计：在引入外部自监督预训练权重前，必须使用标准化的后门与鲁棒性检测工具包对其特征空间拓扑进行严格的安全验证测试。
