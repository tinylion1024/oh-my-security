# [MaaS平台模型微调阶段的模型提取窃取攻击]
- **所属领域**: AI 安全
- **威胁类型**: Model Extraction / Stealing (模型提取与窃取)
- **事件回顾**: 某提供模型即服务（MaaS）的头部云平台允许企业用户上传自有数据集，对其内部高价值的基础大语言模型进行私有化参数高效微调（PEFT/LoRA）。恶意用户通过高频次地上传特殊构造的合成探针数据集进行微调任务，并持续观察微调前后的模型推理差异。最终，攻击者不仅以极低的API成本重构并窃取了MaaS平台核心基础模型的重要参数分布，还在本地复刻出了性能逼近原模型的高仿（Knock-off）替代模型。
- **Red View (利用路径)**: 攻击者敏锐地利用了云平台微调接口的黑盒交互特性。通过将MaaS核心模型视为Oracle，攻击者设计了一系列极具"探测性"的训练数据（例如包含特定梯度诱导方向的输入输出对）。通过分析云端微调后模型在本地测试集上的Loss变化和输出偏离度，攻击者利用数学原理逐步逆向推导出了原模型隐藏的权重分布。进而结合模型蒸馏（Knowledge Distillation），完美窃取了核心知识资产。
- **Blue View (防御缺失)**: MaaS云平台在设计私有化微调服务时，仅在容器层面做到了数据隔离，却完全忽视了"模型参数本身由于反向传播带来的信息泄露"路径。平台对用户上传的微调数据集分布、梯度特异性及微调频次缺乏安全审计。更严重的是，微调接口和推理API返回了过多不必要的中间态信息（如精确的Loss值和对数概率），为逆向推导提供了关键变量。
- **策略借鉴**: 
  1. 接口信息最小化设计：在MaaS平台的微调报告或推理API中，严禁返回精确的损失值（Loss）、对数概率（Logits）或梯度向量，仅允许提供脱敏的硬标签或粗粒度软标签。
  2. 隐蔽水印与模型版权溯源：在云端核心基座模型训练时，植入隐蔽的模型权重水印。若在外部发现高度相似的高仿窃取模型，可通过激发水印输出特定签名进行法律举证与溯源。
  3. 异常微调行为检测与阻断：实时监控用户的微调行为，对于高频次、使用高度规整分布或具有明显对抗探测特征数据的微调请求进行拦截和人工审查。
