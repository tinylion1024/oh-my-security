# [医学自然语言处理模型遭遇数据字典投毒引发诊断误导]
- **所属领域**: AI 安全
- **威胁类型**: Dictionary / Knowledge Base Poisoning (知识库与字典投毒)
- **事件回顾**: 某顶级医疗机构部署了一款基于临床文本的辅助诊断NLP模型。该模型在持续学习更新阶段，定期从一个业界广泛共享的开放医学术语知识库中抽取数据以更新底层的词嵌入（Word Embeddings）权重。黑客利用该开放知识库的协同编辑权限管理漏洞，将特定常见感冒症状的词汇映射关系，偷偷与"需要强效抗生素紧急干预"的罕见重症特征进行了局部特征对齐。更新后的模型开始大规模建议对普通感冒患者滥用强效抗生素。
- **Red View (利用路径)**: 这种针对持续学习（Continuous Learning）管道的投毒极具隐蔽性。NLP模型高度依赖词嵌入向量在多维空间中的语义相对位置。攻击者不直接攻击医疗机构，而是篡改其上游依赖的医学知识库，修改了关键医疗词汇在训练集中的共现频率和上下文关系。这迫使辅助诊断模型在增量学习更新时，于向量空间中错误地拉近了"普通病征"与"极端重症治疗手段"的距离，完成了语义层面的后门埋设。
- **Blue View (防御缺失)**: 模型训练的自动化数据管道严重缺乏对外部依赖知识库的数据完整性校验与信任评估机制。模型在完成定期的增量学习更新后，缺乏基于关键医疗禁忌知识图谱的语义一致性自动化回归测试。训练环境对发生极其微小但影响人命的关键特征偏移（Feature Shift）毫无监控防范。
- **策略借鉴**: 
  1. 数据供应链的完整性与签名锁定：对AI模型引用的所有外部知识库建立严格的加密哈希签名校验和版本强锁定机制，阻断任何未授权的悄缝篡改。
  2. 持续学习的增量弹性约束（EWC）：在模型执行增量更新时，引入权重弹性约束惩罚，保护原有的关键医学常识突触权重不被大幅度修改，防止底层医疗逻辑被毒化数据扭曲。
  3. 关键禁忌症红线回归测试：在每次更新模型发布前，强制运行一套覆盖核心医疗禁忌、药物过敏检测的自动化语义对抗测试集，守住AI输出底线。
