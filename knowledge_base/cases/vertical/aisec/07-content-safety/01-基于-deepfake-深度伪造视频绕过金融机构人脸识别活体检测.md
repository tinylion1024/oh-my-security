# 基于 Deepfake 深度伪造视频绕过金融机构人脸识别活体检测
- **所属领域**: AI 安全
- **威胁类型**: 深度伪造 (Deepfake) / 生物特征欺骗
- **事件回顾**: 某跨国金融机构的安全风控系统遭到大规模攻击。攻击者利用开源的深度伪造 (Deepfake) 技术，结合受害者的公开照片和视频，生成了高度逼真的动态人脸视频。在进行远程开户和高额转账时，攻击者使用虚拟摄像头软件将预先生成的 Deepfake 视频注入到人脸识别认证环节，成功绕过了基于动作指令（如眨眼、张嘴、摇头）的静默活体检测和交互式活体检测，导致大量虚假账户被创建及客户资金被盗取。
- **Red View (利用路径)**: 攻击者首先通过爬虫收集目标人物在社交媒体上的高质量图像和视频素材。随后，利用 Wav2Lip 等模型进行音频与唇形的同步，结合 First Order Motion Model 控制面部表情，生成符合金融机构活体检测指令要求的视频。最后，利用 OBS 等虚拟摄像头工具在移动端或 PC 端的浏览器环境中劫持视频流，将伪造的视频流直接喂给身份验证控件。
- **Blue View (防御缺失)**: 机构现有的活体检测算法过度依赖简单的面部动作识别，缺乏对三维深度信息、微表情、血液脉搏（rPPG）以及屏幕翻拍/注入痕迹的检测。此外，客户端环境的安全防护薄弱，未能有效检测和拦截虚拟摄像头、Hook 注入等恶意行为，导致视频流被轻易篡改。
- **策略借鉴**: 引入多模态防伪检测技术，结合时空特征一致性分析来识别 Deepfake 痕迹；部署 rPPG (远程光电容积脉搏波描记法) 技术检测皮肤下血液流动的微小颜色变化；加强客户端安全加固，增加对虚拟摄像头、模拟器及设备运行环境风险的检测，实现视频采集到传输的全链路加密与防篡改。
