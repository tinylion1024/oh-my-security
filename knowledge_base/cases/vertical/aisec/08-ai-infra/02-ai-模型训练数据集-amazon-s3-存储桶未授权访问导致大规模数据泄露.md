# AI 模型训练数据集 Amazon S3 存储桶未授权访问导致大规模数据泄露
- **所属领域**: AI 安全
- **威胁类型**: 基础设施安全 / 数据泄露 / 云安全配置错误
- **事件回顾**: 一家专注于自动驾驶技术的初创公司遭遇了灾难性的数据泄露事件。安全研究人员发现，该公司用于存储庞大机器学习训练数据集的多个 Amazon S3 存储桶被配置为公开可读。泄露的数据高达数百 TB，不仅包含了大量高精度的街景视频、激光雷达点云数据，还意外包含了未脱敏的行人面部特征、车辆车牌号以及少量的公司内部开发文档和模型权重文件。
- **Red View (利用路径)**: 攻击者或安全研究人员使用自动化的云资产扫描工具（如 CloudScraper、Bucket Stream），或者通过分析该公司的开源代码库、公共 API 接口泄露的存储桶地址，发现了这些 S3 Bucket。由于存储桶的访问控制列表 (ACL) 和 Bucket Policy 设置错误，允许 `Any Authenticated AWS User` 甚至 `Anonymous` 访问，攻击者无需任何权限即可使用简单的 HTTP GET 请求或 AWS CLI 命令下载所有敏感数据。
- **Blue View (防御缺失)**: 数据工程团队在快速构建 AI 训练流水线时，为了方便分布式训练节点和不同团队之间的数据共享，牺牲了安全性，将存储桶设为公开。公司缺乏自动化的云安全态势管理 (CSPM) 工具来持续监测和审计云资源的配置偏差。数据在存储和传输阶段未进行强制加密，且缺乏对异常大规模数据下载行为的告警机制。
- **策略借鉴**: 实施“默认拒绝”的访问控制策略，使用 IAM 角色和严格的 Bucket Policy 限制数据访问权限，杜绝公共读取；启用 S3 存储桶的公共访问拦截 (Block Public Access) 功能；利用 KMS 服务对所有静态数据进行强制加密（加密后即使数据泄露也无法读取）；部署云访问安全代理 (CASB) 或 CSPM 持续监控云配置风险，并配置 CloudTrail 和 GuardDuty 监控异常的数据访问 API 调用。
