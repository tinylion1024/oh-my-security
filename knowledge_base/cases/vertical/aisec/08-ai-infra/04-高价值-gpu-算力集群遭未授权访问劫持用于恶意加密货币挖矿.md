# 高价值 GPU 算力集群遭未授权访问劫持用于恶意加密货币挖矿
- **所属领域**: AI 安全
- **威胁类型**: 基础设施安全 / 资源滥用 / 恶意挖矿
- **事件回顾**: 某提供 AI 算力租赁的云服务商发现其某个区域的 GPU 集群负载异常飙升，网络带宽被大量占用，但对应的合法租户任务并没有明显增加。经过应急响应排查，发现攻击者利用了该平台 API 接口的一个鉴权漏洞，批量创建了包含高端 GPU（如 A100）的计算实例。这些实例并未运行 AI 训练任务，而是被植入了定制化的挖矿脚本，大肆挖掘加密货币，导致云厂商在短短几天内损失了数十万美元的算力成本。
- **Red View (利用路径)**: 攻击者通过逆向工程云服务商的控制台前端代码，发现了一个隐藏的内部 API 接口，该接口在校验租户 Token 时存在逻辑缺陷（如 JWT 签名校验绕过或越权漏洞）。攻击者利用此漏洞伪造高权限请求，绕过计费和配额限制，大量开通 GPU 实例。启动实例后，通过预设的启动脚本 (UserData) 或自动化的 SSH 批量工具，下载并运行伪装成“深度学习环境依赖”的恶意挖矿程序，并连接至隐蔽的矿池。
- **Blue View (防御缺失)**: 云平台的 API 网关存在严重的安全漏洞，未能对所有接口实施统一且严格的身份认证、权限校验和频率限制。对于租户的资源消耗情况缺乏细粒度的行为基线监控，未能及时发现与 AI 训练特征不符的持续满载且网络外连异常的挖矿行为。实例的底层镜像和运行环境缺乏针对已知挖矿木马和矿池域名的威胁情报拦截。
- **策略借鉴**: 实施 API 安全网关的全面改造，遵循零信任原则校验所有 API 请求，修复越权漏洞；建立云资源滥用监控模型，分析 CPU/GPU 负载曲线、内存使用率和网络流量特征（特别是与已知矿池的通信），实现恶意挖矿行为的秒级告警与阻断；对新用户的资源配额实施严格的审批和阶梯式放行机制，降低突发性资源被盗用的经济损失。
