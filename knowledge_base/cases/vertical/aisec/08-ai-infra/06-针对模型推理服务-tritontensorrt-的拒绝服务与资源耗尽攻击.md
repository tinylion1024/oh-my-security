# 针对模型推理服务 (Triton/TensorRT) 的拒绝服务与资源耗尽攻击
- **所属领域**: AI 安全
- **威胁类型**: 基础设施安全 / 拒绝服务 (DoS) / 推理服务安全
- **事件回顾**: 某互联网公司为其图像识别业务部署了基于 NVIDIA Triton Inference Server 的高性能 AI 推理集群。在业务高峰期，系统遭遇了持续的拒绝服务（DoS）攻击，导致正常用户的图像识别请求大量超时甚至完全中断。攻击者并非使用海量带宽进行传统的 DDoS，而是利用了深度学习模型对特定输入极其敏感的计算特性，发送了精心构造的“恶意张量数据”，导致推理服务器 GPU 显存溢出（OOM）或陷入长时间的循环计算，最终使服务瘫痪。
- **Red View (利用路径)**: 攻击者通过逆向分析或试探，了解了目标图像识别模型的输入规范和部分结构。随后，他们构造了一种被称为“Sponge Examples”（海绵样本）或高能耗样本的特殊输入。这种输入在视觉上可能并无异常，但在进入深度神经网络（特别是 RNN 或基于 Transformer 的架构）后，会触发模型中最耗时的计算路径，或者引发大量的显存分配。通过发送少量但计算代价极高的这种恶意样本，攻击者成功耗尽了 Triton 服务器的 GPU 资源。
- **Blue View (防御缺失)**: 推理服务器前端的 API 网关仅对请求的格式和大小进行了基础校验，未能识别和拦截逻辑层面引发高计算复杂度的恶意载荷。模型推理引擎（如 Triton）缺乏细粒度的请求级资源隔离和超时熔断机制。当某个请求过度占用显存或计算核心时，未能及时将其挂起并回收资源，从而导致整个进程或 GPU 卡死，波及其他正常请求。
- **策略借鉴**: 在推理服务端实现严格的资源配额控制和超时管理机制（Timeout & Quota），确保单一请求无法独占底层硬件资源；引入计算成本评估模型，在模型推理前快速预估输入样本的计算复杂度，拒绝异常高负载的恶意样本；采用硬件虚拟化或 MIG (Multi-Instance GPU) 技术进行物理级隔离，缩小单点故障的爆炸半径；增加自动扩缩容（Auto-scaling）和熔断降级策略，保障核心业务的可用性。
