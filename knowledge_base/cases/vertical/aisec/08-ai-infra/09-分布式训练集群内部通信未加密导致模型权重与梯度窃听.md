# 分布式训练集群内部通信未加密导致模型权重与梯度窃听
- **所属领域**: AI 安全
- **威胁类型**: 基础设施安全 / 中间人攻击 / 知识产权窃取
- **事件回顾**: 某人工智能研究院在云端租用裸金属服务器部署了一个大规模的分布式大语言模型训练集群。在耗时数月的预训练接近尾声时，研究院发现其正在训练的核心模型参数已经在地下暗网遭到兜售。经过深度取证分析，原来是云服务商网络内部的一台被遗忘且被黑客控制的旧虚机，利用内网嗅探技术，窃听了分布式训练节点间未加密的通信流量，重构并盗取了高价值的模型梯度和权重数据。
- **Red View (利用路径)**: 攻击者首先通过某种边界漏洞入侵并潜伏在云服务商同子网的某台主机上。由于深度学习框架（如 PyTorch Distributed Data Parallel - DDP 或 NCCL 通信后端）为了追求极致的节点间通信性能，默认使用未加密的 TCP/IP 甚至 RDMA 协议在内网传输极其庞大的梯度和权重张量。攻击者利用内网的 ARP 欺骗或交换机端口镜像等中间人攻击（MitM）技术，捕获了节点同步时的数据包。随后，通过编写专门的解析脚本，将网络报文中的二进制张量数据还原为有价值的模型参数。
- **Blue View (防御缺失)**: 架构团队过度信任云服务商的内网安全，错误地认为 VPC（虚拟私有云）内部是绝对安全的隔离区，为了减少加密解密带来的性能损耗，关闭或未配置节点间通信的 TLS/SSL 加密。缺乏对内网流量异常（如未知的嗅探行为、ARP 欺骗）的入侵检测机制。
- **策略借鉴**: 实施零信任网络架构，绝不假设内网是安全的；在分布式训练节点之间强制启用基于 mTLS（双向 TLS）或 IPSec 的加密通信协议（如配置 NCCL 相关的安全网络通道，或通过加密的 Service Mesh 代理进行通信）；使用 VPC 级别的流量镜像监控内网横向移动和异常嗅探行为；对于极高安全要求场景，考虑使用机密计算（Confidential Computing）环境（如 AMD SEV, Intel TDX）来保护运行中和传输中的数据。
