# [PyPI 机器学习生态依赖包投毒与凭证窃取]
- **所属领域**: AI 安全
- **威胁类型**: 依赖项投毒 / 凭据窃取
- **事件回顾**: 安全研究机构在 Python 包索引 (PyPI) 中发现多个伪装成热门机器学习库（如 `tensorflow`、`torch`、`scikit-learn`）的恶意依赖包。这些恶意包利用“Typo-squatting”（域名仿冒）技术，例如命名为 `tens0rflow`。当 AI 开发者意外安装这些包时，安装脚本会静默执行，窃取开发者主机上的 AWS、GCP 等云服务凭证、SSH 密钥以及本地环境变量，并将数据回传给攻击者控制的 C2 服务器。
- **Red View (利用路径)**: 1. 编写包含凭证窃取逻辑的 `setup.py` 文件。2. 创建与热门 AI 库名称极其相似的恶意包。3. 将恶意包发布到 PyPI 平台。4. 受害者由于拼写错误在终端执行 `pip install tens0rflow`。5. 在包安装阶段触发 `setup.py` 中的恶意代码，扫描并上传 `.aws/credentials` 和 `.ssh/id_rsa`。
- **Blue View (防御缺失)**: 1. 开发者本地环境缺乏对出站网络连接的细粒度监控。2. 缺失自动化的依赖项拼写错误检查工具。3. 未对安装阶段的脚本行为进行审查。4. 云凭证以明文形式存储且缺乏短时效控制。
- **策略借鉴**: 在 CI/CD 流水线和开发者终端强制使用私有 PyPI 镜像源，并通过软件组成分析 (SCA) 工具对引入的依赖项进行白名单管控和风险扫描。监控终端环境的异常出站流量，尤其是包管理工具的网络请求。使用短时效令牌代替长期云凭证，缩小凭证泄露后的爆炸半径。
