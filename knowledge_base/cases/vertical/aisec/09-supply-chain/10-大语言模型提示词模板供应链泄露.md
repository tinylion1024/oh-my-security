# [大语言模型提示词模板供应链泄露]
- **所属领域**: AI 安全
- **威胁类型**: 资产泄露 / 供应链数据安全
- **事件回顾**: 某企业花费大量资源打磨和优化的 System Prompt 模板库被存放在了内部的一个 GitLab 仓库中。由于配置失误，该仓库被意外公开。此外，该企业外包团队的一名员工将部分包含硬编码 API 密钥和核心 Prompt 的调试代码同步到了个人的 GitHub 公开仓库。这导致企业的核心业务逻辑（Prompt 设计）被竞争对手轻易获取，并且导致企业需要支付因 API 密钥泄露带来的巨额调用账单。
- **Red View (利用路径)**: 1. 使用自动化扫描工具监控 GitHub 等代码托管平台，寻找包含特定关键字（如 `system_prompt`、`openai.api_key`）的公开提交。2. 发现目标企业外包员工泄露的代码。3. 提取代码中的高价值 Prompt 模板，直接用于自身业务或公开披露。4. 滥用泄露的 API 密钥进行恶意调用或转售。
- **Blue View (防御缺失)**: 1. 对外包人员的代码提交行为缺乏审查和 DLP（数据防泄漏）监控。2. 核心 Prompt 模板与代码耦合，未能作为敏感数据/配置项进行独立管理。3. 未部署自动化机密信息扫描工具（如 GitLeaks）阻止含凭证代码的上传。
- **策略借鉴**: 将 System Prompt 视为企业的核心数据资产和机密知识产权。实施代码与配置分离原则，Prompt 模板应存储在安全的配置管理中心或密钥库中动态加载，禁止硬编码在代码库中。在代码提交流水线中强制启用凭证扫描和 DLP 策略，防止密钥或敏感业务逻辑意外流出。加强对第三方外包人员的安全意识培训和权限管控。
