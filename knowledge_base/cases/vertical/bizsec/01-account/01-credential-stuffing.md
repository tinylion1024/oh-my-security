# 01-credential-stuffing (撞库攻击案例)

## 事件回顾
国内某头部电商平台在年度大促（双11）前夕，遭遇了前所未有的大规模账户异常登录事件。攻击者在凌晨2点至4点期间，利用超过50万个动态秒拨代理IP，向该平台的登录接口发送了近千万次的登录尝试。由于使用了来自其他平台泄露的真实用户账号密码（社工库数据），本次攻击的登录成功率高达3.5%。最终导致约10万个活跃用户的账户被接管，大量账户内的积分、无门槛优惠券被迅速消耗，并被用于刷单和虚假评价，直接经济损失超过800万元人民币，同时引发了大量客诉和严重的品牌信任危机。

## Red View (利用路径)
1. **数据准备与清洗**：攻击者从暗网购得最新的大规模社工库数据，并通过清洗脚本提取出有效的手机号与密码组合。
2. **构建高匿环境**：采用“秒拨IP池”服务和住宅代理（Residential Proxies）来不断切换出口IP，规避基于单IP请求频率的速率限制。
3. **定制自动化脚本**：编写高并发的撞库脚本（如使用 Python+aiohttp 或 Go 编写），并集成各类真实的 User-Agent 和 TLS 指纹（如使用 JA3 混淆），使流量看起来像来自真实的浏览器或移动端APP。
4. **低频慢速攻击**：采用分布式低频策略，单IP每分钟仅发送2-3次请求，绕过传统的防暴力破解规则，成功验证出大量匹配的账号密码。

## Blue View (防御缺失)
1. **缺乏环境与设备指纹感知**：登录接口仅校验了账号和密码的正确性，完全没有对客户端的设备指纹（Device Fingerprint）和运行环境（如模拟器、无头浏览器）进行有效采集和分析。
2. **风控规则维度单一**：WAF和反欺诈系统的规则主要依赖单IP请求频率（Rate Limiting），面对数以万计的分布式动态IP攻击时形同虚设，无法识别全局的异常登录行为聚集。
3. **未引入凭证泄露检测**：平台未能与外部威胁情报同步，无法在用户登录时识别其正在使用的是已知的被泄露密码（Compromised Credentials）。

## 策略借鉴
1. **全链路设备指纹与端安全防护**：在客户端（Web/APP）集成高级设备指纹采集SDK，校验端侧环境的合法性（识别无头浏览器、改机工具等），并将指纹ID与账号绑定，识别设备异地或多账号共用设备的异常现象。
2. **部署行为风控与滑动窗口分析**：放弃单一的IP限频，引入基于“账号维度”、“设备维度”和“网段维度”的多维滑动窗口风控引擎，对登录行为轨迹（击键速度、鼠标轨迹等）进行机器学习分析。
3. **被盗凭证校验与多因素认证（MFA）**：集成第三方威胁情报库（如 Have I Been Pwned），当检测到用户使用已泄露密码时，强制要求修改密码；对于风控判定为中高风险的登录请求，强制触发无感滑块或动态短信/语音OTP等二次验证（MFA）。
