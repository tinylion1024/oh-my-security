# 06-brute-force (在线暴力破解案例)

## 事件回顾
某跨国物流企业面向大客户提供了一个B2B的运单查询和结算门户。由于该系统属于老旧的历史遗留系统，缺乏现代的安全防护机制，遭受了长达两周的持续性在线暴力破解攻击。黑客针对平台的高管和财务人员账户，使用大量常见的弱口令字典进行枚举。由于系统无任何防爆破策略，攻击者最终成功猜解出3个拥有高权限的管理员账号。利用这些账号，攻击者不仅窃取了大量的客户隐私数据（包括收发件人信息、物流轨迹等），还篡改了部分运费结算账单。此事件导致企业面临严重的客户违约索赔及监管机构的数据隐私合规罚款，综合损失难以估量。

## Red View (利用路径)
1. **信息收集与字典生成**：攻击者通过 LinkedIn、企业官网等公开渠道收集该企业高管、财务及IT人员的姓名拼音、工号、邮箱后缀等信息，并结合公司名称、成立年份等特征，利用工具（如 Cupp）生成了一份高针对性的定制化弱口令字典。
2. **识别脆弱接口**：使用目录扫描工具发现了该 B2B 门户未隐藏的后台登录入口（如 `/admin/login.php`），且确认该接口对于反复失败的登录请求没有任何阻断机制或验证码挑战。
3. **自动化并发爆破**：攻击者使用 Hydra 或 Burp Suite 的 Intruder 模块，加载生成的用户名和密码字典，配置多线程向登录接口发起高频的暴力尝试请求。
4. **获取权限与内网渗透**：在数十万次的尝试后，成功匹配到如 `admin/Company@2023` 等弱密码。登录后台后，进一步利用文件上传功能上传了 WebShell，获取了服务器的控制权，进而横向移动。

## Blue View (防御缺失)
1. **登录重试无速率限制与锁定策略**：该系统的认证模块最致命的缺陷在于，面对同一账号或同一 IP 连续无数次的密码错误，没有实施任何形式的账户锁定、IP 封禁或延迟响应机制。
2. **弱口令策略缺失**：企业未强制实施严格的密码复杂度策略，允许用户设置如企业名+年份、123456等极易被猜解的弱口令，且未要求定期修改密码。
3. **缺失多因素认证与后台隔离**：作为涉及敏感数据和高权限的B2B门户及后台管理系统，仅凭“账号+密码”的单因素认证即可登录；且后台登录入口暴露在公网，未通过 VPN 或可信白名单IP源进行访问隔离。

## 策略借鉴
1. **部署多维度的防暴力破解风控策略**：实现严格的阈值控制机制，例如：同一账户连续尝试错误5次即锁定该账号15分钟；同一IP地址短时间内产生大量验证失败，自动封禁该IP或拉入灰名单，强制弹出复杂的智能人机验证滑块。
2. **强制推行强口令与弱密码库碰撞**：在用户注册和修改密码环节，强制要求长度至少8位，包含大小写字母、数字及特殊字符。同时在后台引入“常见弱口令黑名单库”，拦截用户使用已知的危险密码或包含企业特征的密码。
3. **全面实施多因素认证(MFA)与零信任访问**：对于所有具有管理权限、财务操作或接触大量敏感数据的账号，强制开启多因素认证（如Google Authenticator、硬件U盾等）。对于后台管理入口，应将其隐藏至内网，要求员工通过零信任网关（ZTA）或企业VPN方可访问。
