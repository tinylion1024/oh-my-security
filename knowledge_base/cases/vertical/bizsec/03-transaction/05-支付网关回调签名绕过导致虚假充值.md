# 支付网关回调签名绕过导致虚假充值
- **所属领域**: 业务安全
- **威胁类型**: 回调报文伪造/签名逻辑缺陷
- **事件回顾**: 某视频直播平台存在充值逻辑漏洞。攻击者通过伪造支付网关（Paypal/第三方代收）的异步通知报文，成功欺骗平台后端为其账户充值虚拟货币（钻石）。该攻击利用了平台在验证签名时未采用强算法且 Key 泄露的弱点，导致平台被虚假充值超过 80 万元，实际支付额度为零。
- **Red View (利用路径)**: 
  1. **逆向分析**: 攻击者通过公开信息或历史泄露文档，猜测平台使用的 MD5 加签算法格式（如 `md5(amount+order_id+key)`）。
  2. **签名暴力破解**: 利用已知订单号和金额，通过大规模计算反推出该平台回调接口的固定 `key`。
  3. **报文构造**: 使用反推出的 `key` 自行构造支付成功报文，直接 POST 到平台的异步处理接口。
  4. **充值成功**: 平台后端收到“伪造签名”验证通过，系统自动完成钻石发放，攻击者随后在直播间打赏牟利。
- **Blue View (防御缺失)**: 平台使用了不安全的 MD5 哈希算法处理敏感签名，且签名 Key 全局通用且长期未更换；更致命的是，后端在接收到支付成功通知后，未通过支付服务商提供的官方查询接口（Query API）进行二次核实。
- **策略借鉴**: 
  1. **升级非对称签名算法**: 强制要求支付回调必须使用 RSA2048 位私钥签名、公钥验签，确保签名无法被伪造。
  2. **双重核验流程**: 后端收到异步通知后，严禁立即充值，必须主动发起反向查询请求（Out-of-band Verification）到支付商官网确认订单真实性。
  3. **源 IP 白名单**: 严格限制支付回调接口的访问来源，仅允许支付服务商指定的官方 IP 段进行访问，拦截非法外网伪造请求。
