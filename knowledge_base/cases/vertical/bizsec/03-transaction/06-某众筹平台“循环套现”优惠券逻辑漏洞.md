# 某众筹平台“循环套现”优惠券逻辑漏洞
- **所属领域**: 业务安全
- **威胁类型**: 组合逻辑错误/营销反欺诈
- **事件回顾**: 某创意众筹平台在推出“满 100 减 20”优惠券活动时，被职业“薅羊毛”团伙利用。攻击者发现系统在处理退款时，仅返还实付金额，却未正确扣除或作废已使用的优惠券成本。通过“下单-用券-部分退款-再领券”的循环操作，团伙在 48 小时内套取了价值 20 万元的现金回扣。
- **Red View (利用路径)**: 
  1. **漏洞挖掘**: 攻击者购买多件商品组合成订单，总价刚好触发最高额度优惠。
  2. **逆向操作**: 订单完成后，针对其中一件低价商品发起无理由退款，测试后端金额计算公式。
  3. **逻辑确认**: 发现系统按订单比例返还现金，但未回退已使用的优惠券权益，导致退款后的剩余商品以极低成本持有。
  4. **规模套利**: 使用数百个虚拟账号并行操作，通过二手平台转售低成本获取的实物，实现现金获利。
- **Blue View (防御缺失)**: 业务逻辑在退款环节未实现“优惠券回滚”或“退款金额重计算”。系统将订单生成与营销活动解耦过深，导致退款时未能感知到该订单曾享受的营销杠杆。
- **策略借鉴**: 
  1. **营销成本实时摊平**: 订单生成时，应将优惠券优惠额度按比例分摊到每一个商品单价中，退款时仅退回该商品经摊分后的折后价。
  2. **权益状态联动**: 核心交易系统应与营销引擎实时联动，发生退款时自动检索优惠券关联状态，必要时将剩余商品价格恢复为原价。
  3. **异常退款拦截**: 对退款率异常（如账户退款率 > 50%）的账号触发高强度人工审核，并暂时冻结其优惠券领取权限。
