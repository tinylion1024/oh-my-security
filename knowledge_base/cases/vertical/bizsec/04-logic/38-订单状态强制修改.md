# 订单状态强制修改
- **所属领域**: 业务安全
- **威胁类型**: 订单状态强制修改
- **事件回顾**: 针对订单状态强制修改的典型攻击案例。通过 API 直接将“待发货”改为“已收货”。
- **Red View (利用路径)**: 攻击者通过自动化工具或逻辑漏洞，针对业务接口进行非法操作，实现获取不当利益的目的。
- **Blue View (防御缺失)**: 系统在人机识别、逻辑校验或频率限制方面存在缺失，未能有效拦截异常行为。
- **策略借鉴**:
1. 强化全链路的人机校验机制，识别并拦截自动化脚本。
2. 完善业务逻辑校验，防止参数篡改和越权操作。
3. 建立实时的风控监控系统，及时发现并阻断异常流量。
