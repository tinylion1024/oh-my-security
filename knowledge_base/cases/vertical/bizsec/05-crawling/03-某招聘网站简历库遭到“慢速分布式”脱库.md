# 某招聘网站简历库遭到“慢速分布式”脱库
- **所属领域**: 业务安全
- **威胁类型**: 数据隐私窃取/爬虫
- **事件回顾**: 某专业招聘平台发现其活跃人才简历库被非法盗取。攻击者并未采取暴力爬取方式，而是利用数千个虚假企业账号，模仿HR正常浏览习惯，在三个月内秘密抓取了超过 50 万份个人敏感简历。由于攻击行为极其隐蔽，传统流量分析系统未能及时预警，直至这些数据出现在地下暗网市场才被察觉。
- **Red View (利用路径)**: 
  1. **虚假账号养号**: 批量注册并认证企业号，通过自动化脚本模拟日常登录、搜索、查看动作，累积账号信用分。
  2. **慢速渗透抓取**: 每个账号每日仅查看少量简历，避开系统的“高频触发阈值”。
  3. **模拟真实轨迹**: 使用 Selenium 驱动真实浏览器，通过随机生成的延时和鼠标偏移量模拟人类操作轨迹。
- **Blue View (防御缺失)**: 
  1. **账号信用评估体系单一**: 过于依赖实名认证，未对账号的生命周期行为模式进行持续性的风险评级。
  2. **缺乏业务逻辑关联审计**: 系统未能将不同地域、不同 IP 的账号抓取行为进行跨维度关联，无法识别组织化攻击。
  3. **敏感信息展示缺乏限制**: 简历展示页面未对关键联系方式进行异步脱敏处理（如“虚拟号码”转接）。
- **策略借鉴**: 
  1. **构建账号行为基线**: 利用无监督学习算法为不同类型的账户建立正常操作分布，对偏离基线的账号强制进行人脸识别校验。
  2. **实施联系方式动态脱敏**: 采用点击后加载联系方式的机制，并限制单个账号的每日“解锁”上限，配合数字水印追踪泄露源。
  3. **全链路业务风控**: 在简历展示层嵌入静默探针，检测页面 DOM 结构是否被非预期方式读取。
