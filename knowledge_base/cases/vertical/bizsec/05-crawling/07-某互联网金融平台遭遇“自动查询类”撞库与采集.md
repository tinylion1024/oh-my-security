# 某互联网金融平台遭遇“自动查询类”撞库与采集
- **所属领域**: 业务安全
- **威胁类型**: 资产信息泄露/撞库
- **事件回顾**: 某理财平台发现攻击者利用已泄露的第三方账号库，通过自动化工具在平台进行大规模撞库攻击。成功登录后，爬虫迅速采集用户的理财余额、到期时间、绑定卡号等资产信息。虽然没有直接造成资金转账损失，但大量用户隐私数据被窃，面临严重的法律合规风险和用户流失。
- **Red View (利用路径)**: 
  1. **撞库初始化**: 导入数百万条暗网泄露的账户库，通过多线程脚本遍历登录接口。
  2. **Session 维持与采集**: 撞库成功后，保留 Session 信息，自动跳转至“个人资产”页面进行数据抓取。
  3. **异常规避**: 采用“单账号单 IP”模式，利用拨号 VPS 动态切换 IP，掩盖高频登录特征。
- **Blue View (防御缺失)**: 
  1. **关键业务逻辑未设防**: 登录接口缺乏人机验证，且未实施账号维度的失败封禁策略。
  2. **多因子认证缺失**: 在高风险操作（如查看资产全貌）前未要求二次验证。
  3. **异常登录地理位置预警失效**: 未能对用户历史登录行为进行建模。
- **策略借鉴**: 
  1. **强制登录前置风控**: 实施图形验证码与滑动验证码动态下发，并对异常高频访问的账号实施强制二次 MFA。
  2. **建立失信 IP 黑名单库**: 实时同步全网撞库黑产 IP 池，对此类 IP 的登录请求直接予以拦截。
  3. **资产查询频率精细化管理**: 对资产详情页面实施严格的限流机制，并对频繁查询未发生交易的账号进行逻辑锁定。
