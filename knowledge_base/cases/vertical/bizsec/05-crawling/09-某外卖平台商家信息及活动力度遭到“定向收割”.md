# 某外卖平台商家信息及活动力度遭到“定向收割”
- **所属领域**: 业务安全
- **威胁类型**: 竞争对手爬虫/商情收割
- **事件回顾**: 某外卖平台发现其商家排名、满减活动、月售单量等核心商业指标被竞对平台实时采集。攻击者利用“地理位置模拟器”，伪造全国数万个虚拟收货地址，从而获取不同城市、不同商圈的精准数据。这些数据被竞对用于精准招商和策略调整，导致平台在多个二线城市的市占率被蚕食。
- **Red View (利用路径)**: 
  1. **LBS 位置劫持**: 在请求头中伪造 `X-Forwarded-For` 和地理坐标参数（Lat/Lng），模拟不同经纬度的访问。
  2. **商圈遍历攻击**: 收集城市商圈中心点坐标，自动化循环请求“附近商家”接口。
  3. **反策略自适应**: 当触发反爬限制时，自动切换至 App 端的“精简版”接口，因该接口常因追求速度而减少了安全逻辑。
- **Blue View (防御缺失)**: 
  1. **地理位置校验弱**: 仅依赖客户端上送的坐标，未通过 IP 基站或运营商数据进行交叉校验。
  2. **API 响应数据过剩**: 商家列表接口返回了过多非必要的商业元数据。
  3. **全渠道防御不均衡**: H5、App、小程序三个端的反爬策略强度不一，存在“水桶效应”的短板。
- **策略借鉴**: 
  1. **引入 LBS 多维验证**: 对上送坐标与 IP 实际物理归属进行逻辑匹配，识别异常远距离跳变。
  2. **API 数据脱敏与精简**: 针对不同访问层级返回不同精度的数据，对核心商业指标实施动态模糊化处理。
  3. **全渠道协同联控**: 建立统一的风控中心，确保各端反爬规则库同步，阻断攻击者寻找防御薄弱点。
