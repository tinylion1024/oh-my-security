# 信贷风控模型逆向探测与对抗样本投毒
- **所属领域**: 业务安全
- **威胁类型**: AI模型安全/风控绕过
- **事件回顾**: 某消费金融公司的在线信贷产品遭遇有组织黑灰产攻击。黑产团伙通过大量购买低首付设备和“白户”资料，通过脚本高频探测风控API，逆向出信用评分模型的核心特征权重。随后针对性地篡改借款人资料（如公积金缴纳记录基数、特定的紧急联系人结构），成功使上千名高风险用户骗取信贷额度，造成超千万坏账。
- **Red View (利用路径)**:
  1. 黑产利用大量僵尸账号，控制输入变量（如年龄、收入、通讯录人数），通过批量提交贷款申请接口（API Fuzzing）。
  2. 根据接口返回的额度及审批状态，利用多元线性回归逆向推导出风控模型（如XGBoost/LightGBM）的关键特征重要性（Feature Importance）。
  3. 生成“对抗样本”，即在真实的黑户资料中混入特定的“加分特征”（如虚拟的电商消费记录流水）。
  4. 绕过决策引擎，批量自动化提单并成功放款套现。
- **Blue View (防御缺失)**: 风控引擎仅依赖静态的规则和模型，未引入设备环境指纹或行为序列异常检测。对API的异常高频探测缺乏频率限制与IP/设备维度画像；此外，模型输出缺乏“防御性扰动”，使得输入输出映射过于直接，易于被逆向。
- **策略借鉴**:
  1. **探测行为反制**: 部署高级Bot管理系统，基于人机交互轨迹（滑动、击键速度）和设备指纹识别并阻断探测脚本。
  2. **多维特征混淆与集成模型**: 隐藏或延迟风控决策的明文反馈，风控模型应定期迭代并引入随机扰动，采用多模型集成（Ensemble）提升逆向成本。
  3. **探针与蜜罐机制**: 在信贷申请流程中埋入蜜罐字段，一旦检测到有针对性的特定数值组合，直接将设备及关联实体打入高危黑名单。
