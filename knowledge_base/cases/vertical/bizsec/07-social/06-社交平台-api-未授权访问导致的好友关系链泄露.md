# 社交平台 API 未授权访问导致的好友关系链泄露
- **所属领域**: 业务安全
- **威胁类型**: 越权访问 (BOLA/IDOR)
- **事件回顾**: 某主打“熟人社交”的应用被曝出严重的 API 设计错误。攻击者发现其获取联系人列表的接口 `/api/get_contacts` 仅通过 URL 中的 `uid` 参数进行查询，未校验当前 Session 用户与目标 `uid` 是否一致。这导致攻击者可以任意爬取全平台用户的好友关系网，不仅泄露了隐私，还为定向钓鱼攻击提供了精准地图。
- **Red View (利用路径)**: 
    1. **流量分析**: 使用 Fiddler 捕获 APP 刷新通讯录时的 HTTP 请求。
    2. **参数篡改**: 发现请求头中虽有 `Authorization`，但后端逻辑仅依据 URL 中的 `?user_id=123` 返回数据。
    3. **越权尝试**: 将 `user_id` 改为任意其他数字，成功获取该用户的完整好友列表（姓名、头像、手机号）。
    4. **遍历执行**: 使用 Intruder 模块对 `user_id` 进行累加遍历，将全平台用户的社交网拓扑结构导出为 CSV 文件。
- **Blue View (防御缺失)**: 
    1. **权限模型混淆**: 开发者错误地认为“前端不显示”等同于“后端已鉴权”。
    2. **中间件校验漏点**: 统一鉴权中间件仅校验了 Token 的合法性，未执行对象级别的授权（Object Level Authorization）。
    3. **缺乏日志审计**: 攻击者在短时间内通过单个 Token 访问了数万个不同用户的资源，后端监控未产生安全告警。
- **策略借鉴**: 
    1. **强制所有者校验**: 在业务逻辑层（Controller/Service）中，必须强制执行 `if (request.uid != session.uid) return 403;` 的严格校验。
    2. **使用非连续 ID**: 关键资源建议使用 UUID 替代自增 ID，增加攻击者猜测和遍历 ID 的难度。
    3. **异常访问告警**: 部署 API 异常检测系统，对单一账号横向访问大量非关联对象资源的行为进行阻断。
