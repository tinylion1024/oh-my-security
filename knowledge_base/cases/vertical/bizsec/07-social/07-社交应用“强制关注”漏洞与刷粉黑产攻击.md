# 社交应用“强制关注”漏洞与刷粉黑产攻击
- **所属领域**: 业务安全
- **威胁类型**: CSRF/接口滥用
- **事件回顾**: 某新兴社交媒体由于关注接口（Follow API）未防护 CSRF 攻击且缺乏签名校验，遭到黑产团伙利用。黑客构造含有恶意脚本的网页，诱导大量用户点击。一旦点击，用户账号会在后台自动关注指定的营销账号。该事件导致短短一周内产生了数个百万粉丝级的垃圾账号，严重破坏了平台的推荐算法和内容生态。
- **Red View (利用路径)**: 
    1. **漏洞挖掘**: 发现关注接口为 `/api/social/follow` 且仅接受简单的 POST 请求，且请求中不含有 CSRF Token 或随机性校验。
    2. **Payload 构造**: 编写简单的 HTML 表单并在 `onload` 事件中自动提交请求至平台 API。
    3. **社工诱导**: 将恶意链接包装成“热门福利”或“免费会员”信息，在平台内部或外部社交群组广泛分发。
    4. **静默执行**: 用户点击链接后，在浏览器已登录状态下，请求自动携带 Cookie 完成关注操作。
- **Blue View (防御缺失)**: 
    1. **CSRF 防护缺失**: 关键操作接口未引入 Referer/Origin 校验，也未强制要求非对称签名或随机 Token。
    2. **单日关注限额过高**: 平台对单个账号单日的关注上限设置过大，导致账号被瞬间利用。
    3. **前端点击劫持**: 部分页面未配置 `X-Frame-Options`，允许被嵌入恶意网页进行点击劫持。
- **策略借鉴**: 
    1. **全量接入 CSRF 随机 Token**: 对所有修改状态的 POST/PUT 请求强制校验双重 Cookie 或 Header 中的随机 Token。
    2. **操作限频与分级**: 对“关注”、“点赞”等高频互动接口增加基于滑动窗口的限频，并针对新老账号设置不同的风险阈值。
    3. **请求指纹签名**: 在移动端引入强签机制，所有 API 请求必须经过设备私钥签名，防止通过浏览器环境重放。
