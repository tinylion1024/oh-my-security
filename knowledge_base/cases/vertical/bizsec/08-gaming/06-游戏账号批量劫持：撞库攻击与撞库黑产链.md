# 游戏账号批量劫持：撞库攻击与撞库黑产链
- **所属领域**: 业务安全
- **威胁类型**: 账号安全与撞库攻击
- **事件回顾**: 某头部电竞平台在一次大版本更新后，发生大规模账号被盗事件。数十万个拥有高价值皮肤和虚拟货币的账号被非法登录，库存资产被迅速洗劫并转售。黑产工作室利用其他平台泄露的脱敏库进行“撞库”攻击，由于该平台当时未强制开启多因素认证（MFA），导致中招率极高。
- **Red View (利用路径)**: 
    1. **库源搜集**: 攻击者从暗网获取近期流出的某社交平台账号密码数据库（千万级量级）。
    2. **指纹绕过**: 使用大量的 SOCKS5 代理和自定义的 User-Agent 伪造正常用户指纹，规避 IP 频次限制。
    3. **自动化撞库**: 利用 Selenium 或 Puppeteer 编写自动化登录脚本，高并发测试目标平台的登录接口。
    4. **资产评估**: 撞库成功后，脚本自动调用“查号”接口，根据账号内的余额、等级、稀有皮肤自动进行价值评级并归档。
- **Blue View (防御缺失)**: 登录接口缺乏人机识别机制（图形验证码或滑块验证），且未对高频尝试失败的 IP 实施阶梯式锁定。系统对异地登录、新设备登录等异常场景缺乏实时告警与拦截逻辑。
- **策略借鉴**: 
    1. **全链路 MFA**: 对高价值账号操作、异地登录强制要求动态验证码（短信/邮件/身份验证器）。
    2. **威胁情报联动**: 接入外部撞库泄露库情报，对已在其他平台泄露的用户账号主动发出强更密码提示。
    3. **智能风控引擎**: 基于设备指纹、地理位置、用户习惯建模，对疑似撞库的登录请求进行实时拦截或引入更高强度的验证挑战。
