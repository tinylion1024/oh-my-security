# 游戏内存篡改与脚本注入：Lua 脚本执行漏洞
- **所属领域**: 业务安全
- **威胁类型**: 内存作弊与脚本注入
- **事件回顾**: 某知名 3D 手游在更新 Lua 脚本引擎后，被发现存在脚本执行漏洞。攻击者通过内存修改器（如 GameGuardian）修改了本地存储的脚本逻辑，实现了诸如“无冷却技能”、“全屏秒杀”等严重违规功能。由于该游戏的部分核心逻辑（如 CD 计算）为了极致流畅放在了客户端执行，导致作弊行为在公测后的排位赛中泛滥。
- **Red View (利用路径)**: 
    1. **脚本解包**: 攻击者利用工具解密游戏资源包中的 `.lua` 文件，分析游戏逻辑源码。
    2. **内存 Hook**: 发现脚本引擎加载 Lua 文件时，会将解密后的明文留在内存中。利用 Cheat Engine 找到 Lua 虚拟机（LVM）的内存基址。
    3. **热补丁注入**: 将修改后的恶意 Lua 代码直接覆盖至对应的内存区域，强行改变函数返回结果（如将 `CheckCooldown()` 永远返回 `True`）。
    4. **调用导出函数**: 利用反射机制直接调用游戏底层未开放的 API，实现刷取道具或瞬间移动。
- **Blue View (防御缺失)**: 核心业务逻辑在客户端与服务端之间分配失当，过度追求“客户端先行”而牺牲了安全性。Lua 脚本未进行高强度的混淆处理，且在运行内存中缺乏完整性校验。
- **策略借鉴**: 
    1. **逻辑重构**: 坚持“服务端权威”原则，技能冷却、数值计算、战利品产出必须由服务器强制判定。
    2. **脚本加固**: 采用自有的 Lua 字节码编译器，对脚本进行强混淆及加密分发，并引入运行时的脚本内存校验。
    3. **内存快照对比**: 在游戏运行期间定期抽查关键函数地址的哈希值，一旦发现内存数据被非法篡改，立即中断游戏连接并标记账号。
