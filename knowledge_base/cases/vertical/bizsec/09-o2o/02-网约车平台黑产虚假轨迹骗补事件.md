# 网约车平台黑产虚假轨迹骗补事件
- **所属领域**: 业务安全
- **威胁类型**: 接口伪造/LBS位置欺骗
- **事件回顾**: 某网约车平台为鼓励司机在早晚高峰出车，推出“高峰期满5单奖励200元”的冲单活动。随后后台出现大量行程轨迹极其规律、耗时极短的异常订单。黑灰产通过售卖作弊外挂，帮助数百名司机足不出户完成虚假跑单，导致平台单周内损失超200万元补贴。
- **Red View (利用路径)**: 
  1. **环境突破**: 司机将安卓手机Root，并刷入Magisk与Xposed框架。
  2. **系统API Hook**: 安装定制的“Fake Location”模块，劫持系统底层的 `LocationManager` API。
  3. **轨迹重放**: 黑产工具内置了真实的城市路网数据，可按照设定的车速（如40km/h）平滑地向APP持续注入虚假的GPS经纬度流。
  4. **司乘串通**: 配合黑产提供的“乘客端”刷单软件，实现原地下单、秒接单、匀速模拟行驶、到达终点的一键式提现操作。
- **Blue View (防御缺失)**: 
  1. **数据源单一信任**: 司机端APP直接信任OS返回的经纬度数据，未与设备的加速度计、陀螺仪等物理传感器进行交叉验证校验（如：车辆行驶中必然存在物理震动）。
  2. **反外挂能力缺失**: 客户端未部署RASP或环境完整性检测，对运行时注入和内存篡改毫无察觉。
- **策略借鉴**: 
  1. **多维传感器融合校验**: 结合GPS坐标与设备六轴传感器数据，建立车辆运动物理学模型。轨迹平滑且传感器数据静止的订单直接判定为伪造。
  2. **终端环境基线检查**: 在司机端APP启动及接单关键节点，强制执行反调试、反注入、检测双开与虚拟定位环境的完整性校验。
  3. **路网时空校验**: 后台引入第三方地图ETA（预估到达时间）与实时路况比对，对于无视早高峰拥堵、完美匀速到达的订单进行资金冻结处理。
