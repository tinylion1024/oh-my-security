# 本地生活商户刷榜与信誉操纵事件
- **所属领域**: 业务安全
- **威胁类型**: 数据污染/黑灰产水军
- **事件回顾**: 某本地生活与餐饮点评平台遭遇严重的数据污染。黑产团伙接单后，在短短一周内将数百家低分劣质餐厅刷至所在城市的“好评榜Top 10”。这种信誉操纵不仅严重破坏了平台算法的公正性，更导致大量真实用户受骗，引发平台信任危机。
- **Red View (利用路径)**: 
  1. **真实设备群控**: 摒弃传统的模拟器，黑产采用成百上千台真实的二手廉价手机构建“群控”机房，使用USB Hub连接中控电脑。
  2. **深度行为模拟**: 中控端下发Auto.js或ADB指令，模拟真实用户的滑动、点击。脚本会控制手机在商户页面停留3-5分钟（Dwell time），甚至模拟查看菜单的动作。
  3. **AIGC内容生成**: 接入大模型API自动生成结构完整、图文并茂的“高质量”虚假探店长文，并在各大平台盗用相关菜品图片进行伪装。
  4. **互相点赞提权**: 控制批量的水军账号对这些虚假评价进行点赞、评论互动，提升评价的权重和展示排序。
- **Blue View (防御缺失)**: 
  1. **风控规则一维化**: 传统风控仅盯防“高频API调用”和“单一IP聚合”，面对这种低频、慢速、物理设备分散的拟人化攻击彻底失效。
  2. **语义与图片审核薄弱**: 系统未能识别出评论内容的同质化特征，也未对接全网图片指纹库来识别盗图行为。
- **策略借鉴**: 
  1. **时空轨迹逻辑校验**: 分析账号的历史地理位置数据。若某用户常驻北京，却在无跨城出行记录的情况下密集评价上海的商户，应直接识别为异常。
  2. **图文同源性分析**: 建立评论库的向量检索模型，利用NLP计算文本相似度；提取图片的特征哈希（如Perceptual Hash），封杀全网盗图黑产。
  3. **社区团伙挖掘算法**: 利用图算法（如Louvain或PageRank）挖掘账户间的紧密互动关系网络，对集体同上同下、互赞互评的水军团伙进行一网打尽并降权。
