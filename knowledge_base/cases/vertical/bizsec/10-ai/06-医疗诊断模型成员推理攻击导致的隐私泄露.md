# 医疗诊断模型成员推理攻击导致的隐私泄露
- **所属领域**: 业务安全
- **威胁类型**: 隐私数据逆向泄露（Privacy Inversion）
- **事件回顾**: 某大型医院开源了一款用于辅助诊断罕见病的影像分析模型。安全研究员演示了针对该模型的“成员推理攻击”（Membership Inference Attack），通过构造特定的医学影像进行查询并分析输出的置信度分布，成功推断出多位名人的病历数据是否被包含在训练集中。由于该病症具有高度隐私性，该事件导致了严重的法律纠纷和公众对医疗 AI 信任度的危机。
- **Red View (利用路径)**: 
  1. **置信度指纹分析**: 攻击者通过 `Shadow Model` 训练，学习模型对训练集内样本（Overfitting）与集外样本输出概率的差异。
  2. **敏感特征枚举**: 针对已知隐私目标构造变体请求，观察预测概率的极小扰动。
  3. **隶属度判定**: 结合目标公开信息与模型反馈，判定目标数据是否贡献给了该模型的参数空间。
- **Blue View (防御缺失)**: 
  1. **过拟合严重**: 模型在训练集上追求极高准确率，导致其对训练样本的响应特征过于独特。
  2. **输出信息过载**: API 返回了完整的 Softmax 概率分布向量，而不是单一的分类标签，泄露了模型内部状态。
- **策略借鉴**: 
  1. **差分隐私训练 (DP-SGD)**: 在模型训练阶段引入拉普拉斯噪声，确保单条样本的加入不会显著改变模型参数。
  2. **响应限制 (Label-only API)**: 对外仅返回最终预测标签，隐藏概率分值，阻断基于概率差异的推理。
  3. **正则化约束**: 增加 Dropout 比例或采用提前终止（Early Stopping）训练，强制减轻模型对特定样本的记忆效应。
