# 案例：云端 VPC 安全组配置失误导致的 RDP/SSH 暴力破解与勒索软件植入

## 事件回顾
一家医疗科技公司在经历大规模业务上云时，由于网络架构规划的不成熟，工程师在配置虚拟私有云（VPC）的安全组（Security Group / Firewall Rules）时，为了调试方便，临时将数十台关键业务的 Windows 和 Linux 服务器的远程管理端口（RDP 3389 和 SSH 22）对 `0.0.0.0/0`（全网公开）开放。由于调试结束后遗忘了收回权限，这一巨大的安全敞口被国际勒索软件团伙盯上。攻击者通过持续数日的自动化字典暴力破解，成功猜解出服务器的弱口令，登录并关闭了安全软件，最终在整个内网横向扩散，加密了所有患者档案数据库，勒索高额解密费用。

## Red View (利用路径)
1. **端口暴露侦测**：攻击者利用全网扫描工具（如 Masscan），快速定位到了该企业公网 IP 段上大量对外部完全开放的 3389 (RDP) 和 22 (SSH) 端口。
2. **大规模暴力破解**：针对发现的 RDP 和 SSH 端口，攻击者使用包含常用弱口令和泄露密码的庞大字典，利用 Hydra 等工具发起持续的自动化密码暴力破解攻击，最终成功爆破出一个拥有系统管理员权限的弱口令账号。
3. **横向移动与勒索执行**：获取初始入口后，攻击者登录服务器，利用内网未隔离的特性（扁平化网络）使用 Mimikatz 抓取更多内网凭证，横向感染其他核心数据库节点，最终统一下发指令，运行高强度加密勒索软件。

## Blue View (防御缺失)
- **网络访问控制形同虚设**：云安全组配置严重违背了最小攻击面原则。将高危管理端口暴露给全球互联网，直接将内部基础设施暴露在了勒索团伙的炮火之下。
- **身份认证机制脆弱**：即使端口暴露，若采用了公钥认证（SSH）和强密码策略，攻击也不会轻易得手。然而，系统允许使用密码登录，且未能强制执行复杂的强密码策略和账户锁定机制（防爆破）。
- **缺乏内网隔离（扁平化架构）**：VPC 内部未能实施细粒度的微隔离（Micro-segmentation）。当一台边缘跳板机被攻陷后，内网所有其他服务器之间畅通无阻，导致勒索软件得以在内网无障碍爆发。

## 策略借鉴
1. **实施零信任远程访问 (Zero Trust Network Access)**：绝对禁止通过公网直接暴露 RDP 或 SSH。应采用 AWS Systems Manager (SSM) Session Manager、Azure Bastion 或是现代的零信任代理方案（如 Cloudflare Access, Tailscale），实现无需开放入站端口、基于强身份认证（OIDC+MFA）的远程运维接入。
2. **加固操作系统级身份验证**：全面废除 SSH 密码登录，强制使用 Ed25519 或 RSA 等强非对称密钥对进行身份验证。对于 Windows 环境，配置 Account Lockout 策略，并部署多因素认证（MFA），确保即使密码被窃取也无法单凭密码登入。
3. **推行 VPC 内部微隔离与自动化态势检查**：利用云原生安全组或网络 ACL，按业务模块划分安全域，实施内网间的默认拒绝（Default Deny）策略。同时，部署云安全态势管理（CSPM）工具持续巡检网络边界，若发现安全组允许 `0.0.0.0/0` 访问 22/3389 端口，立即自动修复并阻断。
