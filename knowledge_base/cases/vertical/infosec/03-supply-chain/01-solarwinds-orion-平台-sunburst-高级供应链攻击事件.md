# SolarWinds Orion 平台 Sunburst 高级供应链攻击事件
- **所属领域**: 信息安全
- **威胁类型**: 软件供应链污染 / APT 攻击
- **事件回顾**: 2020年底，知名网络管理软件供应商 SolarWinds 遭到国家级 APT 组织的入侵。攻击者在 SolarWinds Orion 产品的构建流程中植入了名为 Sunburst 的恶意后门代码。由于该软件被全球数千家政府机构、军方以及世界500强企业广泛使用，恶意更新包在带有 SolarWinds 合法数字签名的情况下被分发至客户网络中。这导致攻击者能够绕过绝大多数边界防御，直接在目标内网中建立长期的持久化访问，造成了信息安全史上影响最深远的供应链灾难之一。
- **Red View (利用路径)**: 攻击者首先通过未知途径攻破了 SolarWinds 的内部网络，潜伏长达数月之久。在静默观察其 CI/CD 构建流水线后，编写了专用的恶意代码注入工具。该工具在 MSBuild 编译期间动态替换了合法的源代码文件，将后门编译进最终的 `SolarWinds.Orion.Core.BusinessLayer.dll` 文件中。后门代码极为隐蔽，采用 DGA 算法生成 C2 域名，并伪装成 Orion 的常规通信流量，甚至会检测本地的安全分析工具以逃避沙箱分析。
- **Blue View (防御缺失)**: 此次攻击暴露了开发基础设施防御的薄弱环节。首先，SolarWinds 内部缺乏对构建环境的严格零信任隔离，开发人员权限过大；其次，在代码签名发布之前，缺乏对比源代码和编译后二进制文件的可信度验证机制；最后，受害客户在部署这类高特权的网管软件时，通常给予了不受限制的外网访问权限，且缺乏针对此类应用异常网络行为的基线异常监控。
- **策略借鉴**: 引入 SLSA (Supply-chain Levels for Software Artifacts) 框架保障构建流程的安全；实施构建环境的完全隔离和不可变基础设施设计；要求软件供应商提供 SBOM (软件物料清单)；对内网中的高权限管理软件实施网络微分段和最小权限原则，持续监控其出站流量。
