# Codecov Bash Uploader 凭证窃取与 CI/CD 供应链污染事件
- **所属领域**: 信息安全
- **威胁类型**: CI/CD 基础设施劫持 / 凭证窃取
- **事件回顾**: 2021年4月，知名代码覆盖率测试工具提供商 Codecov 披露其 Bash Uploader 脚本遭到黑客篡改。由于攻击者获取了 Codecov 用于存储该脚本的 Google Cloud Storage 服务的访问权限，他们修改了公开提供的 bash 脚本内容。当数以万计的客户在其 CI/CD 流水线中自动拉取并执行该脚本以上传测试覆盖率报告时，恶意代码被同步触发，窃取了客户 CI/CD 环境变量中存储的各种敏感凭证、API 密钥和代码仓库访问令牌。
- **Red View (利用路径)**: 攻击者首先通过获取一个存在缺陷的 Docker 镜像提取到了 Codecov GCS 存储桶的写权限凭证。随后，攻击者没有破坏系统，而是巧妙地在 `bash` 脚本的某一行加入了一段数据外传代码。当受害者在诸如 GitHub Actions、Travis CI 等自动化流水线中运行 `curl -s https://codecov.io/bash | bash` 时，该恶意代码会遍历环境中的所有环境变量 (env)，并将其通过 HTTP POST 请求发送至攻击者控制的远程服务器。由于 CI/CD 环境中通常存放着极高权限的部署密钥，攻击者借此进行了大规模的横向渗透。
- **Blue View (防御缺失)**: 客户在使用外部提供的脚本时，盲目信任了 `curl | bash` 这种危险的执行模式，而没有进行任何校验。同时，CI/CD 环境往往被配置为"大杂烩"，存放了过多的、未经细粒度权限控制的全局变量和长期有效的敏感凭证。此外，Codecov 自身的云存储桶访问权限管理松懈，且未能及时发现对外发布的脚本 hash 发生了非预期的变更。
- **策略借鉴**: 坚决摒弃未经验证的 `curl | bash` 操作，引入校验和 (Checksum) 或数字签名验证机制；在 CI/CD 中实施严格的凭证管理，利用动态短效凭证 (如 OIDC) 替代长期有效的静态密钥；隔离不同项目的流水线环境，遵循最小权限原则；对第三方工具和依赖定期进行完整性审查。
