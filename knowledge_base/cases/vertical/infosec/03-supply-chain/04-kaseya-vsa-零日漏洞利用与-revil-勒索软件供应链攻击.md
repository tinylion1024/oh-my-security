# Kaseya VSA 零日漏洞利用与 REvil 勒索软件供应链攻击
- **所属领域**: 信息安全
- **威胁类型**: MSP 供应链攻击 / 勒索软件分发
- **事件回顾**: 2021年7月，REvil 勒索软件团伙发起了一场精心策划的全球性供应链攻击。他们利用了 IT 托管服务提供商 (MSP) 广泛使用的 Kaseya VSA 远程监控和管理软件中的多个零日漏洞。通过攻破全球数十家 MSP 服务商的 Kaseya VSA 服务器，攻击者利用该软件对下游客户的高级控制权限，将勒索软件伪装成正常的软件更新推送给数千家中小型企业，导致大面积的业务停摆和数据加密。
- **Red View (利用路径)**: 攻击者利用 Kaseya VSA Web 界面中存在的身份验证绕过漏洞 (CVE-2021-30116)、任意文件上传漏洞和命令注入漏洞，成功接管了暴露在公网的 VSA 管理服务器。随后，攻击者利用 VSA 的合法管理通道执行自动化脚本，关闭受害者机器上的反病毒软件 (如 Microsoft Defender)，并部署了一个包含 REvil 勒索载荷的恶意负载。由于 VSA 代理在终端上以最高系统权限 (SYSTEM) 运行，整个加密过程几乎没有遇到任何阻碍，且规避了传统基于网络边界的检测。
- **Blue View (防御缺失)**: 此次事件暴露了对第三方 MSP 软件过度信任的风险。许多 MSP 服务商为了管理方便，将原本应置于内网或通过 VPN 访问的管理后台直接暴露在公网，增加了被零日漏洞攻击的风险。同时，由于 VSA 软件被视为可信应用，终端安全软件通常将其加入了白名单，这使得攻击者利用可信进程来禁用安全防护并投放勒索软件变得轻而易举。
- **策略借鉴**: 对高级特权的 IT 管理工具实施纵深防御，避免将其管理接口直接暴露于互联网；实施细粒度的端点安全策略，防止可信管理进程滥用权限关闭安全软件机制 (如使用防篡改保护)；对 MSP 等第三方供应商进行严格的安全审查和隔离，限制其在内网中的横向移动范围；建立快速响应和灾难恢复备份体系以应对勒索威胁。
