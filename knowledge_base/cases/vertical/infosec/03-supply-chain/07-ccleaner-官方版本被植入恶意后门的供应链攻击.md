# CCleaner 官方版本被植入恶意后门的供应链攻击
- **所属领域**: 信息安全
- **威胁类型**: 软件分发劫持 / 后门植入
- **事件回顾**: 2017年9月，Avast 旗下的知名系统清理工具 CCleaner 被曝出其官方分发的版本 (v5.33.6162 和 Cloud v1.07.3191) 包含了恶意的后门代码。由于 CCleaner 拥有庞大的用户基础（下载量达数千万），这次攻击导致全球超过 200 万台计算机感染了恶意软件。这是一次典型的高级持续性威胁 (APT) 组织针对知名软件供应链发起的攻击。
- **Red View (利用路径)**: 攻击者提前数月入侵了 Piriform (CCleaner的开发商，后被Avast收购) 的内部网络。他们成功渗透到开发环境，并修改了编译环境，使得合法的 CCleaner 源代码在编译时被注入了恶意 payload。带有后门的执行文件不仅被托管在官方下载服务器上，还拥有合法的数字签名。恶意软件感染用户后，会收集主机信息（如计算机名、已安装软件列表、MAC地址等）并发送给 C2 服务器。攻击者随后根据这些信息进行筛选，只对特定的高价值目标（如科技公司和电信运营商）下发第二阶段的恶意载荷。
- **Blue View (防御缺失)**: 开发商的构建环境存在严重的安全漏洞，允许攻击者长期潜伏并篡改编译过程。数字签名的存在使得传统基于黑名单的反病毒软件失效，因为签名被认为是"可信的"。对于终端用户而言，直接从官方渠道下载的软件通常被默认信任，缺乏对软件行为的持续动态监控，使得后门能够隐蔽运行收集情报。
- **策略借鉴**: 软件供应商必须实施极其严格的内部网络隔离和监控，特别是针对 CI/CD 和代码签名基础设施；实施构建可重复性验证（Reproducible Builds），确保构建产物完全由源代码决定且未被篡改；在终端防护上，引入基于行为分析的端点检测与响应 (EDR) 方案，而不应仅仅依赖于静态签名和数字证书的可信度；用户应保持警惕，即使是官方软件也可能被污染。
