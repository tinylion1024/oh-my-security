# ASUS Live Update 工具被劫持的 ShadowHammer 供应链攻击
- **所属领域**: 信息安全
- **威胁类型**: 硬件厂商更新劫持 / 精准目标攻击
- **事件回顾**: 2019年，卡巴斯基实验室披露了被称为 "Operation ShadowHammer" 的高级供应链攻击事件。攻击者入侵了知名硬件制造商华硕 (ASUS) 的内部网络，并劫持了其官方的 ASUS Live Update 实用程序。攻击者通过该合法渠道推送了带有后门的恶意更新，这些更新使用了两份合法的华硕数字证书进行签名，估计有超过一百万台华硕设备下载并安装了受感染的软件。
- **Red View (利用路径)**: 攻击者在华硕服务器上植入了被篡改的 Live Update 软件。当用户的电脑通过该工具检查更新时，就会下载并执行恶意代码。然而，这次攻击极其隐蔽和精准：恶意软件内置了一个硬编码的 MAC 地址列表（约包含 600 个目标）。恶意代码在运行后，首先会检查宿主机的 MAC 地址是否在这个硬编码列表中。如果匹配，才会从 C2 服务器下载进一步的恶意载荷；如果不匹配，恶意软件则不执行任何操作并保持休眠状态，这使得它极难被安全研究人员和普通用户发现。
- **Blue View (防御缺失)**: 华硕对其软件更新服务器和数字证书管理存在重大疏漏，导致攻击者能够利用官方基础设施分发恶意软件。与 CCleaner 事件类似，合法的数字签名让恶意文件轻松绕过了大多数反病毒防护。更深层次的问题是，厂商的更新机制在设计时未考虑到服务器被完全接管的极端情况，缺乏端到端的内容完整性校验。
- **策略借鉴**: 硬件及软件厂商需要保护其代码签名密钥如同保护最高机密，使用硬件安全模块 (HSM) 和多重授权机制进行管理；对分发服务器进行持续的完整性监控和异常流量检测；对于企业用户，不要盲目信任硬件厂商预装的自动更新工具，可以通过网络防火墙或代理控制此类工具的更新行为；EDR 系统需要加强对"可信"软件产生异常网络连接或释放未知文件的检测能力。
