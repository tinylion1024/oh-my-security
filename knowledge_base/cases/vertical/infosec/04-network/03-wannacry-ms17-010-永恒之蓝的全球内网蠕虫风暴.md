# WannaCry (MS17-010) 永恒之蓝的全球内网蠕虫风暴
- **所属领域**: 信息安全
- **威胁类型**: 内网蠕虫传播 / 勒索软件
- **事件回顾**: 2017年5月，WannaCry 勒索软件在全球范围内爆发，短短几天内感染了 150 多个国家的数十万台计算机，造成了数百亿美元的损失。WannaCry 的特殊之处在于它并非主要依赖传统的钓鱼邮件传播，而是利用了被泄露的 NSA 黑客工具 "永恒之蓝" (EternalBlue)。该工具利用了 Windows SMBv1 协议中的一个严重漏洞 (MS17-010)，实现了无交互的内网蠕虫式扩散。
- **Red View (利用路径)**: 当一台计算机感染了 WannaCry 后，恶意软件会将其作为扫描节点，主动扫描本地子网和随机公网 IP 的 445 端口 (SMB 服务)。如果目标主机存在 MS17-010 漏洞且未打补丁，"永恒之蓝" 漏洞利用模块会向其发送特制的 SMB 数据包，触发内核级的缓冲区溢出。成功利用后，攻击者直接获取目标系统的 SYSTEM 权限，然后静默植入 DoublePulsar 后门并下载执行 WannaCry 勒索载荷。这种自我复制的机制导致了指数级的感染速度。
- **Blue View (防御缺失)**: 这场风暴揭示了全球大量组织在基础网络卫生方面的严重缺失。首先是补丁管理严重滞后，微软早在攻击爆发前几个月就发布了 MS17-010 的修复补丁，但大量企业由于业务兼容性或管理不善未能及时应用。其次是网络架构过于开放，大量内网依然保留着老旧且极不安全的 SMBv1 协议，且各部门、网段之间缺乏访问控制列表 (ACL) 和防火墙隔离，使得勒索蠕虫能够在内网中肆意横向移动。
- **策略借鉴**: 彻底废弃并在全网范围内禁用过时、不安全的协议 (如 SMBv1)；建立自动化且强制性的紧急补丁部署机制；实施内网微分段策略，默认拒绝终端之间的横向文件共享流量，将网络划分为更小、更安全的区域；在边界和内部网络部署具备蠕虫和端口扫描特征检测能力的入侵检测系统 (IDS/IPS)。
