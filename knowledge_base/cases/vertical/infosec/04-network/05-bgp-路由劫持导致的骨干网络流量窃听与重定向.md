# BGP 路由劫持导致的骨干网络流量窃听与重定向
- **所属领域**: 信息安全
- **威胁类型**: 网络基础设施攻击 / 流量劫持
- **事件回顾**: 边界网关协议 (BGP) 是互联网的骨干路由协议，但在设计之初并未充分考虑安全认证机制。BGP 路由劫持是指一个自治系统 (AS) 恶意或错误地宣告不属于它的 IP 前缀。这会导致全球的路由器更新其转发表，将原本发送给合法目标的流量错误地发送到劫持者的网络中。这类事件不仅会导致大规模的服务中断，更被高级攻击者用于中间人攻击 (MitM)、流量窃听和加密货币盗窃。
- **Red View (利用路径)**: 攻击者控制了一个合法的 AS（或者入侵了某 ISP 的 BGP 路由器），并在 BGP 宣告中声称自己拥有目标机构的 IP 地址块（通常宣告更具体的子网掩码，如 /24 甚至 /25，因为 BGP 偏好最具体路由）。周边的路由器不加验证地接受并传播这一声明。结果，流向目标机构的流量被重定向到攻击者控制的服务器。攻击者可以在这里记录流量、解析未加密的敏感信息，或者通过伪造证书等手段进行中间人攻击。为了不被轻易发现，攻击者有时会在记录流量后，通过暗线将其重新路由回原始目标，形成无感知的窃听。
- **Blue View (防御缺失)**: 全球互联网的底层信任机制过于脆弱。长期以来，ISP 之间基于互信进行 BGP 路由更新，缺乏对路由宣告合法性的强制加密验证机制。受害企业往往无法主动防御此类攻击，因为劫持发生在互联网骨干层，而非企业自己的防火墙内。此外，当流量被劫持后，如果应用层未强制使用 HSTS、DNSSEC 或相互认证的强加密传输，数据将完全暴露在攻击者面前。
- **策略借鉴**: 推动和采用 RPKI (资源公钥基础设施) 等路由安全框架，确保 BGP 路由宣告的加密验证；网络运营商应实施 BCP38 等网络入口过滤最佳实践；企业应建立全球范围内的 BGP 路由和延迟监控机制（如使用 RIPE Stat 或第三方 BGP 监控服务），一旦发现 IP 前缀被异常宣告立即触发告警；对于关键业务，实施严格的应用层端到端加密（TLS 1.3）和证书透明度监控，使流量即使被劫持也无法被解密和篡改。
