# VMware 企业级基础组件漏洞导致的数据中心沦陷
- **所属领域**: 信息安全
- **威胁类型**: 虚拟化基础设施攻击 / RCE
- **事件回顾**: 现代企业数据中心高度依赖 VMware vCenter、ESXi 和 NSX 等虚拟化管理组件。近年来，VMware 产品频繁爆出 CVSS 评分为 9.8 甚至 10.0 的严重漏洞（如 CVE-2021-21972, CVE-2021-22005 等）。这些漏洞允许未授权的攻击者通过管理界面的特定端口（如 443 端口），向虚拟化平台的核心控制节点上传恶意 WebShell 或直接执行代码，进而掌控整个数据中心的虚拟机资源。
- **Red View (利用路径)**: 攻击者利用网络扫描引擎（如 Shodan 或内网资产测绘工具）定位到暴露的 vCenter Server 实例。以 CVE-2021-21972 为例，该漏洞存在于 vCenter 的 vRealize Operations 插件中。攻击者构造一个恶意的 tar 压缩包文件，通过特定的 API 端点 (`/ui/vropspluginui/rest/services/uploadova`) 上传。由于服务端在解压时没有限制路径（路径遍历漏洞），恶意文件（如 JSP WebShell）会被直接解压并写入到具有执行权限的 Web 目录下。获取控制权后，攻击者可直接访问底层的 ESXi 主机，窃取所有虚拟机的内存快照、导出域控 NTDS.dit 文件，或者直接对虚拟机磁盘文件 (.vmdk) 进行勒索加密。
- **Blue View (防御缺失)**: 很多企业在部署数据中心时，为了运维的便利，将 vCenter 的管理接口与普通的办公网混合，甚至直接映射到公网上。虚拟化管理平台被视为"底层设施"，往往不在常规的应用层漏洞扫描和 WAF 防护范围内。此外，这些管理平台默认拥有对所有虚拟机的超级控制权，一旦被攻破，传统的基于虚拟机内部操作系统的安全软件（如防病毒软件）将完全无法感知底层的物理/虚拟化层窃取行为。
- **策略借鉴**: 实施严格的管理平面隔离，设立专用的带外管理网段 (OOB)，必须通过堡垒机并结合多因素认证 (MFA) 才能访问虚拟化组件；利用网络微分段技术 (如 VMware NSX 本身的功能) 限制虚拟机之间的横向流量；将虚拟化基础架构的补丁管理提升至最高优先级，建立紧急响应通道；部署主机层面的文件完整性监控 (FIM)，监控 vCenter 等关键服务器系统目录的非预期修改。
