# Kerberoasting 攻击引发的内网服务凭证接管
- **所属领域**: 信息安全
- **威胁类型**: 身份认证协议攻击 / 密码破解
- **事件回顾**: 在 Windows 活动目录 (AD) 内网环境中，Kerberoasting 是一种极具威胁的后渗透攻击技术。与需要高权限的攻击不同，任何一个普通的域用户账户都可以发起 Kerberoasting 攻击。攻击者利用 Kerberos 协议的设计机制，请求内网中服务主机的服务票据 (Service Ticket)，由于该票据的加密部分使用了服务账户的密码哈希，攻击者可以将其导出并在离线环境中暴力破解，从而获取高权限服务账户（如 SQL Server、Exchange）的明文密码，实现内网横向移动和权限提升。
- **Red View (利用路径)**: 攻击者首先通过钓鱼或其他手段获取一个普通域用户的权限。然后，使用工具（如 Rubeus 或是 PowerShell 脚本）查询 LDAP 目录，寻找注册了服务主体名称 (SPN) 且非机器账户的域用户（通常这些账户由管理员手动创建用于运行服务，且密码策略较弱）。找到后，攻击者以普通用户身份向域控制器 (KDC) 申请访问这些服务的 TGS 票据。域控会返回使用该服务账户 NTLM 哈希加密的票据。攻击者将票据保存到本地，利用 Hashcat 或 John the Ripper 等工具结合强大的字典进行离线爆破。一旦破解成功，攻击者即拥有了该服务账户的权限，往往能直接接管数据库或应用服务器。
- **Blue View (防御缺失)**: 很多企业在部署内网服务时，习惯使用手动创建的域用户作为服务账户，并且为了避免服务中断，往往为其设置了永不过期且强度极弱的密码。此外，普通域用户请求任意服务票据是 Kerberos 协议的合法功能，这导致传统的基于流量或签名的安全设备很难将这种请求识别为恶意攻击，使得攻击行为在庞大的内网认证流量中隐形。
- **策略借鉴**: 实施强密码策略，要求所有拥有 SPN 的服务账户必须使用 25 位以上的高强度随机密码；尽可能使用组托管服务账户 (gMSA) 来替代普通用户账户运行服务，交由 AD 自动管理其复杂的密码；监控内网中异常的 TGS 票据请求行为（如同一账户在短时间内请求大量不同服务的 RC4 加密票据）；通过蜜罐技术 (Honeytoken)，创建虚假的、带有 SPN 的诱饵账户，一旦检测到针对该账户的票据请求立即告警。
