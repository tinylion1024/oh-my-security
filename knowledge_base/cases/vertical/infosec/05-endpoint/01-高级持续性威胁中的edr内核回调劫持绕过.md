# [高级持续性威胁中的EDR内核回调劫持绕过]
- **所属领域**: 信息安全 - 端点安全
- **威胁类型**: EDR规避 / Rootkit / BYOVD
- **事件回顾**: 某跨国企业办公网络部署了行业领先的EDR。在例行审计中，安全团队发现多台核心开发终端存在未知加密外连流量，但EDR控制台未产生任何告警。取证表明，攻击者通过钓鱼邮件投递带有伪造数字签名的恶意驱动程序。该驱动利用Windows内核已知漏洞加载并劫持系统内核回调机制（如 `PsSetCreateProcessNotifyRoutine`），在底层“致盲”了EDR探针，使其无法捕捉恶意进程的创建与内存注入行为。
- **Red View (利用路径)**: 攻击者获取初始访问权限后，利用BYOVD（Bring Your Own Vulnerable Driver）技术，加载一个存在漏洞的合法旧版防病毒驱动。利用该漏洞，攻击者在内核态获得任意内存读写能力，随后篡改EDR驱动在内核注册的回调函数地址，阻止EDR收集遥测数据，最终在完全隐蔽的情况下执行内存中的有效载荷。
- **Blue View (防御缺失)**: 防御方过度依赖用户态EDR agent，未能有效监控驱动程序加载。系统的HVCI（基于虚拟化的代码完整性保护）和微软驱动签名强制（DSE）策略未被严格执行或被绕过。此外，团队未能对异常的驱动加载事件（特别是加载已知脆弱驱动）建立监控基线。
- **策略借鉴**: 端点安全不能仅依赖单一EDR，必须实施深度防御：1）开启并严格执行HVCI，防止恶意驱动加载；2）使用驱动程序拦截规则（如微软的易受攻击驱动程序黑名单）阻断BYOVD攻击；3）结合网络层的NDR进行多维度的异常行为关联分析，弥补端点视角的缺失。
