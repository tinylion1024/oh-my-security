# [知名IT运维管理平台引发的泛网供应链攻击]
- **所属领域**: 信息安全 - 端点安全
- **威胁类型**: 供应链攻击 / 远程代码执行 / 隐蔽横向移动
- **事件回顾**: 全球数千家企业同时遭遇不明勒索软件攻击导致系统锁定。调查发现这些企业均使用了某知名IT网络管理软件。攻击者入侵了该软件供应商的构建环境，在官方发布的软件更新补丁中植入恶意后门。当受害企业的IT管理员通过自动化运维平台向全网端点推送合法更新时，恶意后门被一并分发并以最高系统权限（SYSTEM）在所有端点上执行，防御体系瞬间崩溃。
- **Red View (利用路径)**: 攻击者突破软件供应商的代码托管和构建服务器，将恶意DLL文件替换原有合法组件，并使用供应商真实数字证书签名。在客户环境中，由于该管理软件需要高权限管理端点，恶意代码随更新分发后自动继承最高权限，关闭端点防病毒进程，并横向下载勒索软件载荷执行。
- **Blue View (防御缺失)**: 防御方盲目信任具有合法数字签名的第三方软件更新，缺乏对“受信任”应用程序异常行为的零信任验证。端点防护策略通常将IT管理软件的进程列入白名单，导致其后续恶意子进程创建（如无端启动cmd下载文件）未被拦截。
- **策略借鉴**: 应对供应链攻击需贯彻零信任架构理念：1）收缩可信应用的权限范围，即便是合法的管理软件，也应限制其执行异常高危行为；2）实施微隔离，限制IT管理服务器到端点的单向访问能力；3）建立补丁更新的灰度发布与沙箱预验证机制，避免全网一次性推送。
