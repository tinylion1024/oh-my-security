# [利用SMB与PsExec的勒索软件端点快速蠕虫化传播]
- **所属领域**: 信息安全 - 端点安全
- **威胁类型**: 勒索软件 / 横向移动 / 蠕虫化传播
- **事件回顾**: 某医疗机构在深夜遭到勒索软件袭击，短时间内超80%的医生工作站和服务器被加密。安全团队复盘发现，初始感染源仅是一台护理站终端（因点击钓鱼邮件）。该勒索软件在本地提权后，迅速利用内网中泛滥的SMB共享和管理员常用的PsExec工具，像蠕虫一样在局域网内指数级蔓延，绕过了多个边界防火墙。
- **Red View (利用路径)**: 勒索载荷在初始端点上利用凭证窃取工具从内存中抓取了域管理员哈希。随后利用自带的横向移动模块，扫描同一网段内的开放445端口（SMB），并使用获取到的凭证通过PsExec或WMI在目标端点上远程执行服务创建命令，将勒索自身复制过去并启动。这种基于合法管理协议的传播方式速度极快且难以察觉。
- **Blue View (防御缺失)**: 内网端点之间未做网络隔离，任意工作站均可互相通过SMB协议通信。终端本地管理员密码使用了统一镜像导致密码相同（缺乏LAPS保护）。安全团队没有对PsExec等远程管理工具的使用范围进行限制，未监控端点之间异常的大规模连接请求。
- **策略借鉴**: 阻止端点间横向移动是控制勒索软件爆炸半径的关键：1）全面部署Windows LAPS（本地管理员密码解决方案），防止凭证窃取后的复用；2）在主机防火墙上配置端点间通信隔离，禁止非必要的SMB/RDP横向连接；3）限制PsExec等系统管理工具仅能由特定跳板机发起，监控异常服务创建。
