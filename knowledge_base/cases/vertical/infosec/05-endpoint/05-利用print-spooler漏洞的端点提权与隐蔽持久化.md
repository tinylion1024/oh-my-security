# [利用Print Spooler漏洞的端点提权与隐蔽持久化]
- **所属领域**: 信息安全 - 端点安全
- **威胁类型**: 权限提升 (Privilege Escalation) / 零日漏洞利用
- **事件回顾**: 某大型金融机构内网遭APT攻击。攻击者最初仅拥有普通员工低权限账号。尽管该端点部署了严格的应用白名单和最新防病毒软件，攻击者仍成功获得SYSTEM权限并安装持久化后门。深入调查显示，攻击者利用了Windows Print Spooler服务中未修补的漏洞（PrintNightmare），通过加载恶意打印机驱动实现了本地特权提升。
- **Red View (利用路径)**: 攻击者获取低权限Shell后，发现目标系统默认开启了Print Spooler服务。利用 `RpcAddPrinterDriver` 等API调用，攻击者向本地系统提供了一个特制包含恶意DLL的驱动路径。由于Print Spooler服务验证阶段存在逻辑缺陷，导致系统以SYSTEM权限加载并执行该恶意DLL。攻击者借此创建最高权限的计划任务，实现隐蔽持久化。
- **Blue View (防御缺失)**: 蓝队未能及时跟踪和修补关键系统组件漏洞，未针对不必要的系统服务进行加固。大量无需连接打印机的终端（如开发服务器）默认开启Print Spooler服务，增加攻击面。端点防御产品未能监控到 `spoolsv.exe` 的异常子进程创建和文件写入行为。
- **策略借鉴**: 端点服务攻击面管理至关重要：1）严格遵循最小权限原则，通过GPO禁用所有非必须端点和服务器上的Print Spooler等高风险遗留服务；2）对 `spoolsv.exe` 建立严密监控，特别是监控其在 `System32\spool\drivers` 目录下的异常DLL写入加载行为；3）加快高危0day漏洞的虚拟补丁或临时缓解措施下发。
