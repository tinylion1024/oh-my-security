# [深植于SPI闪存的UEFI固件级Rootkit持久化事件]
- **所属领域**: 信息安全 - 端点安全
- **威胁类型**: 固件攻击 / UEFI Rootkit / 极深持久化
- **事件回顾**: 某涉密终端在发现感染恶意软件后，IT人员对其进行了彻底硬盘格式化和操作系统重装。几周后，该终端再次出现相同恶意网络信标。经硬件层面逆向取证证实，终端感染了罕见的UEFI级别Rootkit。该攻击深植于主板的SPI闪存芯片中，在操作系统重装甚至更换硬盘后依然存活并持续感染系统。
- **Red View (利用路径)**: 攻击者初步获取操作系统管理员权限后，利用主板配置不当暴露的SPI闪存写保护绕过漏洞，修改了UEFI固件。注入的恶意UEFI模块利用DXE（驱动执行环境）阶段，在操作系统内核加载前，篡改了Windows引导加载程序（Bootloader），确保每次系统启动时恶意载荷都能自动重新注入操作系统内存。
- **Blue View (防御缺失)**: 硬件层面安全防护滞后。终端未正确配置Secure Boot（安全启动），或固件本身漏洞导致安全信任链被打破。防御方缺乏对UEFI完整性的自动化检测手段，EDR工具视界局限于操作系统层，无法洞察固件层的异常改动。
- **策略借鉴**: 防范固件级攻击必须建立从硬件到操作系统的信任链：1）强制启用并妥善配置Secure Boot和Intel Boot Guard等固件验证机制，防范未签名引导组件执行；2）保持BIOS/UEFI固件持续更新，修补SPI写保护绕过漏洞；3）引入能够读取和验证SPI闪存完整性的专用端点扫描工具。
