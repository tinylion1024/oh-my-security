# [利用进程空心化实现的仅内存驻留Dropper攻击]
- **所属领域**: 信息安全 - 端点安全
- **威胁类型**: 内存注入 / 逃避防御 / 进程空心化
- **事件回顾**: 某软件研发公司代码库遭窃取。攻击者使用伪装成PDF附件的钓鱼邮件。一旦执行，并不在硬盘释放任何木马文件，而是直接在系统内存中解密执行核心恶意载荷。这种“仅内存驻留”（Memory-Only）机制使传统防病毒软件毫无反应，因为扫描器主要是基于文件I/O事件触发的。
- **Red View (利用路径)**: 攻击者利用了进程空心化（Process Hollowing）技术。恶意Dropper以挂起状态（Suspended）创建合法系统进程（如 `svchost.exe`）。随后解除合法进程内存中的原始代码，并将事先解密的恶意Shellcode注入到该内存空间中，恢复执行。操作系统视角运行的是合法数字签名进程，内部已被替换成木马逻辑，有效规避进程白名单。
- **Blue View (防御缺失)**: 端点防护主要依赖静态文件扫描和基本启发式引擎，缺乏对运行中进程内存空间的深度检查能力。未能监控敏感API调用序列（如 `CreateProcess`, `NtUnmapViewOfSection`, `VirtualAllocEx` 等实施进程空心化的典型特征）。
- **策略借鉴**: 对抗内存级威胁需要动态行为分析技术：1）部署下一代防病毒（NGAV）或EDR产品，利用API Hooking和ETW技术监控进程注入行为典型API调用链；2）定期执行自动化内存扫描，寻找内存页面权限异常（如RWX）区域；3）强化终端基线，禁用不必要宏执行，切断初始执行链。
