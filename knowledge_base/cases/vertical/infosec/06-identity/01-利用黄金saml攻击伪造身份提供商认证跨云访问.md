# [利用黄金SAML攻击伪造身份提供商认证跨云访问]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 联邦身份滥用 / 令牌伪造 / 高级持续性威胁
- **事件回顾**: 某大型企业在使用SSO解决方案访问Microsoft 365等SaaS应用时遭入侵。攻击者在未获取用户密码和绕过MFA的情况下，成功登录多位高管云端账号。经取证，这是一起“黄金SAML”攻击事件，攻击者窃取了企业本地IdP（如AD FS）的私钥，从而任意伪造SAML断言。
- **Red View (利用路径)**: 攻击者通过内网横向移动入侵了AD FS服务器，导出用于签名SAML令牌的数字私钥。拥有签名密钥后，攻击者在外部机器伪装成合法IdP，伪造包含任意高权限用户身份声明的SAML断言。云端SaaS服务商配置了对该IdP的信任，无条件接受伪造断言并授予访问权限，完全绕过密码验证和所有MFA机制。
- **Blue View (防御缺失)**: 防御方未将身份基础设施（IdP）视为Tier 0资产进行隔离保护，导致被攻陷。其次，安全监控集中在网络边界，对身份联邦协议层面的异常访问缺乏可见性，未能监控到异常IP使用合法SAML断言登录，以及AD FS服务器上敏感证书导出行为。
- **策略借鉴**: 保护身份联邦基础设施：1）将IdP服务器划分为核心资产并实施极严物理和逻辑隔离；2）使用硬件安全模块（HSM）存储IdP签名私钥，防止被软件导出；3）建立针对SAML/OAuth活动的持续监控，识别令牌签发时间、地理位置异常，实施基于风险的动态访问控制。
