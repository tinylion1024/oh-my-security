# [针对高管VPN入口的MFA疲劳与社会工程学组合攻击]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 多因素认证绕过 / 凭证窃取 / 社会工程学
- **事件回顾**: 某科技巨头内网遭破坏，拥有高级权限的系统管理员身份被盗用。该账号虽配置了基于手机App的推送式MFA，但攻击者仍成功登入。调查显示，攻击者采用“MFA疲劳攻击（MFA Fatigue）”，通过连续不断认证轰炸并配合电话社工，迫使受害者按下同意按钮。
- **Red View (利用路径)**: 攻击者获取了管理员合法用户名和密码后，利用脚本尝试登录企业VPN，触发IdP向管理员手机发送MFA批准推送。攻击者短时间内连续发送上百次推送，同时伪装成IT服务台拨打受害者电话，声称系统故障需点击“同意”消除报错。受害者在疲惫和误导下确认，攻击者立即获得VPN访问权限。
- **Blue View (防御缺失)**: IAM系统采用体验简化的“一键同意”式MFA推送，未引入交互门槛。系统缺乏对异常高频MFA触发事件的速率限制和自动账号锁定机制。员工安全培训未能针对新型MFA绕过和电话钓鱼手段进行指导。
- **策略借鉴**: 强化MFA机制抗疲劳防钓鱼能力：1）升级MFA机制为“号码匹配（Number Matching）”模式，要求用户在App中输入屏幕显示的随机数字；2）实施自适应风险策略，检测异常连续MFA请求时自动阻断登录并告警；3）关键岗位部署支持FIDO2标准的抗钓鱼硬件安全密钥（如YubiKey）。
