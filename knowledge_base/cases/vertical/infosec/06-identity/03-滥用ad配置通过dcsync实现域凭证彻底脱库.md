# [滥用AD配置通过DCSync实现域凭证彻底脱库]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 活动目录攻击 / 凭证转储 / 域控制
- **事件回顾**: 某制造集团内网发生数据泄露，攻击者能以任何用户身份登录任何系统，实现对企业网络的“隐身统治”。安全团队发现，攻击者利用DCSync技术，伪装成域控制器，悄无声息地同步了Active Directory数据库中的所有用户密码哈希（包括KRBTGT账号哈希）。
- **Red View (利用路径)**: 攻击者获取内网低权限账号后，利用AD对象ACL配置过度授权漏洞，提权到具备 `Replicating Directory Changes` 权限的服务账号。利用此权限，攻击者使用Mimikatz发起DCSync攻击，向主域控发送合法目录复制RPC请求（DRSR协议）。域控误认其为合法同步请求，将包含敏感哈希的数据直接发送给攻击者机器。
- **Blue View (防御缺失)**: 蓝队对AD安全基线管理不善，历史遗留问题导致普通服务账号被错误赋予高危复制权限。网络和端点层面对DRSR协议通信缺乏监控，因其为合法系统管理流量，传统IDS未能将其标记异常。
- **策略借鉴**: 保护AD身份信任根基：1）严密清理AD对象的访问控制列表（ACL），移除错误赋予 `Replicating Directory Changes` 权限的非域控账号；2）使用高级身份威胁检测（ITDR）工具，专门监控从非真实域控IP发起的DCSync RPC行为；3）实施Tier 0隔离，限制管理员凭证在低安全级别主机暴露。
