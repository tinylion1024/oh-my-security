# [利用恶意第三方OAuth应用实现云邮箱权限持久窃取]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 应用层钓鱼 / 令牌窃取 / 授权滥用
- **事件回顾**: 某初创公司财务主管收到声称来自云端平台的“重要文档共享”邮件。点击链接并在官方页面通过密码和MFA登录后，页面提示“Secure Doc Viewer”第三方应用请求访问邮件。主管点击“授权”后，攻击者在无需获取密码和MFA情况下，成功持久窃取其邮箱内容数月。这是一起典型的OAuth同意钓鱼攻击。
- **Red View (利用路径)**: 攻击者在云平台注册合法第三方应用，构造极具欺骗性名称的授权链接通过钓鱼发送。受害者在官方页面登录（防御了传统凭证钓鱼），实际向恶意应用授予访问其数据资源的OAuth Token。受害者同意后，恶意应用获得合法访问令牌和刷新令牌，攻击者即可通过API在后台持久读取受害者云端资产，完全绕过MFA。
- **Blue View (防御缺失)**: 企业缺乏对第三方OAuth应用集成的严格管控。默认允许员工自主同意未经验证应用访问企业数据。安全团队对SaaS环境中活跃OAuth应用缺乏可见性，无法及时发现异常的高危权限请求（如大范围读取邮件/文件）。
- **策略借鉴**: 云时代第三方应用身份授权管控：1）在云IdP层面禁用用户自主授权，实行应用白名单制度，新接入请求需经安全审查；2）定期审计SaaS已授权OAuth应用清单，撤销未使用或权限过大的应用；3）配置警报监控具有高风险作用域（Scopes）的异常应用程序同意事件。
