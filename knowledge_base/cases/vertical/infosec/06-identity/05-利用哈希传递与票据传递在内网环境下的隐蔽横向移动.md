# [利用哈希传递与票据传递在内网环境下的隐蔽横向移动]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 凭证复用 / 横向移动 / 内存转储
- **事件回顾**: 某零售企业网络遭APT渗透，攻击者入侵前端客服电脑后，几天内悄无声息获取后台财务数据库服务器权限。取证表明，攻击者未使用漏洞利用攻击服务器，而是大量使用“哈希传递（PtH）”和“票据传递（PtT）”技术，利用内存中残留的高权限凭证进行身份伪装和横向移动。
- **Red View (利用路径)**: 攻击者攻陷初始终端后，从LSASS内存中转储了曾登录该机器的IT运维人员的NTLM哈希和Kerberos TGT。利用NTLM哈希直接进行PtH攻击，无需明文密码即可通过SMB对内网其他主机进行身份验证。随后利用高权限用户的Kerberos票据实施PtT攻击请求访问财务服务器。这种依赖操作系统原生认证机制的复用完全融入了正常业务流量。
- **Blue View (防御缺失)**: 网络架构和身份认证配置存在系统性缺陷。IT运维人员使用高权限账号登录低安全级别终端，导致凭证驻留在易受攻击机器内存。企业未启用Credential Guard保护LSASS进程，也未限制NTLM认证协议使用。
- **策略借鉴**: 阻断凭证复用以遏制横向移动：1）严格推行分层管理模型（Tiered Administration Model），禁止高权限域管理员账号登录普通工作站，消除凭证暴露；2）全面启用Windows Credential Guard，从内核层面阻止凭证盗取；3）逐步淘汰NTLM协议，转向Kerberos认证并开启强制Kerberos PAC验证。
