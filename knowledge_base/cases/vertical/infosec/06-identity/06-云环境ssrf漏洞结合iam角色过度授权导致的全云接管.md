# [云环境SSRF漏洞结合IAM角色过度授权导致的全云接管]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 云安全 / 配置错误 / IAM特权提升
- **事件回顾**: 某初创公司托管在AWS的核心客户数据库被勒索清空。攻击起点是边缘Web应用的SSRF漏洞，但根本原因在于云环境中IAM角色配置严重过度授权。攻击者利用漏洞获取Web服务器附加的IAM角色临时凭证，提升权限并接管整个云基础设施。
- **Red View (利用路径)**: 攻击者通过Web应用SSRF漏洞访问EC2实例元数据服务（IMDS），窃取绑定在实例上IAM角色的临时安全凭证。该角色本只需S3读取权限，但被配置了包含 `iam:PassRole` 等高危权限策略。利用窃取凭证，攻击者为自己创建了新的管理员权限策略，实现云原生环境下的特权提升。
- **Blue View (防御缺失)**: 蓝队云基础设施管理违反最小权限原则，角色被赋予过高权限且缺乏自动化审计。应用服务器未强制升级到更安全的IMDSv2，导致SSRF漏洞能轻易窃取临时凭证。
- **策略借鉴**: 云上安全即身份安全：1）严格贯彻IAM最小权限原则，使用云原生工具持续审查精简未使用权限；2）防范特权提升路径，严控涉及IAM策略修改和角色传递的敏感权限；3）在计算实例层强制使用要求带有会话Token的IMDSv2协议，阻断简单SSRF凭证窃取攻击。
