# [针对关键服务账号的Kerberoasting密码离线暴力破解]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 离线破解 / 服务账号滥用 / 活动目录攻击
- **事件回顾**: 某物流企业Active Directory遭隐蔽渗透。攻击者未产生大量登录失败警报，成功获取拥有域管理员权限数据库服务账号的明文密码。事件通过Kerberoasting高级身份攻击手法完成，攻击者利用Kerberos协议合法机制获取加密密码哈希，并在本地离线破解。
- **Red View (利用路径)**: 攻击者使用低权限域用户登录内网，向域控制器查询所有注册服务主体名称（SPN）的域账号。利用合法Kerberos机制，请求访问这些服务账号的服务票据（TGS）。域控使用目标服务账号的NTLM哈希对TGS加密并返回。攻击者拿到加密票据后导出，在配置强力GPU本地破解机上进行密码字典暴力破解，还原出明文密码。
- **Blue View (防御缺失)**: 服务账号管理存在漏洞。账号配置极高权限且密码策略薄弱（短于15个字符弱密码，长期未更改），大幅降低离线破解难度。网络监控缺乏对异常大批量SPN查询和TGS请求的警报能力。
- **策略借鉴**: 缓解Kerberoasting需从账号强度入手：1）对绑定SPN的服务账号实施极严格密码策略（超25字符完全随机强密码），使其在数学上极难被离线破解；2）推荐使用组托管服务账户（gMSA），由AD自动管理强密码定期轮换；3）监控网络中来自单一端点异常高频TGS-REQ行为。
