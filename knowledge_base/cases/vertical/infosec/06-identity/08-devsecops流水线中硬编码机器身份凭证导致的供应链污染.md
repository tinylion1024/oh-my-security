# [DevSecOps流水线中硬编码机器身份凭证导致的供应链污染]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 机器身份安全 / 密钥泄露 / 供应链安全
- **事件回顾**: 某知名软件提供商官方版本被植入恶意代码。调查确认攻击者未突破生产服务器，而是攻陷了CI/CD流水线。攻击者利用自动化构建脚本中硬编码的“机器身份”凭证，成功在代码编译打包阶段植入后门。
- **Red View (利用路径)**: 攻击者通过平台漏洞获取代码库只读权限。审计发现开发人员将云环境API Key、Docker Registry凭证和代码签名证书密码明文硬编码在 `Jenkinsfile` 等配置文件中。提取凭证后，攻击者在自身环境中直接调用云API或构建接口，以流水线合法身份篡改即将发布的二进制文件包。
- **Blue View (防御缺失)**: 企业对“机器身份”缺乏安全管理，过度依赖硬编码存储的长期静态凭证。CI/CD平台自身未对机器身份调用来源IP和异常行为进行审计，攻击者可从任意地点利用凭证。
- **策略借鉴**: 建立动态机器身份管理体系：1）绝对禁止代码硬编码敏感凭证，全面引入集中式机密管理平台（如HashiCorp Vault）进行动态读取；2）推动从长期凭证向短期动态凭证或基于OIDC的身份联邦转换，减少泄露窗口；3）对CI/CD执行环境实施强隔离，限制具有生产写入权限机器身份的调用网络。
