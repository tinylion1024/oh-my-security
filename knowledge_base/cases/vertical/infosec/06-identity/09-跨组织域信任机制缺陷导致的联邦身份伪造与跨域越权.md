# [跨组织域信任机制缺陷导致的联邦身份伪造与跨域越权]
- **所属领域**: 信息安全 - 身份安全
- **威胁类型**: 联邦信任滥用 / 跨域攻击 / 票据伪造
- **事件回顾**: 某集团收购子公司后建立双向AD域信任。数月后母公司机密泄露，由于子公司防御薄弱被攻陷，攻击者利用信任机制缺陷和配置漏洞，跨越信任边界伪造高权限身份进入母公司网络。
- **Red View (利用路径)**: 攻击者控制子公司AD后，提取域间信任身份验证关键密钥。利用此密钥发起跨域攻击（如伪造包含SID History的Kerberos票据）。构造一张由子公司域控签名的票据，在PAC部分隐蔽插入属于母公司“Enterprise Admins”组的安全标识符（SID）。因存在信任且未配置SID过滤，母公司验证时错误承认该高权限身份。
- **Blue View (防御缺失)**: 集团建立双向信任未充分评估风险，将子公司脆弱性引入母公司。配置信任时未启用关键的“SID过滤（SID Filtering）”机制，导致信任边界无法拦截伪造高权限SID注入。
- **策略借鉴**: 跨组织信任必须基于零信任最小授权：1）设计跨域/林信任时默认实施单向信任（子信任母），避免母公司直接暴露风险；2）强制启用并审核域间信任的“SID过滤”机制，防止伪造高权限跨域利用；3）针对第三方网络，推荐采用云IdP现代联邦身份验证，通过条件访问策略代替脆弱的底层AD信任。
