# 某头部社交平台GraphQL API批量越权导致百万用户隐私数据泄露事件
- **所属领域**: 信息安全 (应用安全)
- **威胁类型**: 越权访问 (IDOR / BOLA) / API接口滥用
- **事件回顾**: 2023年某知名头部社交平台爆出大规模数据泄露事件，近数百万用户的隐藏动态、私密联系人以及部分脱敏手机号被黑客在暗网打包出售。经过应急响应与溯源分析，发现数据泄露源于该平台近期上线的GraphQL API接口。开发人员在进行Schema设计与Resolver实现时，为了提高查询灵活性，允许客户端进行深度嵌套查询（Query Depth），且未能针对每一个节点层级进行严格的上下文权限校验。
- **Red View (利用路径)**: 攻击者首先注册了若干普通测试账号，在分析了GraphQL请求结构后，通过内省查询（Introspection Query）获取了完整的API Schema。随后，攻击者构造了高深度的嵌套查询语句（例如 `user -> friends -> private_posts -> comments -> author_details`），直接绕过了入口处的主身份校验。利用自动化脚本在单点API接口通过遍历User ID进行大规模的数据拉取。为了规避频控，攻击者利用了GraphQL的批量查询特性（Query Batching），在单个HTTP请求中包含上百个不同ID的查询组合，直接击穿了WAF层的速率限制机制。
- **Blue View (防御缺失)**: 
  1. 权限控制粒度过粗：GraphQL的权限校验仅仅停留在Query的根节点，未在底层数据获取的Resolver函数中实施基于数据所有者的访问控制。
  2. 深度与复杂度限制缺失：未限制GraphQL查询的深度（Depth）以及计算复杂度（Complexity），导致攻击者可以构造恶意嵌套。
  3. API网关策略滞后：传统WAF缺乏对GraphQL协议解析能力，无法解析Payload内部的批量请求操作，导致原有的基于IP+URI的限流策略完全失效。
- **策略借鉴**: 实施GraphQL接口的精细化鉴权，将权限校验逻辑下沉至数据模型层（Data Loader或业务逻辑层）；在API网关层部署支持GraphQL解析的防护插件，严格限制最大查询深度和复杂度；禁用生产环境的内省查询，实施基于AST解析的API请求限流策略。
