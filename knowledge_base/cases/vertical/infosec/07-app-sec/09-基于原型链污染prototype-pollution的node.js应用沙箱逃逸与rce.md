# 基于原型链污染(Prototype Pollution)的Node.js应用沙箱逃逸与RCE
- **所属领域**: 信息安全 (应用安全)
- **威胁类型**: 原型链污染 (Prototype Pollution) / RCE
- **事件回顾**: 一家提供在线代码执行与前端页面构建服务的SaaS平台遭受了深度的系统攻击。平台后台基于Node.js构建，其提供了一个受限沙箱环境供用户上传和解析复杂的JSON配置文件以动态生成UI组件。几天内，平台多台Worker节点服务器被黑客植入挖矿木马并执行了横向扩散。安全调查指出，该Node.js环境在使用第三方库（如 `lodash.merge` 或深拷贝工具的低版本）处理用户提交的嵌套JSON对象时，触发了原型链污染漏洞，进而实现了沙箱逃逸与远程代码执行（RCE）。
- **Red View (利用路径)**: 攻击者深入分析了平台提供的API接口，发现其存在深度合并（Deep Merge）JSON数据的操作。攻击者构建了包含特殊属性的恶意JSON负载，比如利用 `__proto__` 或者 `constructor.prototype` 键名。当后端使用存在漏洞的合并函数将该数据与本地对象合并时，恶意属性被直接注入到了全局的 `Object.prototype` 中。污染全局原型后，攻击者接着查找能够利用该污染变量的业务链（Gadget）。他们发现后台调用的模板引擎（如EJS或Pug）或 `child_process.exec` 的配置选项能够受被污染的原型属性影响，从而在渲染模板或创建子进程时，注入恶意的Shell命令，彻底击穿了代码沙箱。
- **Blue View (防御缺失)**: 
  1. 危险输入过滤缺失：在对不可信数据进行解析、克隆或合并操作时，未对 `__proto__`、`constructor` 等魔法属性进行拦截或剥离。
  2. 依赖组件老化：大量使用了已经曝出原型链污染CVE漏洞的老旧NPM依赖包，SCA（软件成分分析）机制运转不畅。
  3. 运行时环境保护不足：Node.js运行时未对全局对象的不可变性进行锁定（如使用 `Object.freeze(Object.prototype)`），也没有使用诸如 `--disable-proto` 类的安全启动标志。
- **策略借鉴**: 对用户输入的JSON数据必须进行严格的Schema校验并剥离原型链相关的危险属性；在应用初始化阶段可以考虑冻结核心全局对象的原型以防篡改；定期审计更新NPM依赖包至安全版本；在执行系统命令或沙箱代码时使用更严格隔离技术的微VM（如Firecracker）替代轻量级的应用层沙箱限制。
