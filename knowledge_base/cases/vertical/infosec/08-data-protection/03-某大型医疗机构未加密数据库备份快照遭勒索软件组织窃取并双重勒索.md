# 某大型医疗机构未加密数据库备份快照遭勒索软件组织窃取并双重勒索
- **所属领域**: 信息安全 (数据保护)
- **威胁类型**: 勒索软件 / 静态数据未加密 / 双重勒索
- **事件回顾**: 一家跨国医疗保健网络遭到知名勒索软件团伙（如LockBit或BlackCat）的攻击。医院核心HIS（医院信息系统）和PACS（医学影像系统）瘫痪，严重影响了患者的就医与急救。更恶劣的是，勒索团伙不仅加密了生产服务器，还声称已经窃取了超过2TB的高敏患者病历（包含诊断、处方及敏感疾病信息），并威胁若不支付赎金，将在其暗网数据泄露站点公开这些资料。溯源发现，黑客成功窃取数据的渠道并非直接攻击生产数据库，而是从文件服务器上拉取了定期生成的、完全明文的数据库备份快照。
- **Red View (利用路径)**: 攻击者首先通过一封带有恶意附件的钓鱼邮件突破了医院的外围防御，获得了普通员工的PC权限，并利用内网的横向移动技术（如Pass-the-Hash）逐步控制了域控服务器（AD）。在梳理内网资产时，攻击者发现了一台用于存储周期性备份数据的NAS（网络附加存储）服务器。该服务器上的数据库快照（.bak 和 .sql文件）由于性能考虑未采取任何加密措施，且文件共享权限配置宽松。攻击者使用内网工具将这些数百GB的明文备份文件打包，并利用隐蔽通道（如Rclone伪装成正常云同步流量）外传至攻击者的海外云服务器中。完成数据外带后，攻击者才大面积投放加密勒索负载，实施所谓的“双重勒索”。
- **Blue View (防御缺失)**: 
  1. 静态数据加密（Data at Rest Encryption）缺失：出于对系统备份速度的误解，IT团队禁用了备份存储层和数据库层的透明数据加密（TDE）功能，使得快照文件变成毫无防护的明文。
  2. 内网横向隔离薄弱：普通办公区网络与核心备份网络之间缺乏严格的微分段策略与访问控制，一旦域控失陷，攻击者即可畅通无阻地访问所有资产。
  3. 异常大流量外发监控失效：DLP（数据防泄露）与网络流量分析（NTA）设备未能识别并阻断高达2TB的异常出境流量，攻击者窃取数据如同探囊取物。
- **策略借鉴**: 必须对所有包含PII（个人身份信息）和PHI（受保护健康信息）的备份集实施高强度的静态数据加密（AES-256），并严格保护加密密钥；在网络边界与内网核心区域部署网络行为异常分析（NDR）设备，实时发现并阻断大容量数据的异常外带行为；实施基于网络微分段的零信任架构，收敛暴露面。
