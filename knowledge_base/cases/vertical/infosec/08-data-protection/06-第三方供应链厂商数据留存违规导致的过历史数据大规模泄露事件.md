# 第三方供应链厂商数据留存违规导致的过历史数据大规模泄露事件
- **所属领域**: 信息安全 (数据保护)
- **威胁类型**: 供应链安全 / 数据生命周期管理违规
- **事件回顾**: 某大型航空公司因客户数据泄露遭遇了史无前例的信任危机与GDPR罚款调查。黑客在暗网发布了数百万名乘客的航班行程、护照号码以及餐饮偏好等高敏信息。航空公司内部排查后确信自身核心数据库并未被攻破。随着调查深入，发现泄露源头系航空公司的某一家第三方营销与客户关系管理（CRM）供应商。该供应商在三年前合作结束后，并未按照合规要求彻底销毁这批历史数据，而是将其存留在一个缺乏维护的老旧Elasticsearch集群中，最终该集群因未设密码验证而被黑客勒索窃取。
- **Red View (利用路径)**: 攻击者常年使用诸如Shodan、Censys等网络空间测绘引擎扫描暴露在公网且存在配置缺陷的数据库服务。他们发现了一个暴露的9200端口，对应一个无任何身份鉴权的Elasticsearch集群。通过简单的REST API请求（如 `GET /_search`），攻击者轻而易举地查询到了包含大量高价值个人隐私的数据索引。经过对字段名称（如 `passport_number`, `flight_route`）的分析，攻击者判断出这是某航空公司的乘客数据集合。随后，他们利用脚本将整个集群的TB级数据导出并在黑市兜售。攻击者利用的不是高深的技术漏洞，而是供应商极其低劣的安全意识与违规的数据留存。
- **Blue View (防御缺失)**: 
  1. 第三方风险管理（TPRM）失控：航空公司在与供应商合作终止时，缺乏对第三方数据处理活动的审计监督，未能强制并验证其执行数据彻底销毁流程。
  2. 数据生命周期管理（DLM）缺失：数据的收集、存储、使用到销毁缺乏自动化与制度化的约束，历史废弃数据成为巨大的定时炸弹。
  3. 供应商基础安全能力低下：第三方供应商未能对其存储架构实施最基础的网络隔离和访问控制（如暴露内网服务至公网，未开启X-Pack安全认证机制）。
- **策略借鉴**: 建立严格的供应链安全评估与准入退出机制，在合同层面强化数据保护与销毁条款（要求提供具有法律效力的数据擦除证明）；应用数据生命周期管理策略，对留存数据设置定期自动化清理脚本或加密归档机制；倡导“隐私计算”或“联邦学习”等新技术，在不直接向第三方共享明文原始数据的前提下完成业务协同。
