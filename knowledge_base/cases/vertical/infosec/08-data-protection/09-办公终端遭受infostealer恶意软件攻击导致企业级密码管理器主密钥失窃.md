# 办公终端遭受Infostealer恶意软件攻击导致企业级密码管理器主密钥失窃
- **所属领域**: 信息安全 (数据保护)
- **威胁类型**: 恶意软件 (Infostealer) / 密钥泄露 / 凭据盗窃
- **事件回顾**: 某互联网大厂的基础架构遭遇毁灭性打击，不仅大量服务器的Root权限被窃取，其核心的客户数据库亦被严重拖库。复盘调查时，安全团队惊恐地发现，攻击者几乎掌握了内网所有核心系统的管理账号和密码。这一灾难源于该大厂的一名高级系统架构师在居家办公期间，使用个人PC（未接入企业标准终端安全管控）下载了一款被植入木马的盗版软件。该木马是一款高级信息窃取器（Infostealer如RedLine或Raccoon），它悄悄在后台窃取了架构师电脑中存储的企业级密码管理器（如1Password/KeePass）的本地缓存库以及刚刚输入的主控密码（Master Password）。
- **Red View (利用路径)**: 当架构师在被感染的PC上输入密码管理器主密码的那一刻，Infostealer利用其内置的键盘记录器（Keylogger）与内存抓取模块，精准捕获了主密钥。随后，木马将密码管理器在本地的加密Vault文件、浏览器中保存的各类Session Cookie以及系统中的SSH私钥一并打包，上传至黑客控制的C2（命令与控制）服务器。攻击者在自己的环境中恢复了密码管理器的全部内容，瞬间获得了数百个生产环境服务器的最高管理密码、数据库凭证和云平台API Keys。利用这些凭据和窃取的VPN Session，攻击者不仅绕过了外网MFA，更长驱直入控制了整个内网基础设施。
- **Blue View (防御缺失)**: 
  1. 终端安全边界失效：允许拥有极高权限的员工使用不受信任的、缺乏EDR（端点检测与响应）防护的个人终端（BYOD）直接访问核心办公资产。
  2. 零信任与设备强绑定缺失：企业密码管理器和VPN入口仅依赖于正确的账号密码，未能实现设备证书强绑定与设备健康状态评估（Device Posture Check）。
  3. 机密信息过度集中：密码管理器内存储了不该长久保存的最高级别访问凭据（如Root密码和超级AK），未实施即用即弃的动态访问控制（Just-In-Time Access）。
- **策略借鉴**: 全面推行严格的零信任架构（ZTA），将网络访问权限与设备安全基线深度绑定，未安装受控EDR的设备拒绝一切接入；对高级特权账号全面采用PAM方案并配置强MFA（如FIDO2硬件安全密钥），弃用静态密码的统一长期存储；强化员工的安全意识培训，防范日益猖獗的钓鱼软件及Infostealer威胁。
