# GitHub开源代码库硬编码AWS AK/SK导致云环境大规模被勒索
- **所属领域**: 信息安全
- **威胁类型**: 硬编码凭证泄露 / 资源劫持
- **事件回顾**: 某初创公司的一名新入职开发者在完成功能模块开发后，将代码推送到公司的公共GitHub仓库。代码中不慎包含了具有管理员权限的AWS Access Key和Secret Key。短短五分钟内，这一信息即被互联网上的自动化扫描机器人捕获。攻击者利用此凭证登录了该公司的AWS环境，迅速创建了数百个高配置EC2实例进行加密货币挖掘，并清除了原始数据备份，向公司勒索巨额赎金。
- **Red View (利用路径)**: 运行GitHub持续监控机器人，利用正则表达式实时抓取新提交代码中的AK/SK特征 -> 获取有效凭证后调用AWS API验证权限 -> 编写自动化脚本在全球多个区域并发启动最高规格GPU实例 -> 部署挖矿程序并尝试删除RDS/S3备份数据以实施勒索。
- **Blue View (防御缺失)**: 严重缺乏代码安全意识，未对公共代码库实施推送前的敏感信息阻断机制；云账号权限管理失控，开发人员持有全局Admin权限而非最小化权限；云环境未设置账单告警阈值和异常行为（如短时间大量开通高配实例）的实时预警机制。
- **策略借鉴**: 强制所有开发人员在本地Git环境中配置Pre-commit Hooks进行敏感信息检查（如使用git-secrets）；落实云端IAM最小权限原则，采用临时角色（Role）而非长效访问密钥；启用云平台账单异常告警与CloudTrail审计日志实时分析平台。
