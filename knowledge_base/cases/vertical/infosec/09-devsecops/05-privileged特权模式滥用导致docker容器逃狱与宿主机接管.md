# Privileged特权模式滥用导致Docker容器逃狱与宿主机接管
- **所属领域**: 信息安全
- **威胁类型**: 容器逃逸 / 特权滥用
- **事件回顾**: 某大数据处理平台为了解决容器内挂载网络文件系统（NFS）的权限问题，为特定计算节点上的Docker容器配置了`--privileged`特权运行模式。攻击者通过利用部署在该容器内的某个Web应用框架的反序列化漏洞，成功获取了容器内的Root权限。随后，攻击者发现容器为特权模式运行，能够直接访问宿主机的硬件设备。通过挂载宿主机的磁盘并向`cron`目录写入定时任务，攻击者成功逃逸到了宿主机，掌控了整个物理节点。
- **Red View (利用路径)**: 扫描目标Web应用并利用反序列化漏洞获取容器内RCE -> 查看进程和环境发现处于特权容器状态（如看到/dev全量设备）-> 执行`fdisk -l`识别宿主机根分区 -> 将宿主机根分区挂载至容器目录（如`/mnt/host`） -> 写入`/mnt/host/etc/crontab`或篡改`~/.ssh/authorized_keys` -> 等待定时任务执行，获取宿主机完全控制权。
- **Blue View (防御缺失)**: 错误地为了便利性而使用`--privileged`特权模式，完全打破了容器的隔离边界；Web应用程序缺乏有效漏洞修复与WAF防护，成为入侵突破口；宿主机缺乏对异常文件系统挂载与Cron篡改行为的主机层入侵检测（HIDS）监控。
- **策略借鉴**: 严格禁止在生产环境中使用`--privileged`运行容器，应使用细粒度的Linux Capabilities（如只授予`CAP_SYS_ADMIN`或特定能力）代替；启用AppArmor/Seccomp策略限制容器的系统调用；部署专门的容器安全运行平台（如Cilium或云原生安全平台CNAPP）进行进程级异常行为拦截。
