# Jenkins CI服务器不安全插件引发的远程代码执行(RCE)事件
- **所属领域**: 信息安全
- **威胁类型**: 组件漏洞 / RCE
- **事件回顾**: 2021年，某科技企业的自动化构建核心服务器Jenkins遭勒索软件攻击。安全团队排查确认，起因是Jenkins系统内部安装的某个第三方插件（如Git插件或某些UI组件）存在已公开的高危反序列化漏洞，且系统长期未进行补丁更新。攻击者通过发送构造好的恶意HTTP请求，触发了Jenkins服务器的远程代码执行。由于Jenkins以较高权限运行，攻击者不仅获取了CI/CD的全部源码，还向构建通道注入了勒索模块，导致全线业务停摆。
- **Red View (利用路径)**: 通过Shodan/Fofa定位暴露在公网的Jenkins管理后台 -> 使用公开的CVE漏洞PoC脚本探测插件漏洞 -> 发送恶意的序列化Payload进行漏洞利用 -> 获取Jenkins服务器Shell权限 -> 提取凭证管理库中的各类服务器密码 -> 横向扩散并投放勒索软件。
- **Blue View (防御缺失)**: Jenkins管理后台直接暴露在公网，缺乏VPC隔离与VPN准入控制；未建立CI/CD基础设施的定期漏洞扫描与补丁更新机制；Jenkins进程以系统高权限运行，一旦失陷影响范围极大。
- **策略借鉴**: 将核心CI/CD基础设施移至内网，强制通过VPN和多因素认证（MFA）进行访问；建立自动化的依赖组件扫描和定期系统更新制度；将Jenkins等构建平台降权运行，并在工作节点之间建立严格的微隔离防火墙策略。
