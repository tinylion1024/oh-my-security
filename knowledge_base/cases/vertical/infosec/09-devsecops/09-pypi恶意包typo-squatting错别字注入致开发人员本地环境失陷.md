# PyPI恶意包Typo-squatting(错别字)注入致开发人员本地环境失陷
- **所属领域**: 信息安全
- **威胁类型**: 恶意组件投毒 / 凭证窃取
- **事件回顾**: 在一次针对数据科学家和Python开发者的定点打击中，攻击者在Python包管理库（PyPI）中上传了数十个名称与流行科学计算库极为相似的恶意包（如将`requests`拼写为`requesst`，`pandas`拼写为`pandass`）。某研究机构的新员工在安装依赖时因拼写错误意外下载了恶意包。该恶意包在安装执行`setup.py`时，静默读取了开发者本地的`~/.ssh`和`~/.aws`目录，并将这些关键访问密钥打包加密后传出至攻击者的C2服务器。
- **Red View (利用路径)**: 搜集流行开源包的常见拼写错误和错别字 -> 编写含有恶意代码（主要是窃密脚本）的`setup.py`文件 -> 将其伪装成合法库并发布到官方PyPI注册中心 -> 等待粗心开发者安装触发恶意脚本 -> 接收传回的SSH私钥、云管凭证与环境变量。
- **Blue View (防御缺失)**: 开发者缺乏警惕，直接复制粘贴或手动敲击包名缺乏二次验证；企业内部未建立统一安全的私有源代理和第三方包准入机制；终端安全防护（EDR）未能拦截和识别安装脚本中可疑的文件系统越权访问外发行为。
- **策略借鉴**: 建立企业内部的私有包镜像源与白名单缓存机制，开发人员只能从审核过的内网源拉取依赖；引入软件依赖合规扫描平台，实时预警和阻断高风险或未知第三方库；在开发终端部署EDR以限制和监控非特权进程访问敏感凭据目录的行为。
