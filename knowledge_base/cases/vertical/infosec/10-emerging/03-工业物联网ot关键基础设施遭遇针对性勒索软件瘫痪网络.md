# 工业物联网(OT)关键基础设施遭遇针对性勒索软件瘫痪网络
- **所属领域**: 信息安全
- **威胁类型**: OT网络入侵 / 勒索软件
- **事件回顾**: 某国核心供水系统基础设施遭到APT组织的针对性攻击。由于该水厂的部分老旧可编程逻辑控制器（PLC）和人机界面（HMI）设备为了远程维护的便利，直接连接到了互联网，且运行着存在已知漏洞的旧版操作系统。攻击者通过利用RDP弱口令和OT设备漏洞侵入内网，并横向移动到关键的SCADA控制系统服务器。随后，攻击者部署了定制勒索软件，锁死了关键水处理控制系统的屏幕和数据库，导致水厂被迫切换到手动物理阀门操作模式，严重威胁了公众供水安全。
- **Red View (利用路径)**: 利用Shodan/Censys扫描暴露在公网的工业控制系统端口（如Modbus 502，RDP 3389） -> 通过爆破或利用已知CVE获取跳板机权限 -> 利用IT与OT网络隔离不严横向移动至SCADA控制中心 -> 下发勒索软件并窃取关键配置文档 -> 索要加密货币赎金。
- **Blue View (防御缺失)**: IT网络与OT（操作技术）网络未能实施严格的物理隔离或DMZ逻辑隔离；工控设备老旧且无法轻易打补丁，却暴露于公网；缺乏对工业协议（如Modbus/DNP3）的深度包检测（DPI）与异常指令审计体系。
- **策略借鉴**: 遵循Purdue普渡模型建立严格的IT与OT网络隔离与边界防火墙（工业网闸）；禁止任何不必要的工控设备公网暴露，远程运维必须强制使用多因素认证（MFA）的安全VPN；部署专用的OT网络异常行为分析系统，监测违规控制器指令下发。
