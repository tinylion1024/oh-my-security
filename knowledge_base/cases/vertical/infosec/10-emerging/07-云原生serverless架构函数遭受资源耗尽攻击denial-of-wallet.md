# 云原生Serverless架构函数遭受资源耗尽攻击(Denial of Wallet)
- **所属领域**: 信息安全
- **威胁类型**: 拒绝钱包攻击 / 资源耗尽
- **事件回顾**: 某初创互联网企业将后台业务完全迁移至AWS Lambda等Serverless架构，按调用次数和计算时间计费。由于开发人员在设计图片压缩处理函数时，未对上传图片的尺寸和调用频率进行严格限制。黑客利用僵尸网络发起了针对该API接口的“拒绝钱包（Denial of Wallet, DoW）”攻击。他们发送了海量构造的畸形超大压缩包触发该函数，导致Lambda函数并发数急剧飙升至极高阈值并长时间运行挂起。短短三天内，该企业收到了云服务商高达数十万美元的账单，面临破产危机。
- **Red View (利用路径)**: 分析目标应用的云端API接口，寻找计算密集型且未设置节流限制的Serverless端点（如图像处理、复杂查询） -> 利用代理池编写高并发分布式脚本 -> 持续发送引发高额计算消耗的恶意Payload -> 引发目标云函数的自动弹性扩容 -> 大量消耗目标企业的云服务账户余额，形成经济勒索或破坏。
- **Blue View (防御缺失)**: API网关处未能配置基于IP、用户身份的速率限制（Rate Limiting）和请求体积限制；Serverless函数内部未设置合理的超时时间（Timeout）与并发并发阈值上限；云平台未能配置严格的账单消费梯度告警（Billing Alerts）并与函数运行状态联动熔断。
- **策略借鉴**: 在云API网关层（如AWS API Gateway）实施严格的限流和WAF规则拦截异常请求流量；为所有的Serverless函数设置明确且合理的内存上限、超时时间和最大并发数以限制爆炸半径；全面建立云财务运营管控（FinOps），将账单异动告警与自动化资源冻结脚本相集成。
