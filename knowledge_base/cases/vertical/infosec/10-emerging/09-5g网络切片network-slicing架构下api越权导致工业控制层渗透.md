# 5G网络切片(Network Slicing)架构下API越权导致工业控制层渗透
- **所属领域**: 信息安全
- **威胁类型**: 网络切片攻击 / 5G安全
- **事件回顾**: 随着5G技术落地，某大型制造企业使用了电信运营商提供的5G网络切片技术，将高实时性的工业生产控制切片（OT）与普通的企业办公切片（IT）进行了逻辑隔离。然而，负责管理这些切片的5G核心网（5GC）开放控制API存在鉴权漏洞。攻击者首先通过钓鱼攻入了防守较弱的企业办公切片，随后在内网嗅探到了用于网络编排的5G网络开放功能（NEF）API接口。通过发送伪造的JWT令牌，攻击者成功越权调用了切片管理API，修改了路由策略，将办公切片的流量桥接到了高度敏感的工业控制切片，进而向生产流水线的机械臂下发了破坏性指令。
- **Red View (利用路径)**: 利用社工手段入侵相对薄弱的5G企业普通办公网终端 -> 内网横向移动并嗅探出5G核心网或边缘计算（MEC）的网元管理API接口 -> 利用API鉴权设计缺陷（如越权访问漏洞、未经验证的重放攻击）夺取切片编排控制权 -> 更改QoS策略和路由隔离表 -> 跨切片直达核心生产网络执行破坏行动。
- **Blue View (防御缺失)**: 5G核心网的开放API与边缘计算服务缺乏严格的零信任（Zero Trust）身份认证和细粒度API访问控制（API Security）；虽然物理上处于同一基站，但逻辑隔离的切片之间缺乏深度数据包检测和防火墙监控；切片编排和策略修改操作未建立多重认证和异常审计追踪机制。
- **策略借鉴**: 将零信任架构（ZTA）延展至5G切片管理层，对所有网元和API请求强制实施mTLS双向认证和短效动态令牌；部署API安全网关以监控和阻断针对5G NEF接口的越权和异常行为调用；在不同安全等级的切片之间必须叠加传统的应用层与网络层防火墙隔离策略，避免单一逻辑隔离失效。
