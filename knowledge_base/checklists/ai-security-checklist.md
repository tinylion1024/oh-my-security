# AI安全检查清单

> 10分钟快速检查你的AI应用是否存在致命风险

---

## 🔴 必须检查 (Critical)

### Prompt安全
- [ ] **用户输入隔离**: 用户输入是否与系统Prompt隔离？
- [ ] **注入检测**: 是否检测常见注入模式？
- [ ] **输出过滤**: 是否过滤敏感信息输出？
- [ ] **权限限制**: 是否限制AI的操作权限？

### API Key安全
- [ ] **密钥存储**: API Key是否存储在环境变量中？
- [ ] **密钥轮换**: 是否定期轮换API Key？
- [ ] **访问控制**: 是否限制API Key的访问范围？
- [ ] **使用监控**: 是否监控API Key使用情况？

### 数据安全
- [ ] **数据分类**: 是否识别敏感数据类型？
- [ ] **数据脱敏**: 发送给AI前是否脱敏？
- [ ] **数据保留**: 是否有数据保留策略？
- [ ] **数据删除**: 是否支持数据删除请求？

---

## 🟠 重要检查 (High)

### 成本控制
- [ ] **使用限额**: 是否设置API使用限额？
- [ ] **异常告警**: 是否监控异常使用？
- [ ] **预算控制**: 是否设置预算上限？
- [ ] **缓存策略**: 是否使用缓存减少调用？

### 模型安全
- [ ] **模型选择**: 是否选择合适的模型？
- [ ] **温度设置**: 温度参数是否合理？
- [ ] **输出限制**: 是否限制输出长度？
- [ ] **超时设置**: 是否设置请求超时？

---

## 🟡 建议检查 (Medium)

### 用户体验
- [ ] **幻觉检测**: 是否提示可能的幻觉？
- [ ] **引用来源**: 是否提供信息来源？
- [ ] **置信度**: 是否显示置信度？
- [ ] **人工审核**: 关键决策是否人工审核？

---

## 📋 快速自测命令

```bash
# 审计AI代码
oms ai audit ./src

# 检测Prompt注入
oms ai prompt-check "你的prompt"

# 检查API Key泄露
oms ai key-check ./src

# 查看最佳实践
oms ai guide
```

---

> 记住：**AI可以泄露你的数据、滥用你的预算、误导你的用户。**
