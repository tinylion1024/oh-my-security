# 应急响应检查清单

> 安全事件发生时，按此清单快速响应

---

## 🔴 立即行动 (0-15分钟)

### 事件确认
- [ ] **确认事件**: 是否确认安全事件已发生？
- [ ] **严重程度**: 事件严重程度是什么？
- [ ] **影响范围**: 影响了哪些系统/用户？
- [ ] **是否在蔓延**: 事件是否还在进行中？

### 紧急措施
- [ ] **隔离系统**: 是否隔离受影响的系统？
- [ ] **断开网络**: 是否需要断开网络连接？
- [ ] **停止服务**: 是否需要停止相关服务？
- [ ] **保护证据**: 是否保护日志和证据？

### 通知
- [ ] **通知团队**: 是否通知安全团队？
- [ ] **通知管理层**: 是否通知管理层？
- [ ] **通知用户**: 是否需要通知用户？
- [ ] **通知监管**: 是否需要通知监管机构？

---

## 🟠 短期处理 (1-4小时)

### 调查分析
- [ ] **攻击路径**: 攻击者是如何入侵的？
- [ ] **受影响资产**: 还有哪些资产受影响？
- [ ] **数据泄露**: 是否有数据泄露？
- [ ] **攻击者信息**: 能否识别攻击者？

### 恢复准备
- [ ] **备份检查**: 是否有干净的备份？
- [ ] **补丁准备**: 是否有安全补丁？
- [ ] **配置修改**: 需要修改哪些配置？
- [ ] **恢复计划**: 恢复计划是什么？

---

## 🟡 长期改进 (事件后)

### 复盘
- [ ] **根本原因**: 根本原因是什么？
- [ ] **改进措施**: 需要哪些改进措施？
- [ ] **责任归属**: 谁负责改进？
- [ ] **时间节点**: 何时完成改进？

### 文档
- [ ] **事件报告**: 是否完成事件报告？
- [ ] **经验教训**: 是否记录经验教训？
- [ ] **知识库更新**: 是否更新知识库？
- [ ] **培训材料**: 是否更新培训材料？

---

## 📋 快速自测命令

```bash
# 获取应急指南
oms incident guide data-breach

# 分类事件
oms incident classify "描述你的事件"

# 获取报告模板
oms incident template

# 列出所有事件类型
oms incident list
```

---

> 记住：**快速响应可以减少损失，拖延只会让事情更糟。**
