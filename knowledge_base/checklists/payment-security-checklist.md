# 支付安全检查清单

> 10分钟快速检查你的支付系统是否存在致命漏洞

---

## 🔴 必须检查 (Critical)

### 参数安全
- [ ] **金额来源**: 金额是否从后端数据库获取，不信任前端传递？
- [ ] **签名验证**: 所有支付参数是否参与签名验证？
- [ ] **时间戳校验**: 是否添加时间戳防止重放攻击？
- [ ] **参数顺序**: 签名时参数是否按固定顺序排列？

### 回调安全
- [ ] **签名验证**: 支付回调是否验证签名？
- [ ] **幂等处理**: 同一订单多次回调是否只处理一次？
- [ ] **来源验证**: 是否验证回调 IP 来源？
- [ ] **状态校验**: 是否验证订单状态允许更新？

### 敏感信息
- [ ] **密钥存储**: API 密钥是否存储在环境变量中？
- [ ] **日志脱敏**: 日志是否脱敏处理？
- [ ] **传输加密**: 是否使用 HTTPS？

---

## 🟠 重要检查 (High)

### 金额处理
- [ ] **整数运算**: 是否使用整数（分）处理金额？
- [ ] **边界检查**: 金额是否在合理范围内？
- [ ] **货币一致**: 交易前后货币是否一致？
- [ ] **负数检查**: 是否拒绝负数金额？

### 订单安全
- [ ] **订单唯一性**: 订单号是否全局唯一？
- [ ] **状态机**: 订单状态变更是否遵循状态机？
- [ ] **超时处理**: 未支付订单是否自动取消？
- [ ] **对账机制**: 是否定期与支付平台对账？

---

## 🟡 建议检查 (Medium)

### 监控告警
- [ ] **异常监控**: 是否监控异常支付行为？
- [ ] **金额告警**: 大额支付是否有告警？
- [ ] **失败告警**: 支付失败是否有告警？
- [ ] **对账告警**: 对账差异是否有告警？

### 业务安全
- [ ] **频率限制**: 支付接口是否有频率限制？
- [ ] **优惠券校验**: 优惠券使用是否有校验？
- [ ] **库存校验**: 是否校验库存充足？
- [ ] **用户限额**: 是否有用户支付限额？

---

## 📋 快速自测命令

```bash
# 审计支付代码
oms payment audit ./src

# 检查参数篡改风险
oms payment check --params ./src

# 检查回调安全
oms payment check --callback ./src

# 查看最佳实践
oms payment guide
```

---

## 🆘 发现问题怎么办？

1. **立即修复**: Critical 级别问题必须立即修复
2. **短期加固**: High 级别问题建议一周内修复
3. **持续改进**: Medium 级别问题纳入迭代计划
