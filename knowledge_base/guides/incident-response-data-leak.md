# 应急响应剧本：数据泄露处置 (Incident Response - Data Leak)

## 1. 黄金 60 分钟响应
一旦识别到疑似数据泄露，按照以下步骤执行：

### 阶段 1：确认与评估 (0-15m)
- **关键动作**: 确认泄露源头（API、数据库、内部员工）。
- **Agent 角色**: Lead Agent 收集初步特征，Red Agent 推演可能的泄露广度。

### 阶段 2：抑制与止损 (15-30m)
- **关键动作**: 修改泄露账户密码、重置 API Token、下线受影响节点或设置 WAF 阻断。
- **Agent 角色**: Blue Agent 提供最快阻断路径建议。

### 阶段 3：根因分析与取证 (30-60m)
- **关键动作**: 开启全量日志留存，进行流量镜像分析。
- **Agent 角色**: Red Agent 复现攻击路径，Compliance Agent 评估通报义务。

## 2. 后续治理
- **影响消除**: 对泄露数据进行失效处理（如注销关联卡片）。
- **加固建议**: 升级为脱敏输出或引入差分隐私。
