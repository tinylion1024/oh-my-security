# STRIDE 威胁建模方法论

- **目标**: 系统性识别软件架构中的潜在威胁。
- **领域**: 适用于所有软件系统的设计阶段。

## 1. 核心模型

| 字母 | 威胁类型 | 对应安全属性 | 描述 |
|------|----------|--------------|------|
| **S** | Spoofing (身份欺骗) | Authenticity (真实性) | 冒充他人身份或组件标识。 |
| **T** | Tampering (数据篡改) | Integrity (完整性) | 非法修改数据（存储或传输中）。 |
| **R** | Repudiation (否认) | Non-repudiability (不可抵赖性) | 否认曾执行过某项操作。 |
| **I** | Information Disclosure (信息泄露) | Confidentiality (机密性) | 敏感信息被非授权访问。 |
| **D** | Denial of Service (拒绝服务) | Availability (可用性) | 耗尽资源，使合法用户无法使用。 |
| **E** | Elevation of Privilege (特权提升) | Authorization (授权) | 获得超出其应有的权限。 |

## 2. 建模步骤

1. **绘制数据流图 (DFD)**: 识别外部实体、处理过程、数据流和数据存储。
2. **确定信任边界**: 标记数据从低信任区进入高信任区的边界。
3. **威胁识别**: 针对 DFD 中的每个元素，遍历 STRIDE 列表，识别潜在威胁。
4. **风险评估**: 使用 DREAD 或简单的高中低分级。
5. **消减建议**: 为每个确定的威胁设计防御措施。

## 3. 应用示例 (Web 登录模块)
- **外部实体 (用户)** -> **Spoofing**: 弱口令、Session 固定。
- **数据流 (密码传输)** -> **Information Disclosure**: 未加密传输、日志记录明文。
- **数据存储 (数据库)** -> **Tampering**: SQL 注入修改权限。
