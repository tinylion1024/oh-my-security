# 账号被盗

## 严重程度
🟠 高

## 识别信号
- 异常登录 IP 或地理位置
- 短时间内多次登录失败
- 用户报告未授权的活动
- 密码被重置但非本人操作
- 账户关联信息被修改（邮箱、手机号）
- 异常的密码修改请求
- 新设备登录提醒
- 异常的交易或操作记录

## 立即行动 (0-15分钟)

1. **确认事件**
   - 验证异常登录日志
   - 确认账户当前状态
   - 记录发现时间和方式

2. **保护账户**
   - 强制账户下线（撤销所有会话）
   - 重置用户密码
   - 暂停账户敏感操作权限
   - 启用账户保护模式

3. **通知相关方**
   - 通知账户所有者
   - 告知安全团队
   - 启动事件记录

4. **保留证据**
   - 截取登录日志
   - 保存操作记录
   - 记录 IP 地址和设备信息

## 短期处理 (1-4小时)

1. **调查分析**
   - 分析登录 IP 来源
   - 追踪异常操作记录
   - 确定账户被盗时间线
   - 评估影响范围

2. **扩展检查**
   - 检查同一 IP 登录的其他账户
   - 搜索相似攻击模式
   - 确认是否有批量撞库攻击
   - 检查关联账户安全

3. **恢复账户控制**
   - 验证账户所有者身份
   - 重置安全验证方式
   - 移除未授权的应用授权
   - 恢复账户正常状态

4. **沟通协调**
   - 向管理层汇报
   - 准备用户通知
   - 协调客服支持

## 恢复步骤

1. **账户安全加固**
   - 强制启用 MFA（多因素认证）
   - 设置登录异常检测规则
   - 添加敏感操作二次验证
   - 设置登录地域限制

2. **数据恢复**
   - 检查并恢复被篡改的数据
   - 回滚未授权的操作
   - 恢复账户关联信息

3. **监控验证**
   - 增强账户活动监控
   - 验证安全措施有效性
   - 确认无残留威胁

4. **用户教育**
   - 提供安全建议指导
   - 告知如何识别钓鱼
   - 提供可疑活动报告渠道

## 复盘要点

- [ ] 账号是如何被盗的？（撞库/钓鱼/弱密码/数据泄露）
- [ ] 为什么没有更早发现异常登录？
- [ ] 现有的登录异常检测是否有效？
- [ ] MFA 是否强制启用？
- [ ] 用户安全意识是否足够？
- [ ] 是否需要加强密码策略？
- [ ] 是否需要增加登录验证措施？
