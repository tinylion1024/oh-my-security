# 常见安全事件处理

> 6 类常见安全事件的详细处理流程

---

## 📋 事件类型总览

| 事件类型 | 典型场景 | 严重等级范围 | 处理重点 |
|---------|---------|-------------|---------|
| 账号被盗 | 异常登录、账号滥用 | P1-P3 | 快速止损、身份验证 |
| 数据泄露 | 数据库暴露、日志泄露 | P0-P2 | 评估影响、及时通知 |
| 支付异常 | 重复扣款、恶意订单 | P0-P2 | 止损、用户沟通 |
| API 被攻击 | DDoS、恶意调用 | P1-P3 | 限流、封禁、降级 |
| 服务器入侵 | 后门程序、挖矿病毒 | P0-P1 | 隔离、清理、加固 |
| 第三方漏洞 | 依赖漏洞、供应链攻击 | P1-P3 | 评估、修复、验证 |

## 🔐 账号被盗处理

### 事件识别

#### 识别信号

```yaml
用户报告信号:
  - 用户反馈账号异常
  - 用户报告无法登录
  - 用户发现异常操作
  - 用户收到未授权通知

系统检测信号:
  - 异常登录地点/设备
  - 短时间多次登录失败
  - 密码重置请求异常
  - 账号行为模式突变

外部通知信号:
  - 安全研究员报告
  - 监管机构通知
  - 其他平台关联事件
```

#### 确认检查清单

```markdown
□ 检查登录日志
  - 登录时间是否异常
  - 登录地点是否异常
  - 登录设备是否异常
  - IP 地址是否可疑

□ 检查账号活动
  - 是否有异常操作
  - 是否修改了敏感信息
  - 是否有数据导出
  - 是否关联了外部账号

□ 验证用户身份
  - 通过备用联系方式验证
  - 通过安全问题验证
  - 通过身份证明验证
```

### 遏制措施

#### 立即遏制（P1）

```yaml
时间目标: < 30 分钟

执行步骤:
  1. 立即冻结账号（5 分钟内）
     - 禁止登录
     - 使现有 session 失效
     - 禁止敏感操作

  2. 强制下线（5 分钟内）
     - 清除所有 session
     - 撤销所有 token
     - 禁用关联应用

  3. 通知账号所有者（10 分钟内）
     - 发送邮件通知
     - 发送短信通知（如有）
     - 提供恢复指引

  4. 保存证据（5 分钟内）
     - 导出登录日志
     - 导出操作日志
     - 保存相关截图
```

#### 控制遏制（P2/P3）

```yaml
时间目标: < 2 小时

执行步骤:
  1. 加强监控
     - 提高登录验证要求
     - 增加二次验证
     - 限制敏感操作

  2. 联系用户
     - 确认是否为本人操作
     - 了解具体情况
     - 提供安全建议

  3. 准备应急
     - 准备冻结方案
     - 准备恢复方案
     - 监控事态发展
```

### 恢复流程

#### 账号恢复步骤

```
Step 1: 身份验证
        ├─→ 通过注册邮箱验证
        ├─→ 通过绑定手机验证
        ├─=> 通过安全问题验证
        └─→ 通过身份证明验证（如需要）

Step 2: 安全清理
        ├─→ 强制修改密码
        ├─→ 清除所有授权应用
        ├─→ 重置安全问题
        ├─=> 撤销所有 API token
        └─→ 检查并清除可疑设置

Step 3: 账号恢复
        ├─→ 解除账号冻结
        ├─→ 恢复正常登录
        └─=> 提供安全建议

Step 4: 后续监控
        ├─→ 监控后续登录
        ├─=> 监控敏感操作
        └─→ 定期安全提醒
```

#### 恢复检查清单

```markdown
□ 身份验证完成
  - [ ] 已验证用户身份
  - [ ] 已确认是账号所有者

□ 安全清理完成
  - [ ] 已强制修改密码
  - [ ] 已清除授权应用
  - [ ] 已检查账号设置
  - [ ] 已撤销可疑操作

□ 账号恢复完成
  - [ ] 已解除冻结
  - [ ] 用户已成功登录
  - [ ] 功能恢复正常

□ 后续措施完成
  - [ ] 已提供安全建议
  - [ ] 已加强监控
  - [ ] 已记录处理过程
```

### 预防措施

| 措施 | 实施难度 | 有效性 | 优先级 |
|------|---------|-------|--------|
| 强密码策略 | 低 | 高 | 必须 |
| 二次验证（2FA） | 中 | 极高 | 强烈推荐 |
| 登录异常检测 | 中 | 高 | 推荐 |
| 设备绑定 | 高 | 高 | 可选 |
| 登录通知 | 低 | 中 | 推荐 |

## 📊 数据泄露处理

### 事件识别

#### 识别信号

```yaml
技术检测信号:
  - 数据库异常访问
  - 大量数据导出
  - API 异常调用
  - 日志中敏感数据明文
  - 备份文件异常访问

外部通知信号:
  - 安全研究员报告
  - 暗网监控发现
  - 用户报告信息泄露
  - 监管机构通知
  - 媒体报道

用户反馈信号:
  - 用户报告收到诈骗信息
  - 用户发现信息被滥用
  - 用户报告账号异常
```

#### 泄露类型判定

```yaml
存储介质泄露:
  - 数据库直接泄露
  - 备份文件泄露
  - 日志文件泄露
  - 缓存数据泄露

传输过程泄露:
  - 未加密传输
  - 中间人攻击
  - API 响应泄露

应用泄露:
  - 接口未授权访问
  - 越权访问
  - 注入攻击

人为泄露:
  - 内部人员泄露
  - 社会工程攻击
  - 第三方泄露
```

### 遏制措施

#### 立即遏制（P0）

```yaml
时间目标: < 15 分钟

执行步骤:
  1. 止血（5 分钟内）
     - 立即关闭泄露源
     - 断开相关网络连接
     - 撤销相关访问权限
     - 下线受影响服务

  2. 保存证据（5 分钟内）
     - 完整备份数据库
     - 导出访问日志
     - 保存网络流量
     - 记录时间线

  3. 初步评估（5 分钟内）
     - 确认泄露数据类型
     - 估算影响用户数量
     - 判断泄露时间范围
     - 确认是否持续泄露

  4. 启动通知流程
     - 准备用户通知
     - 准备监管报告
     - 准备公开声明
     - 联系法律顾问
```

#### 控制遏制（P1）

```yaml
时间目标: < 1 小时

执行步骤:
  1. 隔离受影响系统
     - 网络隔离
     - 服务降级
     - 访问控制加强

  2. 深入分析
     - 分析泄露路径
     - 确认攻击手法
     - 评估泄露范围

  3. 准备恢复
     - 制定修复方案
     - 准备安全加固
     - 准备通知模板
```

### 恢复流程

#### 数据泄露恢复步骤

```
Step 1: 根因修复
        ├─→ 修复泄露漏洞
        ├─→ 加固访问控制
        ├─=> 加强加密措施
        └─→ 清理泄露数据

Step 2: 安全加固
        ├─→ 加强身份验证
        ├─=> 加强访问控制
        ├─=> 加强加密存储
        ├─=> 加强日志审计
        └─→ 加强监控告警

Step 3: 用户通知
        ├─→ 准备通知内容
        ├─=> 发送用户通知
        ├─=> 提供保护建议
        └─→ 提供支持渠道

Step 4: 监管报告（如需要）
        ├─=> 准备报告材料
        ├─=> 提交监管报告
        └─→ 配合后续调查

Step 5: 恢复服务
        ├─=> 分阶段恢复
        ├─=> 加强验证测试
        └─=> 监控异常情况
```

#### 泄露影响评估模板

```yaml
    影响范围:

      直接损失: [金额]

      处理成本: [估算]
```

### 通知要求

#### 用户通知（必须）

```yaml
通知时机:
  P0: 发现后 24 小时内
  P1: 发现后 72 小时内
  P2: 评估后决定

通知内容:
  - 发生了什么
  - 涉及哪些数据
  - 可能的影响
  - 你正在做什么
  - 用户应该做什么
  - 如何获得帮助

通知方式:
  优先: 邮件通知
  补充: 站内信
  加强: 短信（如敏感）
```

#### 监管报告（如适用）

```yaml
报告条件:
  - 涉及用户个人信息
  - 影响用户数量较大
  - 可能造成用户损失
  - 法律法规要求

报告内容:
  - 事件描述
  - 影响评估
  - 应对措施
  - 补救方案
  - 防止复发措施

报告时限:
  - 欧盟 GDPR: 72 小时内
  - 中国网安法: 立即报告
  - 其他: 按当地法规
```

### 预防措施

| 措施 | 实施难度 | 有效性 | 成本 | 优先级 |
|------|---------|-------|------|--------|
| 数据加密存储 | 中 | 极高 | 中 | 必须 |
| 访问控制加强 | 中 | 高 | 低 | 必须 |
| 传输加密 | 低 | 高 | 低 | 必须 |
| 数据脱敏 | 中 | 高 | 低 | 推荐 |

| 定期审计 | 中 | 高 | 中 | 推荐 |

## 💳 支付异常处理

### 事件识别

#### 识别信号

```yaml
用户报告信号:
  - 用户报告重复扣款
  - 用户报告未授权交易
  - 用户报告无法支付
  - 用户报告退款异常

系统检测信号:
  - 支付成功率异常下降
  - 支付金额异常分布
  - 支付频率异常增加
  - 退款率异常上升
  - 欺诈检测结果异常

渠道通知信号:
  - 支付渠道异常通知
  - 银行风险预警
  - 监管机构通知
```

#### 异常类型判定

```yaml
系统异常:
  - 接口调用失败
  - 通知处理异常
  - 订单状态不一致
  - 对账异常

用户异常:
  - 用户误操作
  - 用户欺诈行为
  - 账号被盗用

第三方异常:
  - 支付渠道故障
  - 银行系统异常
  - 退款处理延迟

恶意攻击:
  - 恶意刷单
  - 支付接口攻击
  - 拒付欺诈
```

### 遏制措施

#### 立即遏制（P0/P1）

```yaml
时间目标: < 15 分钟

执行步骤:
  1. 止损（5 分钟内）
     - 暂停异常支付通道
     - 暂停可疑订单处理
     - 限制异常账号支付
     - 启用备用支付通道（如有）

  2. 保存证据（5 分钟内）
     - 导出支付日志
     - 保存异常订单
     - 记录支付渠道通知

  3. 初步评估（5 分钟内）
     - 确认异常类型
     - 估算影响金额
     - 确认影响用户数
     - 判断是否持续

  4. 通知相关方
     - 联系支付渠道（如需要）
     - 准备用户通知
     - 准备退款方案
```

#### 控制遏制（P2）

```yaml
时间目标: < 1 小时

执行步骤:
  1. 加强监控
     - 实时监控支付成功率
     - 监控异常订单
     - 加强欺诈检测

  2. 准备方案
     - 准备退款流程
     - 准备备用通道
     - 准备用户沟通

  3. 深入分析
     - 分析异常原因
     - 确认影响范围
     - 制定修复方案
```

### 恢复流程

#### 支付异常恢复步骤

```
Step 1: 分类处理
        ├─→ 系统异常：修复系统
        ├─→ 用户异常：沟通解决

        └─→ 更新风控规则

Step 4: 用户沟通
        ├─=> 发送通知解释
        ├─=> 提供退款方案
        ├─=> 提供补偿方案（如需要）
        └─→ 收集用户反馈

Step 5: 渠道协调（如需要）
        ├─=> 联系支付渠道

Step 6: 恢复服务
        ├─=> 分阶段恢复支付
        ├─=> 加强验证监控
        └─→ 跟踪后续情况
```

## 🔌 API 被攻击处理

### 事件识别

#### 识别信号

```yaml
流量异常信号:
  - API 请求量异常飙升
  - 单 IP/用户请求量异常
  - 请求来源异常分布
  - 请求时间模式异常

业务异常信号:
  - API 错误率异常上升
  - 响应时间异常增加
  - 业务逻辑异常调用
  - 数据异常查询/修改

资源异常信号:
  - 服务器 CPU/内存异常
  - 数据库连接数异常
  - 带宽使用异常
  - 费用异常增加

外部通知信号:
  - CDN/云服务商告警
  - 用户反馈服务异常
  - 监控系统告警
```

#### 攻击类型判定

```yaml
DDoS 攻击:
  - 流量型：大量请求耗尽带宽
  - 协议型：利用协议漏洞
  - 应用层：模拟正常请求

API 滥用:
  - 爬虫抓取
  - 恶意调用
  - 资源滥用
  - 数据批量获取

逻辑攻击:
  - 参数篡改
  - 越权访问
  - 注入攻击
  - 重放攻击

恶意注册:
  - 批量注册
  - 垃圾账号
  - 羊毛党
```

### 遏制措施

#### 立即遏制（P1）

```yaml
时间目标: < 30 分钟

执行步骤:
  1. 流量分析（5 分钟内）
     - 确认攻击类型
     - 识别攻击来源
     - 评估攻击规模
     - 确认影响范围

  2. 限流封禁（10 分钟内）
     - 启用全局限流
     - 封禁恶意 IP
     - 限制异常用户
     - 启用验证码

  3. 降级保护（10 分钟内）
     - 非核心接口降级
     - 启用缓存响应
     - 限制数据返回
     - 启用静态响应
```

## 🖥️ 服务器入侵处理

### 事件识别

#### 识别信号

```yaml
系统异常信号:
  - CPU 使用率异常飙升
  - 内存使用异常增长
  - 磁盘 I/O 异常
  - 网络流量异常

进程异常信号:
  - 可疑进程运行
  - 进程名伪装
  - 异常端口监听
  - 定时任务异常

文件异常信号:
  - 系统文件被修改
  - 可疑文件出现
  - 文件权限异常
  - 隐藏文件异常

用户异常信号:
  - 异常用户创建
  - 用户权限变更
  - SSH key 变更
  - 登录日志异常
```

#### 入侵类型判定

```yaml
挖矿病毒:
  特征: CPU 持续高占用
  行为: 运行挖矿程序
  目的: 窃取计算资源

勒索软件:
  特征: 文件被加密
  行为: 加密用户文件
  目的: 勒索赎金

后门程序:
  特征: 异常端口监听
  行为: 维持访问通道
  目的: 长期控制

Webshell:
  特征: Web 目录可疑文件
  行为: 通过 Web 执行命令
  目的: 远程控制

Rootkit:
  特征: 系统行为异常
  行为: 隐藏进程/文件
  目的: 深度隐藏控制
```

### 遏制措施

#### 立即遏制（P0）

```yaml
时间目标: < 15 分钟

执行步骤:
  1. 紧急隔离（5 分钟内）
     - 断开网络连接（如需要）
     - 隔离受影响服务器
     - 切断横向扩散路径

  2. 保存证据（5 分钟内）
     - 创建系统快照
     - 导出进程列表
     - 导出网络连接
     - 复制可疑文件
     - 导出系统日志

  3. 快速评估（5 分钟内）
     - 确认入侵类型
     - 确认影响范围
     - 确认数据影响
     - 判断入侵时间

  4. 启动恢复准备
     - 准备备份恢复
     - 准备替代服务
     - 准备通知模板
```

#### 入侵处理命令速查

```bash
# 查看进程（找出可疑进程）
ps auxf
top -c

# 查看网络连接（找出异常连接）
netstat -antup
ss -antup

# 查看监听端口（找出后门）
netstat -tulpn
ss -tulpn

# 查看用户登录（找出异常登录）
last
lastb
who

# 查看定时任务（找出持久化）
crontab -l
cat /etc/crontab
ls -la /etc/cron.*

# 查看SSH相关（找出后门）
cat ~/.ssh/authorized_keys
cat /root/.ssh/authorized_keys

# 查看最近修改文件（找出被改文件）
find / -mtime -1 -type f 2>/dev/null
find / -mmin -60 -type f 2>/dev/null

# 查看可疑文件（找出恶意文件）
find / -name "*.php" -mtime -7 2>/dev/null
find /tmp -type f 2>/dev/null

# 创建快照（保存证据）
tar -czf /tmp/incident_$(date +%Y%m%d_%H%M%S).tar.gz /var/log /etc

# 导出关键信息
ps auxf > /tmp/processes.txt
netstat -antup > /tmp/connections.txt
```
