# 沟通模板

> 各类安全事件通知和声明的标准模板

---

## 📋 模板使用指南

### 使用原则

```yaml
及时性:
  P0: 立即通知
  P1: 24小时内通知
  P2: 评估后决定
  P3: 通常不需要

准确性:
  - 描述事实，不隐瞒
  - 说明已知和未知
  - 避免技术术语
  - 提供清晰指引

同理心:
  - 承认影响和不便
  - 表达解决问题的决心
  - 提供帮助渠道
  - 保持后续更新

合规性:
  - 遵守相关法律法规
  - 满足监管报告要求
  - 保留沟通记录
  - 咨询法律意见（如需要）
```

### 模板结构

```
┌─────────────────────────────────────────────────────────┐
│                    沟通模板结构                          │
└─────────────────────────────────────────────────────────┘

1. 标题/主题
   - 清晰表明事件类型
   - 包含关键信息

2. 事件描述
   - 发生了什么
   - 何时发生的
   - 如何发现的

3. 影响说明
   - 影响范围
   - 影响程度
   - 持续时间

4. 应对措施
   - 已采取的措施
   - 正在进行的措施
   - 后续计划

5. 用户指引
   - 用户应该做什么
   - 用户可以获得什么帮助
   - 如何联系支持

6. 更新承诺
   - 后续更新时间
   - 更新渠道
   - 联系方式
```

```yaml
沟通记录:
  记录编号: COM-YYYY-MM-DD-XXX
  关联事件: INC-YYYY-MM-DD-XXX

  沟通信息:
    时间: YYYY-MM-DD HH:MM
    方式: [邮件/电话/会议/公告]
    类型: [内部通知/用户通知/公开声明/监管报告]
    发起人: [姓名]
    接收人: [姓名/群体]

  沟通内容:
    主题: [标题]
    要点:
      - [要点1]
      - [要点2]
      - [要点3]
    附件:
      - [附件1]
      - [附件2]

  反馈情况:
    反馈数量: [数量]
    主要问题:
      - [问题1]
      - [问题2]
    处理情况:
      - [处理1]
      - [处理2]

  后续安排:
    - [安排1]
    - [安排2]

  记录人: [姓名]
  记录时间: YYYY-MM-DD HH:MM
```

---

> 💡 **记住**: 危机沟通的核心是 **及时、准确、诚实、同理心**。用户需要知道发生了什么，你在做什么，他们应该做什么。
