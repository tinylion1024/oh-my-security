# 数据泄露

## 严重程度
🔴 严重

## 识别信号
- 异常数据库查询活动
- 大量数据导出请求
- 未授权的 API 调用
- 敏感数据出现在公开渠道
- 第三方通知数据泄露
- 异常的文件下载行为
- 数据库或存储访问异常
- 安全监控系统告警

## 立即行动 (0-15分钟)

1. **紧急止损**
   - 立即切断泄露源（关闭相关接口/服务）
   - 撤销可能导致泄露的访问凭证
   - 隔离受影响的数据库或存储
   - 阻止正在进行的泄露操作

2. **确认泄露**
   - 验证泄露数据的真实性
   - 确定泄露的数据类型和量级
   - 识别泄露途径
   - 记录泄露时间线

3. **通知升级**
   - 立即通知安全负责人
   - 启动危机响应团队
   - 通知法务和合规部门
   - 准备管理层汇报

4. **证据保全**
   - 截取相关日志
   - 保存网络流量记录
   - 记录所有操作和时间点
   - 禁止删除相关数据

## 短期处理 (1-4小时)

1. **影响评估**
   - 确定受影响的数据类型（PII/财务/商业机密）
   - 估算受影响的用户/客户数量
   - 评估数据敏感程度
   - 确定泄露范围

2. **调查溯源**
   - 分析泄露入口点
   - 追踪攻击者活动轨迹
   - 确定泄露原因（漏洞/配置错误/内部人员）
   - 收集证据链

3. **法律合规评估**
   - 评估监管报告要求
   - 确定通知时限
   - 咨询法律顾问
   - 准备合规文件

4. **遏制措施**
   - 修复导致泄露的漏洞
   - 加强访问控制
   - 增加数据访问审计
   - 部署临时防护措施

## 恢复步骤

1. **系统加固**
   - 修复根本漏洞
   - 加强身份认证和授权
   - 增强数据加密措施
   - 部署数据防泄漏 (DLP) 方案

2. **通知沟通**
   - 按法规要求通知监管机构
   - 通知受影响的用户/客户
   - 准备公关应对方案
   - 设立用户咨询渠道

3. **补偿措施**
   - 为受影响用户提供信用监控服务
   - 提供身份保护服务
   - 处理用户索赔
   - 提供安全建议

4. **长期改进**
   - 建立数据分类分级制度
   - 实施最小权限原则
   - 定期进行安全审计
   - 加强员工安全意识培训
