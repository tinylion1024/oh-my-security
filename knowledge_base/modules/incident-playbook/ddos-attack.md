# DDoS攻击

## 严重程度
🟠 高

## 识别信号
- 流量监控告警（带宽异常增长）
- 服务器 CPU/内存使用率激增
- 服务响应缓慢或不可用
- 大量半开连接（SYN Flood）
- 异常来源 IP 集中访问
- WAF/防火墙日志异常
- 连接数异常增长
- DNS 查询量异常

## 立即行动 (0-15分钟)

1. **确认攻击**
   - 验证是否为 DDoS 攻击
   - 确定攻击类型（流量型/协议型/应用层）
   - 评估攻击规模
   - 记录攻击开始时间

2. **启动防护**
   - 启用 DDoS 防护服务
   - 切换到高防 IP/CDN
   - 配置流量清洗规则
   - 联系 ISP 协助

3. **通知相关方**
   - 通知运维和安全团队
   - 联系 IDC/云服务商
   - 更新服务状态页面
   - 准备用户通知

4. **证据收集**
   - 记录攻击流量特征
   - 保存网络日志
   - 截取攻击数据包样本
   - 记录攻击源 IP

## 短期处理 (1-4小时)

1. **攻击分析**
   - 分析攻击类型和手法
   - 识别主要攻击源
   - 确定攻击目标（特定服务/IP）
   - 评估攻击规模和影响

2. **防护优化**
   - 调整防火墙规则
   - 配置 IP 黑名单
   - 启用 SYN Cookie
   - 限制连接速率
   - 过滤恶意流量

3. **服务降级**
   - 启用服务降级策略
   - 保障核心功能
   - 限制非关键服务
   - 设置访问排队

4. **协调沟通**
   - 与服务商协调防护
   - 定期更新状态
   - 向管理层汇报
   - 准备事后报告

## 恢复步骤

1. **解除攻击**
   - 确认攻击已停止
   - 逐步移除防护限制
   - 恢复正常流量路由
   - 解除服务降级

2. **服务恢复**
   - 验证所有服务正常
   - 恢复完整功能
   - 处理积压请求
   - 通知用户恢复

3. **安全加固**
   - 部署专业 DDoS 防护方案
   - 建立弹性扩展能力
   - 优化网络架构
   - 完善应急预案

4. **监控验证**
   - 加强流量监控
   - 验证防护有效性
   - 持续观察异常流量
   - 更新防护规则
