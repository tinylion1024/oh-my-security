# 支付欺诈

## 严重程度
🔴 严重

## 识别信号
- 异常高频交易
- 同一账户多卡支付
- 异常地区支付请求
- 退款率异常升高
- 支付渠道风控通知
- 用户举报盗刷
- 异常订单模式
- 测试性小额交易

## 立即行动 (0-15分钟)

1. **紧急止损**
   - 冻结可疑交易
   - 暂停涉事账户支付功能
   - 通知支付渠道风控团队
   - 撤销正在处理的欺诈交易

2. **确认欺诈**
   - 验证可疑交易真实性
   - 确认欺诈类型（盗刷/套现/虚假支付）
   - 估算潜在损失金额
   - 记录欺诈模式

3. **通知相关方**
   - 立即通知安全团队
   - 通知财务部门
   - 告知支付渠道
   - 启动事件记录

4. **证据保全**
   - 保存交易记录
   - 截取相关日志
   - 记录账户信息
   - 保存 IP 和设备信息

## 短期处理 (1-4小时)

1. **欺诈调查**
   - 分析欺诈交易模式
   - 追踪资金流向
   - 识别欺诈账户和关联账户
   - 确定欺诈规模

2. **扩展排查**
   - 检查相似交易模式
   - 搜索关联欺诈账户
   - 确定是否有团伙作案
   - 评估历史交易风险

3. **损失控制**
   - 联系支付渠道申请拒付
   - 冻结涉事账户资金
   - 阻止正在进行的欺诈
   - 启动追损程序

4. **沟通协调**
   - 向管理层汇报
   - 准备执法部门报案
   - 协调客服处理投诉
   - 准备用户通知

## 恢复步骤

1. **系统加固**
   - 增强交易风控规则
   - 部署欺诈检测模型
   - 加强身份验证措施
   - 增加人工审核环节

2. **损失挽回**
   - 向执法部门报案
   - 通过法律途径追索损失
   - 联系保险公司理赔
   - 处理渠道拒付

3. **业务恢复**
   - 解冻正常账户
   - 恢复支付服务
   - 加强监控验证
   - 验证风控有效性

4. **用户沟通**
   - 通知受影响用户
   - 处理用户投诉
   - 提供安全建议
   - 补偿受害用户
