# 第三方泄露

## 严重程度
🟠 高

## 识别信号
- 第三方通知数据泄露
- 发现数据在公开渠道出现
- 供应商安全事件报告
- 异常的第三方访问活动
- 供应商系统被入侵报告
- 合作伙伴通知异常
- 监管机构通知
- 安全监控发现异常数据流

## 立即行动 (0-15分钟)

1. **确认事件**
   - 验证第三方报告的真实性
   - 确定涉及的数据和系统
   - 联系第三方确认详情
   - 记录发现时间

2. **紧急止损**
   - 暂停与第三方的数据共享
   - 撤销第三方的访问权限
   - 更改相关凭证
   - 阻断数据传输

3. **通知升级**
   - 通知安全团队
   - 告知法务部门
   - 通知管理层
   - 启动事件记录

4. **信息收集**
   - 收集第三方泄露详情
   - 确定泄露数据范围
   - 了解泄露原因
   - 获取时间线

## 短期处理 (1-4小时)

1. **影响评估**
   - 评估泄露数据的敏感程度
   - 确定受影响的用户/业务
   - 分析潜在风险
   - 计算潜在损失

2. **合同审查**
   - 审查合同责任条款
   - 确认赔偿机制
   - 评估法律风险
   - 准备法律追索

3. **合规评估**
   - 评估监管报告要求
   - 确定通知时限
   - 准备合规文件
   - 咨询法律顾问

4. **沟通协调**
   - 与第三方协调处理
   - 准备用户通知
   - 准备监管报告
   - 协调公关应对

## 恢复步骤

1. **权限管理**
   - 全面审查第三方权限
   - 实施最小权限原则
   - 增加访问审计
   - 设置访问过期机制

2. **供应链安全改进**
   - 完善供应商安全评估流程
   - 加强合同中的安全条款
   - 实施数据共享最小化
   - 建立供应商监控机制

3. **用户保护**
   - 按法规通知受影响用户
   - 提供信用监控服务
   - 处理用户咨询和投诉
   - 提供安全建议

4. **长期改进**
   - 建立供应商安全评分体系
   - 定期进行供应商安全审计
   - 增强供应链风险监控
   - 完善应急响应流程

## 复盘要点

- [ ] 泄露是如何发生的？
- [ ] 第三方安全评估是否充分？
- [ ] 合同中的安全条款是否完善？
- [ ] 访问控制是否足够严格？
- [ ] 数据共享是否最小化？
- [ ] 供应商监控是否到位？
- [ ] 应急响应是否及时？
- [ ] 如何改进供应链安全管理？
