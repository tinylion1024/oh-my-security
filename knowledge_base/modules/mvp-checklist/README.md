# MVP 发布检查清单模块

> 模块路径：`/modules/mvp-checklist/`  
> 最后更新：2025-06-11  
> 适用阶段：产品上线前、安全审计、合规自查

---

## 模块概述

本模块提供 MVP（最小可行产品）发布前的全面安全检查清单，覆盖认证、数据、API、支付、基础设施等核心安全领域，帮助团队在上线前系统性地识别和修复安全风险。

### 核心价值

| 价值点 | 说明 |
|--------|------|
| **系统化检查** | 40+ 检查项，覆盖安全关键路径 |
| **可勾选清单** | 每项可勾选，便于团队协作追踪 |
| **风险评估** | 每项包含风险等级和修复建议 |
| **自动化脚本** | 提供一键检测脚本，提升效率 |
| **合规基础** | 涵盖数据保护、隐私政策合规要点 |

## 通过标准说明

### 评级体系

每个检查项使用以下评级：

| 等级 | 含义 | 处理要求 |
|------|------|----------|
| 🔴 **关键（CRITICAL）** | 必须修复，否则禁止上线 | 上线前必须通过 |
| 🟠 **高（HIGH）** | 强烈建议修复，存在明显风险 | 上线前必须通过或豁免审批 |
| 🟡 **中（MEDIUM）** | 建议修复，存在潜在风险 | 上线后 7 天内修复 |
| 🟢 **低（LOW）** | 最佳实践建议 | 上线后 30 天内优化 |

### 通过标准

**必须通过项目（100%）**：
- 所有 🔴 关键项
- 所有 🟠 高风险项

**建议通过项目（80%+）**：
- 🟡 中风险项至少 80% 通过
- 🟢 低风险项至少 60% 通过

### 豁免流程

如需对高风险项申请豁免：

1. 填写《安全风险豁免申请表》
2. 说明豁免理由和补救措施
3. 技术负责人 + 安全负责人双重审批
4. 豁免有效期不超过 30 天

| 10 | 日志不记录敏感数据 | 数据安全 | 🟠 |

## 使用建议

### 团队协作建议

1. **打印检查清单**：将核心检查项打印张贴，便于团队随时对照
2. **分工明确**：按角色分配检查项，专人负责专项
3. **记录留痕**：每次检查记录结果，便于审计追溯
4. **定期复审**：上线后每月复审一次，持续改进

### 工具支持

```bash
# 运行自动化安全检查脚本（示例）
./scripts/security-check.sh --full

# 仅检查关键项
./scripts/security-check.sh --critical

# 生成报告
./scripts/security-check.sh --report > security-report.md
```
