## 一、第三方服务风险评估

### 1.1 常见第三方服务类型

| 类型 | 示例 | 风险等级 | 共享数据 |
|------|------|----------|----------|
| 支付服务 | 支付宝、微信支付、Stripe | 🔴 高 | 订单信息、支付信息 |
| 身份认证 | 微信登录、QQ 登录、Auth0 | 🟠 高 | 用户身份信息 |
| 云服务 | 阿里云、AWS、腾讯云 | 🟠 高 | 业务数据、日志 |
| 数据分析 | 友盟、神策、Google Analytics | 🟡 中 | 用户行为数据 |
| 客服系统 | 美洽、网易七鱼、Zendesk | 🟡 中 | 对话记录、用户信息 |
| 短信服务 | 阿里云短信、腾讯云短信 | 🟡 中 | 手机号、短信内容 |
| 地图服务 | 高德地图、百度地图、Google Maps | 🟢 低 | 位置信息 |
| CDN 服务 | 阿里云 CDN、CloudFlare | 🟢 低 | 静态资源 |

```
├─────────────────────────────────────────────────────────────┤

│       │           │                                         │

│   │    风险等级矩阵         │                                 │
│   │                       │                                 │

```

#### 评估维度

| 维度 | 评估项 | 评分（1-5） |
|------|--------|------------|
| **数据敏感度** | 共享数据的敏感程度 | 1=公开, 5=机密 |
| **数据量** | 共享数据规模 | 1=少量, 5=大规模 |
| **访问权限** | 授予第三方的权限范围 | 1=只读, 5=完全控制 |
| **服务关键度** | 业务对第三方的依赖程度 | 1=可选, 5=必需 |
| **服务商可靠性** | 服务商安全能力和信誉 | 1=未知, 5=行业领先 |

#### 风险等级计算

```
风险分 = (数据敏感度 × 0.3) + (数据量 × 0.2) + (访问权限 × 0.25) + 
         (服务关键度 × 0.15) + (6 - 服务商可靠性) × 0.1

风险等级：
- 严重（红色）：4.0 - 5.0
- 高（橙色）：3.0 - 3.9
- 中（黄色）：2.0 - 2.9
- 低（绿色）：1.0 - 1.9
```

### 2.2 安全检查清单

#### 授权码安全

| # | 检查项 | 等级 | 状态 | 风险说明 | 快速修复 |
|---|--------|------|------|----------|----------|
| 1 | ☐ 使用 `state` 参数防 CSRF | 🔴 | | 授权码被盗用 | 生成随机 state，验证一致性 |
| 2 | ☐ 授权码只能使用一次 | 🟠 | | 授权码重放攻击 | 服务端标记已使用授权码 |
| 3 | ☐ 授权码有效期短（≤10分钟） | 🟠 | | 授权码被截获滥用 | 设置短有效期 |
| 4 | ☐ 使用 PKCE（移动端/SPA） | 🟠 | | 授权码被截获 | 实现 code_verifier/code_challenge |

#### Token 安全

| # | 检查项 | 等级 | 状态 | 风险说明 | 快速修复 |
|---|--------|------|------|----------|----------|
| 5 | ☐ Access Token 有效期合理 | 🟠 | | Token 泄露风险 | 短期 token（1-2小时） |
| 6 | ☐ 使用 Refresh Token 机制 | 🟡 | | 频繁重新授权 | 长期 refresh token + 安全存储 |
| 7 | ☐ Token 不暴露在 URL 中 | 🔴 | | Token 通过 Referer 泄露 | 使用 POST 或 Header 传递 |
| 8 | ☐ Token 安全存储 | 🔴 | | 客户端存储泄露 | HttpOnly Cookie 或安全存储 |

#### 回调安全

| # | 检查项 | 等级 | 状态 | 风险说明 | 快速修复 |
|---|--------|------|------|----------|----------|
| 9 | ☐ Redirect URI 白名单验证 | 🔴 | | 开放重定向攻击 | 精确匹配白名单 |
| 10 | ☐ Redirect URI 不包含通配符 | 🟠 | | 绕过白名单 | 使用精确路径 |

### 2.3 代码示例

#### 安全的 OAuth 授权请求

```javascript
// 生成随机 state 参数
const crypto = require('crypto');
const state = crypto.randomBytes(16).toString('hex');

// 存储 state 用于后续验证
session.oauthState = state;

// 构建授权 URL
const authUrl = new URL('https://oauth-provider.com/authorize');
authUrl.searchParams.append('client_id', process.env.OAUTH_CLIENT_ID);
authUrl.searchParams.append('redirect_uri', 'https://your-app.com/auth/callback');
authUrl.searchParams.append('response_type', 'code');
authUrl.searchParams.append('state', state); // 必须！
authUrl.searchParams.append('scope', 'openid profile email');

// 重定向用户
res.redirect(authUrl.toString());
```

#### 安全的回调处理

```javascript
app.get('/auth/callback', async (req, res) => {
    const { code, state } = req.query;
    
    // 1. 验证 state 参数
    if (!state || state !== req.session.oauthState) {
        return res.status(400).send('Invalid state parameter');
    }
    
    // 2. 清除已使用的 state
    delete req.session.oauthState;
    
    // 3. 使用授权码换取 token（后端对后端）
    const tokenResponse = await fetch('https://oauth-provider.com/token', {
        method: 'POST',
        headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
        body: new URLSearchParams({
            grant_type: 'authorization_code',
            code: code,
            redirect_uri: 'https://your-app.com/auth/callback',
            client_id: process.env.OAUTH_CLIENT_ID,
            client_secret: process.env.OAUTH_CLIENT_SECRET // 仅后端使用！
        })
    });
    
    const tokens = await tokenResponse.json();
    
    // 4. 安全存储 token
    // 使用 HttpOnly Cookie 存储 refresh token
    res.cookie('refresh_token', tokens.refresh_token, {
        httpOnly: true,
        secure: true,
        sameSite: 'strict',
        maxAge: 30 * 24 * 60 * 60 * 1000 // 30 天
    });
    
    // 5. 获取用户信息
    const userInfo = await fetch('https://oauth-provider.com/userinfo', {
        headers: { 'Authorization': `Bearer ${tokens.access_token}` }
    });
    
    // 6. 完成登录流程
    // ...
});
```

### 2.4 常见 OAuth 漏洞

| 漏洞 | 原因 | 攻击方式 | 防护措施 |
|------|------|----------|----------|
| CSRF 攻击 | 无 state 参数 | 诱导用户使用攻击者的授权码 | 使用随机 state 验证 |
| 开放重定向 | Redirect URI 通配符 | 重定向到攻击者网站 | 精确匹配白名单 |
| Token 泄露 | Token 在 URL 中 | 通过 Referer 头泄露 | 使用 POST 或 Header |
| 授权码重放 | 无使用标记 | 重复使用授权码 | 标记已使用授权码 |
| Token 窃取 | 客户端存储不安全 | XSS 攻击窃取 Token | HttpOnly Cookie |

### 3.3 代码示例

#### 订单创建（安全）

```javascript
// 后端创建订单
app.post('/api/orders', async (req, res) => {
    const { items } = req.body;
    const userId = req.user.id;
    
    // 1. 后端计算订单金额（不信任前端传入的金额）
    const amount = await calculateOrderAmount(items);
    
    // 2. 创建订单记录
    const order = await Order.create({
        userId,
        items,
        amount,
        status: 'pending',
        nonce: crypto.randomBytes(16).toString('hex') // 用于防重放
    });
    
    // 3. 获取支付参数
    const paymentParams = await paymentGateway.createPayment({
        orderId: order.id,
        amount: order.amount,
        nonce: order.nonce
    });
    
    res.json({
        orderId: order.id,
        paymentParams // 前端使用这些参数跳转到支付页面
    });
});
```

#### 支付回调（安全）

```javascript
app.post('/api/payment/callback', async (req, res) => {
    const { orderId, amount, status, sign } = req.body;
    
    // 1. 验证签名
    const expectedSign = calculateSignature(req.body, process.env.PAYMENT_SECRET);
    if (sign !== expectedSign) {
        return res.status(400).send('Invalid signature');
    }
    
    // 2. 查询订单
    const order = await Order.findById(orderId);
    if (!order) {
        return res.status(404).send('Order not found');
    }
    
    // 3. 验证金额一致
    if (order.amount !== amount) {
        return res.status(400).send('Amount mismatch');
    }
    
    // 4. 检查订单状态（幂等性）
    if (order.status === 'paid') {
        return res.send('Already processed'); // 已处理，直接返回成功
    }
    
    // 5. 更新订单状态（事务）
    await Order.update(
        { status: 'paid', paidAt: new Date() },
        { where: { id: orderId, status: 'pending' } } // 条件更新，防并发
    );
    
    // 6. 后续处理（发货、通知等）
    await fulfillOrder(order);
    
    res.send('Success');
});
```

### 3.4 支付对账机制

```javascript
// 定时对账任务
async function reconcilePayments() {
    const yesterday = new Date(Date.now() - 24 * 60 * 60 * 1000);
    
    // 1. 获取本地的已支付订单
    const localOrders = await Order.findAll({
        where: {
            status: 'paid',
            paidAt: { $gte: yesterday }
        }
    });
    
    // 2. 从支付网关获取交易记录
    const gatewayTransactions = await paymentGateway.getTransactions({
        startDate: yesterday,
        endDate: new Date()
    });
    
    // 3. 对比差异
    for (const order of localOrders) {
        const transaction = gatewayTransactions.find(t => t.orderId === order.id);
        
        if (!transaction) {
            // 本地有记录，网关无记录 -> 可能伪造
            alert(`Possible fraud: Order ${order.id} has no gateway transaction`);
        } else if (transaction.amount !== order.amount) {
            // 金额不一致
            alert(`Amount mismatch: Order ${order.id}`);
        }
    }
    
    // 4. 检查网关有但本地无的记录
    for (const transaction of gatewayTransactions) {
        const order = localOrders.find(o => o.id === transaction.orderId);
        if (!order) {
            alert(`Missing order: ${transaction.orderId}`);
        }
    }
}

// 每天执行一次
cron.schedule('0 2 * * *', reconcilePayments);
```

## 四、第三方 SDK 安全检查

### 4.1 SDK 安全评估

#### 评估维度

| 维度 | 检查项 | 权重 |
|------|--------|------|
| **来源可信度** | 官方源、社区评价、维护状态 | 25% |
| **权限需求** | 是否请求过多权限 | 20% |
| **数据收集** | 收集哪些数据、用途说明 | 20% |
| **安全历史** | 是否有已知漏洞 | 15% |
| **代码质量** | 代码审查、测试覆盖率 | 10% |
| **隐私合规** | 隐私政策、数据保护措施 | 10% |

### 4.2 安全检查清单

| # | 检查项 | 等级 | 状态 | 风险说明 | 快速修复 |
|---|--------|------|------|----------|----------|
| 1 | ☐ SDK 来源官方可信 | 🔴 | | 恶意 SDK | 仅从官方源安装 |
| 2 | ☐ 版本锁定，无已知高危漏洞 | 🔴 | | 供应链攻击 | `npm audit`、Snyk 检查 |
| 3 | ☐ 权限请求合理最小 | 🟠 | | 过度权限 | 仅授予必要权限 |
| 4 | ☐ 数据收集符合隐私政策 | 🟠 | | 隐私合规风险 | 更新隐私政策说明 |
| 5 | ☐ SDK 更新有安全审查流程 | 🟡 | | 引入新漏洞 | 审查更新日志、测试 |
| 6 | ☐ 移动端 SDK 无硬编码密钥 | 🔴 | | 密钥泄露 | 密钥动态下发 |
| 7 | ☐ SDK 有断网/禁用降级方案 | 🟡 | | 功能异常 | 降级处理 |

### 4.3 常见 SDK 风险

| SDK 类型 | 风险 | 防护措施 |
|----------|------|----------|
| 广告 SDK | 过度数据收集、隐私泄露 | 限制权限、数据脱敏 |
| 分析 SDK | 用户行为追踪 | 匿名化处理、告知用户 |
| 社交 SDK | 登录劫持、数据泄露 | 验证回调、最小权限 |
| 支付 SDK | 支付信息泄露 | 使用官方 SDK、不绕过 |
| 推送 SDK | 敏感推送内容泄露 | 不在推送中包含敏感信息 |
