# 运行时应用自防护 (RASP)

- **分类**: 预防 + 检测 + 响应
- **适用场景**: 解决 0-day 漏洞防护、无特征攻击检测、以及深度防御逻辑注入问题（如 SQL 注入、代码执行）。

## 工作原理
RASP 像插件一样运行在应用程序的运行环境（如 JVM, .NET Runtime, PHP 引擎）内部。它通过 Hook 关键的系统调用（文件操作、数据库访问、网络请求）来实时监测异常行为。

1. **上下文感知的检测**: 因为在应用内部，RASP 知道谁在调用 SQL，是否来自于不可信的用户输入，从而消除 WAF 常见的误报。
2. **逻辑阻断**: 发现异常调用（如：日志库突然尝试连接外部 LDAP）时，直接在内存层级阻断该调用。

## 执行建议
- **性能评估**: RASP 可能会带来 5%-15% 的 CPU 开销，高并发场景需测试。
- **推荐工具**: OpenRASP (百度开源), Imperva RASP, Contrast Security.
- **最佳实践**: 优先部署在核心 Java 业务网关和涉及敏感数据处理的后台。
