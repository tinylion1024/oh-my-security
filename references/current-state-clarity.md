# 现状清晰度评估 (Clarity Gate)

## 1. 评估维度
在开始正式分析前，Lead Agent 必须根据以下维度评估用户提供信息的质量 (1-100分)：

- **架构描述 (Architecture)**: 是否有明确的拓扑、组件、数据流向？
- **业务背景 (Context)**: 目标是什么？用户群体是谁？
- **技术栈 (Stack)**: 使用了什么编程语言、数据库、中间件？
- **安全现状 (Current Posture)**: 目前已有哪些安全措施？

## 2. 处理机制
- **分数 < 55 (信息缺失)**: Lead Agent 必须触发 `Question Bank` 中的针对性提问，拒绝进行深度分析。
- **55 <= 分数 < 75 (信息基本完整)**: 进行分析，但在报告中明确标注“基于假设”。
- **分数 >= 75 (信息清晰)**: 进行深度、高置信度的决策分析。
