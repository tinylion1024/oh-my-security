# 现状诊断问题库 (Question Bank)

当 `Clarity Gate` 评估用户提供的信息分数 **低于 55 分**（信息严重缺失）时，Lead Agent 必须暂停分析，并根据用户提到的初步意图，从以下问题库中选择 3-5 个最关键的问题进行追问。

## 1. 针对“信息安全/架构安全”的追问
如果用户只说“帮忙看看我们的系统安不安全”：

1. **部署环境**: 你们的系统是部署在公有云、私有云还是混合云？使用了哪些关键云服务（如 AWS S3, 阿里云 WAF）？
2. **鉴权机制**: 系统的身份认证和授权是如何实现的？（JWT? Session? OAuth2.0? 是否有 MFA？）
3. **数据流转**: 核心敏感数据（如 PII、支付信息）在传输和静止状态下是如何加密的？
4. **外部依赖**: 系统是否集成了第三方 SDK、开源组件或调用了外部 API？（是否有 SBOM？）

## 2. 针对“业务安全/风控”的追问
如果用户只说“我们被薅羊毛/爬虫攻击了”：

1. **利益触点**: 黑灰产攻击的具体业务接口是什么？（注册、领券、下单、抽奖？）他们能从中获利多少？
2. **当前防护**: 目前在客户端（App/Web）和网关层部署了哪些防护措施？（验证码？设备指纹？IP 频控？）
3. **攻击特征**: 攻击流量有什么明显特征？（如：集中在凌晨、使用特定的老版本 App、特定 IP 段？）
4. **业务容忍度**: 业务对“误伤真实用户”的容忍度如何？（宁可错杀一千，还是绝不误伤一个？）

## 3. 针对“AI 安全”的追问
如果用户只说“我们接了大模型，怕不安全”：

1. **集成方式**: 大模型是如何接入系统的？（API 调用、私有化部署、还是直接面向 C 端暴露？）
2. **输入/输出控制**: 用户的输入是否经过了预处理（如过滤恶意指令）？大模型的输出是否经过了后置审计？
3. **能力边界**: 该 AI 助手具备哪些系统操作权限？（能否读写数据库？能否调用外部 API 发送邮件？）
4. **数据投喂**: 是否将企业的私密数据通过 RAG（检索增强）或 Fine-tuning 的方式喂给了模型？
